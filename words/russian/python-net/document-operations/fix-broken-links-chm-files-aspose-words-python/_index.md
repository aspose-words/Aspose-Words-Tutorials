## Руководство по внедрению

Давайте разберем реализацию на ключевые этапы для оптимизации CHM-ссылок с использованием API Aspose.Words.

### Указание исходного имени файла с помощью ChmLoadOptions

**Обзор:**
Эта функция позволяет указать исходное имя файла .chm, гарантируя правильное разрешение всех внутренних ссылок.

#### Шаг 1: Импорт необходимых модулей
Начните с импорта `aspose.words` и `io`:

```python
import aspose.words as aw
import io
```

#### Шаг 2: Настройте параметры загрузки
Создать экземпляр `ChmLoadOptions` и задайте исходное имя файла:

```python
load_options = aw.loading.ChmLoadOptions()
load_options.original_file_name = 'amhelp.chm'
```
**Объяснение:**
Установка `original_file_name` помогает Aspose.Words точно разрешать ссылки в вашем CHM-файле, предотвращая появление неработающих URL-адресов.

#### Шаг 3: Загрузите и сохраните документ
Используйте эти параметры для загрузки документа .chm:

```python
doc = aw.Document(
    stream=io.BytesIO(system_helper.io.File.read_all_bytes(YOUR_DOCUMENT_DIRECTORY + 'Document with ms-its links.chm')),
    load_options=load_options
)
```
Сохраните его как HTML-файл, сохранив исправленные ссылки:

```python
doc.save(file_name=YOUR_OUTPUT_DIRECTORY + 'ExChmLoadOptions.OriginalFileName.html')
```
**Совет по устранению неполадок:**
Убедитесь, что путь к файлу .chm правильный и доступный. Если пути неверны, исправьте их соответствующим образом в коде.
