### Разрешение конфликтов в нумерации списков

#### Обзор
При объединении документов с идентичными пользовательскими форматами списков могут возникнуть конфликты номеров. Эта функция позволяет вам выбрать, сохранить ли исходную нумерацию или объединить их в непрерывную последовательность.

#### Пошаговая реализация

1. **Настройте свои документы**
   Клонируйте исходный документ для дальнейшей обработки.
   ```java
   Document srcDoc = new Document("Custom list numbering.docx");
   Document dstDoc = srcDoc.deepClone();
   ```

2. **Настроить параметры импорта**
   Использовать `ImportFormatOptions` для управления объединением документов.
   ```java
   ImportFormatOptions importFormatOptions = new ImportFormatOptions();
   importFormatOptions.setKeepSourceNumbering(true); // или false для объединения нумерации
   ```

3. **Настройка импортера узлов**
   Использовать `NodeImporter` для обработки операций на уровне узлов при импорте документа.
   ```java
   NodeImporter importer = new NodeImporter(srcDoc, dstDoc, ImportFormatMode.KEEP_DIFFERENT_STYLES, importFormatOptions);
   ```

4. **Импорт и добавление узлов**
   Пройдитесь по абзацам исходного документа и добавьте их в целевой документ.
   ```java
   for (Paragraph paragraph : srcDoc.getFirstSection().getBody().getParagraphs()) {
       Node importedNode = importer.importNode(paragraph, true);
       dstDoc.getFirstSection().getBody().appendChild(importedNode);
   }
   ```

5. **Обновить список меток**
   Убедитесь, что метки списка документа обновлены в соответствии с выбранной стратегией нумерации.
   ```java
   dstDoc.updateListLabels();
   ```
