# Включение предупреждений о шрифтах в Aspose.Words – Полное руководство по C#  

Задумывались когда‑нибудь, почему документ Word внезапно выглядит иначе после программной загрузки? **Enable Font Warnings** и вы сразу узнаете, когда Aspose.Words заменяет отсутствующий шрифт на запасной. В этом руководстве мы пройдем практический пример, который не только ловит эти замены, но и объясняет *почему* они происходят.

Мы расскажем обо всём, что нужно для начала работы: требуемый пакет NuGet, точную конфигурацию `LoadOptions` и аккуратный вывод в консоль, показывающий, какие шрифты были заменены. К концу вы получите надёжный, переиспользуемый шаблон для **C# document processing**, который работает с любой версией Aspose.Words.

## Что вы узнаете

- Как создать экземпляр `LoadOptions`, отслеживающий изменения шрифтов.  
- Назначение события `SubstitutionWarning` и как его подключить.  
- Полный, исполняемый пример кода, выводящий чёткие предупреждения в консоль.  
- Советы по обработке крайних случаев, например документов, содержащих только стандартные шрифты.  

Предыдущий опыт работы с Aspose.Words не требуется — достаточно базовых знаний C# и .NET.

## Шаг 1: Настройка LoadOptions и включение предупреждений о шрифтах

Первое, что вам нужно, — объект `LoadOptions`. Этот контейнер сообщает Aspose.Words, как обрабатывать файл, который вы собираетесь загрузить. Присвоив новый экземпляр `FontSettings`, вы открываете доступ к событиям, связанным со шрифтами.

```csharp
using Aspose.Words;
using Aspose.Words.Fonts;

// Step 1: Prepare load options and enable font substitution warnings
LoadOptions loadOptions = new LoadOptions
{
    // Attach a new FontSettings object – this is where warnings are emitted.
    FontSettings = new FontSettings()
};
```

**Почему это важно:**  
Если пропустить присвоение `FontSettings`, Aspose.Words всё равно будет заменять отсутствующие шрифты, но вы не получите уведомления. Механизм предупреждений находится внутри `FontSettings`, поэтому его инициализация *критически* важна для нашей цели.

> **Полезный совет:** Вы также можете указать `FontSettings` на пользовательскую папку со шрифтами, используя `SetFontsFolder`. Это уменьшит количество предупреждений, которые вы увидите, поскольку Aspose.Words действительно сможет найти недостающие гарнитуры.

## Шаг 2: Подписка на событие SubstitutionWarning (замена шрифтов)

Теперь, когда объект `FontSettings` существует, мы подключаемся к его событию `SubstitutionWarning`. Это событие срабатывает **каждый раз**, когда Aspose.Words заменяет запрошенный шрифт на другой.

```csharp
// Step 2: Subscribe to the SubstitutionWarning event to be notified when a font is replaced
loadOptions.FontSettings.SubstitutionWarning += (sender, e) =>
{
    // e.FontName – the name that the document asked for
    // e.SubstitutedFontName – the font that Aspose.Words actually used
    Console.WriteLine($"[Warning] Font \"{e.FontName}\" was substituted with \"{e.SubstitutedFontName}\".");
};
```

**Почему это важно:**  
Без этого обработчика вы не будете видеть процесс замены. Строка в консоли предоставляет быстрый журнал аудита, что особенно удобно при автоматических сборках или при генерации PDF для отраслей с высокими требованиями к соответствию.

> **Частый вопрос:** *Что если я хочу подавить предупреждения?*  
> Вы можете просто отсоединить обработчик или установить `FontSettings.SubstitutionWarning += null;`. Однако сохранение предупреждений обычно является самым безопасным вариантом, поскольку тихие замены могут привести к проблемам с разметкой.

## Шаг 3: Загрузка документа с настроенными параметрами (C# document processing)

С готовой системой предупреждений загрузка документа становится простой. Передайте экземпляр `LoadOptions` в конструктор `Document`, и Aspose.Words сделает остальное.

```csharp
// Step 3: Load the document using the configured options
string filePath = @"C:\Docs\DocumentWithMissingFont.docx";

Document doc = new Document(filePath, loadOptions);

// Optional: Save to PDF to see the visual impact of the substitution
doc.Save(@"C:\Docs\Output.pdf");
```

**Почему это важно:**  
Объект `LoadOptions` служит мостом между исходным файлом и инфраструктурой предупреждений. Если его опустить, документ загрузится тихо, и любые отсутствующие шрифты будут заменены без следа.

> **Крайний случай:** Некоторые документы встраивают точные файлы шрифтов, которые им нужны. В этом случае предупреждение не появится, потому что Aspose.Words находит встроенный шрифт. Приведённый выше код всё равно работает; вы просто увидите пустой вывод в консоли.

## Шаг 4: Проверка вывода и распространённые подводные камни

Запустите программу из командной строки или отладчика вашей IDE. Если исходный документ содержит шрифт, который не установлен на машине (или недоступен в пользовательской папке со шрифтами), вы увидите строки вроде:

```
[Warning] Font "Comic Sans MS" was substituted with "Arial".
[Warning] Font "MyCustomFont" was substituted with "Times New Roman".
```

Если ничего не выводится, то либо:

1. Все шрифты найдены, **или**  
2. Обработчик `SubstitutionWarning` не был правильно подключён (проверьте Шаг 2).

### Почему происходят замены шрифтов?

- **Отсутствующий системный шрифт:** ОС не имеет запрошенной гарнитуры.  
- **Неподдерживаемый формат шрифта:** Aspose.Words может читать TrueType и OpenType, но не каждый проприетарный формат.  
- **Ограничения лицензии:** Некоторые коммерческие шрифты блокируют встраивание, заставляя использовать запасной.

Понимание *почему* помогает решить, стоит ли поставлять недостающие шрифты вместе с приложением или скорректировать стили документа.

## Бонус: Управление запасным шрифтом

Если вы хотите, чтобы каждый отсутствующий шрифт заменялся на определённую гарнитуру (например, “Calibri”), вы можете задать глобальное правило замены:

```csharp
loadOptions.FontSettings.SubstitutionSettings.FontSubstitutes.AddSubstitutes(
    "AnyMissingFont", // wildcard – applies to any font not found
    new[] { "Calibri" } // the preferred substitute
);
```

Теперь консоль всё равно будет предупреждать, но визуальный результат будет одинаковым для всех отсутствующих шрифтов.

- Загрузите документ, используя настроенные параметры, и при желании сохраните в PDF, чтобы увидеть визуальный эффект.
