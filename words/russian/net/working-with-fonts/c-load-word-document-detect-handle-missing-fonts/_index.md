# c# load word document – Обнаружение и обработка отсутствующих шрифтов

Когда‑нибудь вам нужно было **c# load word document** и вы задавались вопросом, будет ли каждый шрифт отображаться корректно? Вы не одиноки. Отсутствующие шрифты — тихий виновник, который может превратить идеально отформатированный отчёт в беспорядочный набор символов.  

В этом руководстве мы пройдёмся по полному, готовому к запуску решению, которое **detects missing fonts** и **handles missing fonts** изящно, используя Aspose.Words for .NET. К концу вы точно будете знать, как обнаружить отсутствующие гарнитуры, записать полезные предупреждения и сохранить документ в отличном виде, даже если оригинальные шрифты отсутствуют на машине.

## Что вы узнаете

- Почему регистрация обработчика предупреждений является рекомендуемым способом выявления проблем со шрифтами.
- Практические советы по отладке проблем со шрифтами и предоставлению резервных шрифтов при необходимости.

## Шаг 1: Настройка LoadOptions для предупреждений о замене шрифтов

Когда вы **c# load word document**, Aspose.Words использует свой внутренний механизм настройки шрифтов. По умолчанию он тихо заменяет отсутствующие шрифты, что может скрывать проблемы. Чтобы заставить механизм «говорить», мы создаём экземпляр `LoadOptions` и присоединяем объект `FontSettings`.

```csharp
using Aspose.Words;
using Aspose.Words.Fonts;

// Create LoadOptions and enable font substitution warnings
LoadOptions loadOptions = new LoadOptions
{
    FontSettings = new FontSettings()
};
```

**Почему это важно:**  
Без этой настройки библиотека тихо заменяет отсутствующий шрифт на общий. Такая замена может изменить переносы строк, повлиять на макет и в конечном итоге нарушить визуальную точность вашего отчёта. Включение предупреждений даёт вам возможность записывать или реагировать на такие замены.

## Шаг 2: Регистрация обработчика предупреждений для обнаружения отсутствующих шрифтов

Aspose.Words генерирует событие предупреждения каждый раз, когда не может найти запрошенную гарнитуру. Подключив обработчик, мы можем захватить точное название отсутствующего шрифта и решить, что делать дальше.

```csharp
// Register a warning handler to report missing fonts
loadOptions.FontSettings.SubstitutionSettings.WarningHandler = (sender, args) =>
{
    // args.FontInfo may be null for some warnings, so we guard against it
    string missingFont = args.FontInfo?.FullFontName ?? "Unknown Font";
    Console.WriteLine($"[Font warning] Missing: {missingFont}");
};
```

**Pro tip:**  
Если вы планируете запускать это в веб‑службе, замените `Console.WriteLine` на подходящий фреймворк логирования (Serilog, NLog и т.д.). Так вы сохраните постоянный журнал того, какие шрифты отсутствуют на сервере.

## Шаг 3: Загрузка документа с использованием настроенных параметров

Теперь, когда инфраструктура предупреждений готова, мы наконец **c# load word document**. Конструктор `Document` принимает путь к файлу и `LoadOptions`, которые мы только что подготовили.

```csharp
// Load the document using the configured options
string inputPath = @"C:\Docs\input.docx"; // adjust to your file location
Document document = new Document(inputPath, loadOptions);
```

Если какой‑либо шрифт отсутствует, обработчик предупреждений из Шага 2 сработает *до* полной загрузки документа, предоставив вам полный список отсутствующих гарнитур.

## Шаг 4: Проверка вывода – чего ожидать

Запустите программу из консоли или юнит‑теста и наблюдайте вывод. Для каждого отсутствующего шрифта вы увидите строку вроде:

```
[Font warning] Missing: Times New Roman
```

Если все шрифты присутствуют, консоль будет молчать, а объект `document` будет готов к дальнейшей обработке (сохранение в PDF, редактирование и т.д.).

### Быстрый тест

Создайте небольшой файл Word, который использует шрифт, который вы знаете, что не установлен (например, “Papyrus”). Укажите `inputPath` на этот файл и выполните код. Вы должны увидеть напечатанное предупреждение, подтверждающее, что **detect missing fonts** работает как задумано.

## Шаг 5: Необязательно – Предоставить резервный шрифт

Иногда вы хотите, чтобы документ сохранял единый вид, даже если оригинальный шрифт недоступен. Aspose.Words позволяет сопоставить отсутствующие шрифты с резервным шрифтом по вашему выбору.

```csharp
// Map any missing font to Arial as a fallback
loadOptions.FontSettings.SubstitutionSettings.DefaultFontName = "Arial";
```

Добавьте эту строку *до* загрузки документа. Теперь, каждый раз, когда шрифт не найден, Aspose.Words автоматически заменит его на Arial, и вы всё равно получите предупреждение из Шага 2. Такой подход **handles missing fonts** без нарушения макета.

## Полный, готовый к запуску пример

Ниже приведена полная программа, которую вы можете скопировать и вставить в новое консольное приложение. Она включает все шаги, правильные директивы using и несколько дополнительных комментариев для ясности.

```csharp
using System;
using Aspose.Words;
using Aspose.Words.Fonts;

class Program
{
    static void Main()
    {
        // -------------------------------------------------
        // Step 1: Prepare LoadOptions with font settings
        // -------------------------------------------------
        LoadOptions loadOptions = new LoadOptions
        {
            FontSettings = new FontSettings()
        };

        // -------------------------------------------------
        // Step 2: Hook into the warning system to detect missing fonts
        // -------------------------------------------------
        loadOptions.FontSettings.SubstitutionSettings.WarningHandler = (sender, args) =>
        {
            string missingFont = args.FontInfo?.FullFontName ?? "Unknown Font";
            Console.WriteLine($"[Font warning] Missing: {missingFont}");
        };

        // -------------------------------------------------
        // Optional: Define a fallback font (handles missing fonts)
        // -------------------------------------------------
        loadOptions.FontSettings.SubstitutionSettings.DefaultFontName = "Arial";

        // -------------------------------------------------
        // Step 3: Load the Word file while using the options above
        // -------------------------------------------------
        string inputPath = @"C:\Docs\input.docx"; // change to your file path
        Document doc = new Document(inputPath, loadOptions);

        // -------------------------------------------------
        // Step 4: Save as PDF to verify everything works
        // -------------------------------------------------
        string outputPath = @"C:\Docs\output.pdf";
        doc.Save(outputPath);
        Console.WriteLine($"Document saved to {outputPath}");
    }
}
```

**Что делает этот код:**  
1. Настраивает `LoadOptions` для вывода предупреждений о замене шрифтов.  
2. Регистрирует обработчик, который выводит название каждого отсутствующего шрифта.  
3. (Опционально) заставляет любой неизвестный шрифт использовать Arial в качестве резервного.  
4. Загружает файл Word, записывает любые отсутствующие шрифты и в конце сохраняет результат в PDF.

Запустите программу, и вы увидите сообщения‑предупреждения, а затем «Document saved to …». Если открыть PDF, вы заметите, что любой отсутствующий шрифт заменён на Arial, сохраняя читаемость.

## Часто задаваемые вопросы и особые случаи

- **Что если `args.FontInfo` равно null?**  
  Некоторые предупреждения (например, когда файл шрифта повреждён) могут не предоставлять `FontInfo`. Наш обработчик защищает от этого, используя «Unknown Font» в качестве запасного варианта.

- **Работает ли это с файлами .doc?**  
  Да. Те же `LoadOptions` можно использовать для *.doc, *.docx, *.rtf и даже форматов OpenOffice. Просто измените расширение файла в `inputPath`.

- **Можно ли подавлять предупреждения для конкретных шрифтов?**  
  Вы можете добавить условную логику в обработчик предупреждений, чтобы игнорировать шрифты, которые намеренно отсутствуют.

- **Существует ли влияние на производительность?**  
  Нагрузка минимальна — Aspose.Words всё равно должен просканировать таблицу шрифтов документа. Обработчик предупреждений работает синхронно, поэтому он не замедлит обычную операцию загрузки заметно.
