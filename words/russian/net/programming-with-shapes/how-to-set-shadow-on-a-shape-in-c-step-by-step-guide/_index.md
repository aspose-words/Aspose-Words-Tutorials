# Как задать тень для фигуры в C# – Полный пошаговый пример

Когда‑то задавались вопросом **как задать тень** фигуре при программном создании документов Word? Вы не одиноки. В многих отчётах, презентациях или листовках лёгкая падающая тень может сделать графику более выразительной, не выглядя при этом дешёвой. Хорошая новость: с Aspose.Words for .NET вы можете добавить тень к фигуре всего в несколько строк кода.

В этом руководстве мы пройдём весь процесс: загрузим DOCX, получим первую фигуру и затем **применим тень к фигуре** — включая цвет, размытие, расстояние и угол. К концу вы получите готовый фрагмент кода, который можно вставить в любой C#‑проект. Никаких дополнительных библиотек, никакой скрытой магии.

## Что понадобится

- **Aspose.Words for .NET** (версия 23.9 или новее) — библиотека, упрощающая работу с Word.  
- Среда разработки .NET (Visual Studio 2022, Rider или CLI).  
- Пример DOCX, уже содержащий хотя бы одну фигуру (прямоугольник, изображение или SmartArt подойдёт).  

Если чего‑то не хватает, установите NuGet‑пакет командой `Install-Package Aspose.Words` и создайте простой Word‑файл с вручную вставленной фигурой — для демонстрации.

## Шаг 1: Загрузка документа (подготовка к добавлению тени)

Первое, что нужно сделать — открыть исходный файл. Здесь начинается операция **добавления тени к фигуре**.

```csharp
using System;
using Aspose.Words;
using Aspose.Words.Drawing;
using System.Drawing;

class ShadowDemo
{
    static void Main()
    {
        // Load the DOCX that holds the shape you want to enhance
        Document doc = new Document("input.docx");
```

> **Почему это важно:** Загрузка документа даёт вам объект `Document`, владеющий всеми узлами, включая фигуры. Без него нечего модифицировать.

## Шаг 2: Получение целевой фигуры (выбираем нужную)

Далее мы находим фигуру, которую собираемся стилизовать. В этом примере мы берём первую фигуру в первом абзаце, но запрос можно адаптировать под любую коллекцию узлов.

```csharp
        // Grab the first shape inside the first paragraph of the first section
        Shape targetShape = doc.FirstSection.Body.FirstParagraph
            .GetChildNodes(NodeType.Shape, true)[0] as Shape;

        if (targetShape == null)
        {
            Console.WriteLine("No shape found – check your input file.");
            return;
        }
```

> **Совет:** `GetChildNodes(NodeType.Shape, true)` рекурсивно проходит поддерево, гарантируя, что вы не пропустите вложенные фигуры, такие как WordArt.

## Шаг 3: Доступ к объекту форматирования тени (где живёт магия)

Каждая `Shape` раскрывает свойство `ShadowFormat`. Этот объект управляет видимостью, цветом, размытием, расстоянием и углом — все параметры, необходимые для **применения тени к фигуре**.

```csharp
        // The ShadowFormat object holds all shadow‑related settings
        ShadowFormat shadow = targetShape.ShadowFormat;
```

> **Зачем использовать `ShadowFormat`:** Он абстрагирует низкоуровневое XML‑представление, позволяя настраивать тени без работы с сырым OpenXML.

## Шаг 4: Делаем тень видимой и выбираем цвет (добавляем тень к фигуре)

Тень не появится, пока вы не установите `Visible` в `true`. После этого можно задать любой `System.Drawing.Color`. Здесь мы используем средний серый, но экспериментировать можно свободно.

```csharp
        // Turn the shadow on and give it a subtle gray tone
        shadow.Visible = true;
        shadow.Color = Color.FromArgb(80, 80, 80);   // dark gray
```

> **Распространённая ошибка:** Забвение включить `Visible` приводит к тихим сбоям — фигура выглядит без изменений, хотя остальные свойства заданы.

## Шаг 5: Настраиваем внешний вид — размытие, расстояние и угол (тонкая настройка)

Теперь формируем визуальное воздействие. `BlurRadius` смягчает края, `Distance` отодвигает тень от фигуры, а `Angle` определяет направление источника света.

```csharp
        // Adjust how the shadow looks
        shadow.BlurRadius = 5.0;   // in points – higher = softer
        shadow.Distance   = 3.0;   // how far the shadow is offset
        shadow.Angle      = 45.0;  // degrees clockwise from the horizontal
```

> **Особый случай:** Если задать отрицательное расстояние, тень появится *внутри* фигуры, что может быть полезно для эффекта тиснения.

## Шаг 6: Сохранение обновлённого документа (смотрим результат)

Наконец, записываем изменения на диск. Можно перезаписать исходный файл или создать новый.

```csharp
        // Persist the changes – you’ll see the shadow in Word or any viewer
        doc.Save("output-with-shadow.docx");
        Console.WriteLine("Shadow applied successfully! Check output-with-shadow.docx");
    }
}
```

Запуск программы создаёт `output-with-shadow.docx`. Откройте его в Microsoft Word, и вы увидите, что выбранная фигура теперь имеет мягкую серую тень под углом 45°, размытую на 5 пт и смещённую на 3 пт.

## Как добавить тень – типичные варианты и особые случаи

Хотя основные шаги просты, в реальных проектах часто требуются тонкие настройки. Ниже перечислены несколько «что‑если» ситуаций, с которыми вы можете столкнуться.

### 1. Несколько фигур, разные тени

Если в документе несколько графических объектов, пройдитесь по коллекции фигур и задайте уникальные параметры тени для каждой.

```csharp
        NodeCollection shapes = doc.GetChildNodes(NodeType.Shape, true);
        foreach (Shape shp in shapes)
        {
            ShadowFormat sf = shp.ShadowFormat;
            sf.Visible = true;
            sf.Color = Color.FromArgb(100, 100, 150); // bluish tint
            sf.BlurRadius = 3.0;
            sf.Distance = 2.0;
            sf.Angle = 30.0;
        }
```

### 2. Прозрачные тени

Aspose.Words позволяет задать альфа‑канал через `Color.FromArgb(alpha, r, g, b)`. Используйте небольшое значение альфа (например, 50) для лёгкого полупрозрачного эффекта.

```csharp
        shadow.Color = Color.FromArgb(50, 0, 0, 0); // 20% opacity black
```

### 3. Удаление тени

Иногда требуется отключить тень после её применения. Просто установите `Visible` в `false`.

```csharp
        shadow.Visible = false;
```

### 4. Проблемы совместимости

Используемые здесь функции тени поддерживаются в Word 2007 + (формат DOCX). Если вы работаете со старым бинарным форматом `.doc`, тень может быть проигнорирована, поскольку в нём отсутствуют необходимые XML‑элементы. В таких случаях рекомендуется сохранять как DOCX или использовать альтернативный визуальный приём.

- **Получили доступ** к её объекту `ShadowFormat`.  
- **Включили** тень, задали цвет, радиус размытия, расстояние и угол.  
- **Сохранили** новый файл, наглядно демонстрирующий эффект.  

Все эти шаги отвечают на вопрос **как задать тень** фигуре, а также показывают, как **добавить тень к фигуре**, **применить тень к фигуре** и даже **как добавить тень** в более сложных сценариях.

Каждая из этих тем базируется на тех же принципах объектной модели, что и в данном руководстве, так что вы будете чувствовать себя как дома.
