# Создание прямоугольной фигуры в Word с помощью C# – Полный учебник

Когда‑то вам нужно **создать прямоугольную фигуру** в документе Word, но вы не знали, с чего начать? Вы не одиноки — многие разработчики сталкиваются с тем же, когда хотят **добавить тень к фигуре** для более профессионального вида. В этом учебнике мы пошагово пройдем процесс **вставки фигуры в Word**, применим мягкую тень и, наконец, **c# generate word document** файлы, готовые к распространению среди пользователей.

Мы охватим всё: от настройки проекта до тонкой настройки свойств тени, и закончим готовым к запуску примером кода. Никакого лишнего, только практические детали, которые действительно работают.

## Что вы узнаете

- Как **создать прямоугольную фигуру** с помощью Aspose.Words (или Open XML) в C#  
- Какие именно свойства нужны, чтобы **добавить тень к фигуре** и придать глубину  
- Где разместить фигуру, используя `DocumentBuilder`  
- Как сохранить файл, чтобы он корректно открывался в Microsoft Word  
- Советы, подводные камни и варианты применения в реальных сценариях  

### Предварительные требования

- .NET 6.0 или новее (код работает на .NET Core и .NET Framework)  
- NuGet‑пакет, позволяющий работать с файлами Word – мы будем использовать **Aspose.Words for .NET**, потому что его API лаконичен. Если вы предпочитаете Open XML SDK, концепции те же, меняются только классы.  
- Visual Studio, VS Code или любой другой IDE для C#  

> **Pro tip:** Если бюджет ограничен, Aspose предлагает бесплатную пробную версию, идеально подходящую для обучения. Просто замените строку с лицензией на комментарий при тестировании.

## Шаг 1: Установите библиотеку для работы с Word

Сначала добавьте библиотеку в проект. Откройте терминал в папке решения и выполните:

```bash
dotnet add package Aspose.Words
```

Если вы используете Open XML SDK, команда будет `dotnet add package DocumentFormat.OpenXml`. Остальная часть руководства построена на Aspose.Words, но заменить вызовы API на Open XML достаточно просто.

## Шаг 2: Создайте новый пустой документ

Теперь, когда библиотека готова, мы можем **создать прямоугольную фигуру**, начав с чистого объекта `Document`. Представьте это как чистый холст.

```csharp
using Aspose.Words;
using Aspose.Words.Drawing;
using System.Drawing;

// Step 2: Initialize a blank Word document
Document document = new Document();
DocumentBuilder builder = new DocumentBuilder(document);
```

`DocumentBuilder` предоставляет высокоуровневый способ вставки содержимого без необходимости погружаться в низкоуровневые узлы.

## Шаг 3: Вставьте прямоугольную фигуру

Имея builder, мы можем **вставить фигуру в Word**. Метод `InsertShape` принимает тип фигуры и её размеры (ширина, высота) в пунктах.

```csharp
// Step 3: Insert a rectangle shape – 150pt wide, 80pt high
Shape rectangle = builder.InsertShape(ShapeType.Rectangle, 150, 80);
```

На этом этапе прямоугольник появляется в документе, но выглядит несколько плоско. Здесь на помощь приходит следующий шаг.

## Шаг 4: Добавьте тень к фигуре

Тени придают фигуре ощущение глубины. Объект `Shadow` позволяет точно настроить размытие, расстояние, угол, цвет и прозрачность. Ниже полная конфигурация, хорошо работающая для большинства отчетов.

```csharp
// Step 4: Configure a subtle shadow
rectangle.Shadow = new Shadow
{
    BlurRadius = 5.0,          // Soft edges
    Distance = 4.0,            // How far the shadow is offset
    Angle = 45,                // Direction in degrees (45° = down‑right)
    Color = Color.Black,       // Shadow color
    Transparency = 0.3         // 30 % transparent for a gentle look
};
```

**Почему именно такие значения?**  
- **BlurRadius** = `5.0` сохраняет гладкость края без излишней размытости.  
- **Distance** = `4.0` смещает тень достаточно, чтобы её было заметно.  
- **Angle** = `45` имитирует естественное освещение сверху‑слева, что часто используется в UI.  
- **Transparency** = `0.3` не позволяет тени перебить заливку фигуры.

Если нужен более драматичный эффект, увеличьте `BlurRadius` и уменьшите `Transparency`. Для почти незаметного подъёма наоборот — уменьшите `BlurRadius` и увеличьте `Transparency`.

## Шаг 5: Сохраните документ

Наконец, запишите файл на диск. Метод `Save` определяет формат по расширению файла, так что `.docx` сохраняет документ в современном формате Word.

```csharp
// Step 5: Persist the document
string outputPath = @"C:\Temp\ShadowRectangle.docx";
document.Save(outputPath);
```

Откройте `ShadowRectangle.docx` в Microsoft Word, и вы увидите чёткий прямоугольник с мягкой тенью — именно то, что вы искали, задавая вопрос «**how to add shape**» с профессиональной отделкой.

## Полный рабочий пример

Собрав всё вместе, получаем полностью готовую к запуску программу. Скопируйте‑вставьте её в консольное приложение и нажмите **F5**.

```csharp
using Aspose.Words;
using Aspose.Words.Drawing;
using System.Drawing;

namespace WordShapeDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // 1️⃣ Create a new blank document
            Document doc = new Document();
            DocumentBuilder builder = new DocumentBuilder(doc);

            // 2️⃣ Insert a rectangle shape (150pt × 80pt)
            Shape rect = builder.InsertShape(ShapeType.Rectangle, 150, 80);

            // 3️⃣ Add a subtle shadow
            rect.Shadow = new Shadow
            {
                BlurRadius = 5.0,
                Distance = 4.0,
                Angle = 45,
                Color = Color.Black,
                Transparency = 0.3
            };

            // 4️⃣ Save the file
            string filePath = @"C:\Temp\ShadowRectangle.docx";
            doc.Save(filePath);

            System.Console.WriteLine($"Document saved to {filePath}");
        }
    }
}
```

### Ожидаемый результат

- Сгенерированный `ShadowRectangle.docx` содержит **одну прямоугольную фигуру**, центрированную в месте положения курсора.  
- Прямоугольник отображает **мягкую, 30 % прозрачную черную тень**, смещённую под углом 45°.  
- Другого содержимого нет, файл лёгкий и его легко включать в более крупные отчёты.

## Часто задаваемые вопросы и особые случаи

### Что делать, если нужна другая фигура?

Замените `ShapeType.Rectangle` на любое другое значение перечисления `ShapeType` (например, `Ellipse`, `Triangle`). API тени работает одинаково, так что конфигурацию можно переиспользовать.

### Как изменить цвет заливки?

```csharp
rect.FillColor = Color.LightBlue;   // or any System.Drawing.Color
```

### Можно ли добавить фигуру в конкретный абзац?

Да. Переместите `DocumentBuilder` в нужный абзац с помощью `builder.MoveToParagraph(index)` перед вызовом `InsertShape`. Это гарантирует точное расположение фигуры.

### Что с более старыми форматами Word (.doc)?

Просто измените расширение:

```csharp
doc.Save(@"C:\Temp\ShadowRectangle.doc", SaveFormat.Doc);
```

Поддержка тени присутствует, начиная с Word 2003, так что эффект будет виден.

### Использовать Open XML SDK вместо Aspose?

Шаги остаются теми же: создайте `WordprocessingDocument`, добавьте элемент `Drawing`, задайте свойства `<a:shadow>`. XML будет более объёмным, но те же концепции (размер, размытие, расстояние, угол) применимы.

## Советы по избежанию подводных камней

- **Не забудьте лицензию**, если используете платную версию Aspose; иначе будет водяной знак.  
- **Единицы измерения — пункты**, а не пиксели. Один типичный экранный пиксель ≈ 0.75 pt, так что размеры следует корректировать.  
- **Свойства тени игнорируются**, если `WrapType` фигуры установлен в `Inline`. Используйте `WrapType = WrapType.Square` для плавающих фигур, которые учитывают тень.  
- **Сохранение на сетевой ресурс** может требовать соответствующих прав; всегда проверяйте путь заранее.
