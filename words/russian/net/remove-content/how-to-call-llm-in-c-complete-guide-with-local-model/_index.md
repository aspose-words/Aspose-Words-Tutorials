# Как вызвать LLM в C# – Полное руководство с локальной моделью

Когда‑то задумывались **как вызвать LLM** из .NET‑приложения, не отправляя данные в облако? Вы не одиноки. Многие разработчики хотят хранить свои подсказки и документы локально, особенно когда речь идёт о конфиденциальных текстах. В этом руководстве мы пройдём реальный сценарий: использование собственного LLM‑эндпоинта для переписывания Word‑документа, удаления всего содержимого, редактирования файла и, наконец, **как сохранить docx** обратно на диск.  

Мы также покажем, как **использовать локальный LLM**, предоставим точный код для **удаления всего содержимого** из `Document` библиотеки Aspose.Words и объясним нюансы программного редактирования Word‑файлов. К концу вы получите готовое решение «копировать‑вставить», которое работает с Aspose.Words 7+ и любой моделью, совместимой с OpenAI, размещённой локально.

## Предварительные требования – Что нужно перед началом

- **.NET 6+** (или .NET Framework 4.7.2, если предпочитаете классический вариант)
- NuGet‑пакет **Aspose.Words for .NET** (`Aspose.Words` и `Aspose.Words.AI`)
- **Локальный LLM**, предоставляющий совместимый с OpenAI `/v1` эндпоинт (например, сервер GPT‑Neo на `http://localhost:8000/v1`)
- Пример файла `input.docx`, размещённого в папке, которой вы управляете
- Visual Studio, Rider или любой другой редактор – в скриншотах я использую VS Code

> **Pro tip:** Если у вас ещё нет локальной модели, посмотрите бесплатный Docker‑образ для GPT‑Neo 2.7B – он поднимается менее чем за минуту и поддерживает тот же API‑контракт, который мы используем здесь.

## Шаг 1 – Настройка эндпоинта локального LLM (Как вызвать LLM)

Первое, что нужно сделать, когда вы хотите **как вызвать llm** из C#, – создать клиентский объект, указывающий на ваш собственный сервис. Aspose.Words.AI поставляется с помощником `LocalLargeLanguageModel`, который абстрагирует HTTP‑вызовы.

```csharp
using Aspose.Words;
using Aspose.Words.AI;

// Configure the self‑hosted LLM endpoint
var llm = new LocalLargeLanguageModel
{
    Endpoint = "http://localhost:8000/v1",   // your local server
    ModelName = "my-gpt-neo"                // name as registered in the server
};
```

> **Почему это важно:** Настраивая эндпоинт самостоятельно, вы сохраняете полный контроль над полезной нагрузкой запросов, аутентификацией и задержками. Это ядро **как вызвать llm** без зависимости от внешних сервисов.

## Шаг 2 – Загрузка исходного Word‑документа (Как редактировать Word)

Далее мы загружаем оригинальный `.docx` в объект Aspose `Document`. Это классический шаг «**как редактировать word**»: как только файл находится в памяти, вы можете выполнять запросы, модификацию или полностью заменять его содержимое.

```csharp
// Load the source document from disk
Document document = new Document("YOUR_DIRECTORY/input.docx");
```

Если файл не существует, будет выброшено `FileNotFoundException`, поэтому убедитесь, что путь указан правильно. При работе с загрузками можно также загрузить из `Stream`.

## Шаг 3 – Генерация исправленного текста с помощью локального LLM (Как вызвать LLM)

Теперь начинается магия: мы просим LLM переписать весь текст в формальном тоне. Промпт формируется конкатенацией короткой инструкции с «сырой» строкой, полученной через `document.GetText()`.

```csharp
// Ask the model to rewrite the whole document in a formal tone
string prompt = "Rewrite the following in formal tone:\n" + document.GetText();

string revisedText = llm.GenerateText(prompt);
```

> **Пограничный случай:** Если исходный документ огромный (более 10 k токенов), вы можете превысить лимит контекста модели. В этом случае разбейте текст на абзацы и вызывайте `GenerateText` для каждого куска.

## Шаг 4 – Удаление всего существующего содержимого (Remove All Content)

Прежде чем вставлять новый текст, нужно очистить документ. Aspose предоставляет `RemoveAllChildren()`, который удаляет секции, абзацы, таблицы — по‑настоящему всё. Это канонический способ **удалить всё содержимое** из Word‑файла.

```csharp
// Clear the document completely
document.RemoveAllChildren();
```

> **А если нужно удалить только тело, а заголовки оставить?** Используйте `document.Sections.Clear()` и затем восстановите нужные секции.

## Шаг 5 – Вставка исправленного текста (Как редактировать Word)

Имея чистый лист, мы записываем сгенерированный LLM текст обратно. `DocumentBuilder` — удобный обёртка, позволяющая добавлять абзацы, таблицы, изображения и т.д. Здесь мы просто записываем всю строку как один абзац.

```csharp
// Re‑populate the document with the revised text
DocumentBuilder builder = new DocumentBuilder(document);
builder.Writeln(revisedText);
```

Если требуется более сложное форматирование (жирный шрифт, заголовки), можно разобрать вывод LLM на маркеры markdown и соответственно настроить свойства `builder.Font`.

## Шаг 6 – Сохранение обновлённого документа (Как сохранить Docx)

Наконец, сохраняем изменения в новый файл. Это демонстрирует **как сохранить docx** после программных правок.

```csharp
// Save the edited document
document.Save("YOUR_DIRECTORY/output.docx");
```

Метод `Save` автоматически определяет формат по расширению файла, поэтому вы также можете экспортировать в PDF, HTML или ODT, изменив лишь одну строку.

### Ожидаемый результат

При открытии `output.docx` вы должны увидеть полностью переписанное оригинальное содержание в отшлифованном, формальном стиле. Никаких оставшихся таблиц, заголовков или колонтитулов из исходного файла — только свежий текст, сгенерированный LLM.

## Часто задаваемые вопросы и устранение неполадок

### 1. «Что делать, если мой LLM возвращает ошибку?»

Метод `GenerateText` бросает `HttpRequestException` при ответах, не являющихся 2xx. Оберните вызов в `try/catch` и проверьте `ex.Message`. Часто проблема — отсутствующий заголовок API‑ключа или превышение лимита токенов модели.

```csharp
try
{
    string revisedText = llm.GenerateText(prompt);
}
catch (HttpRequestException ex)
{
    Console.WriteLine($"LLM call failed: {ex.Message}");
    // fallback logic, e.g., return the original text
}
```

### 2. «Можно ли редактировать отдельные части документа, а не стирать всё?»

Конечно. Используйте `document.GetChildNodes(NodeType.Paragraph, true)`, чтобы перечислить абзацы, и заменяйте свойство `Paragraph.Text` только там, где нужны изменения. Такой подход позволяет **как редактировать word** на уровне отдельных элементов, сохраняя стили.

### 3. «Есть ли способ сохранить оригинальное форматирование?»

Если нужно сохранить стили, рассмотрите вариант возврата LLM‑вывода как простого текста, а затем применяйте `builder.Font.StyleIdentifier` к каждому абзацу согласно вашему шаблону. Альтернативно, используйте `DocumentBuilder.InsertHtml()`, если LLM может выдавать HTML.

### 4. «Как обрабатывать большие документы?»

Разбейте документ на секции (`document.Sections`) и обрабатывайте каждую отдельно. Это не только помогает избежать лимитов токенов, но и снижает нагрузку на память.

## Советы по производительности

- **Повторно используйте экземпляр `LocalLargeLanguageModel`** для нескольких вызовов; внутренний `HttpClient` будет поддерживать соединение живым.
- **Кешируйте сгенерированный текст**, если планируете многократно запускать один и тот же запрос — вызовы к LLM могут быть дорогими даже на локальном оборудовании.
- **Параллелизуйте** обработку секций с помощью `Parallel.ForEach`, если у вас многопроцессорный CPU и потокобезопасный клиент LLM.

## Следующие шаги – Расширение рабочего процесса

Теперь, когда вы знаете **как вызвать llm**, **использовать локальный llm**, **удалять всё содержимое**, **как редактировать word** и **как сохранить docx**, вы можете исследовать:

- **Пакетную обработку**: пройтись по папке с `.docx`‑файлами и применить ту же логику переписывания.
- **Пользовательские промпты**: настроить инструкцию для генерации резюме, маркированных списков или переводов.
- **Интеграцию с ASP.NET Core**: создать HTTP‑эндпоинт, принимающий загрузку файла, запускающий LLM и возвращающий отредактированный документ.
- **Продвинутое стилизование**: парсить markdown из LLM и сопоставлять его со стилями Word через `DocumentBuilder`.

Все эти расширения базируются на ядре, которое мы рассмотрели, так что адаптировать код будет просто.
