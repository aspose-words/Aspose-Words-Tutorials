# Ανάκτηση Κατεστραμμένου DOCX – Άνοιγμα & Φόρτωση Εγγράφου Word

Έχετε προσπαθήσει ποτέ να **ανακτήσετε κατεστραμμένο docx** και να συναντήσετε εμπόδιο επειδή το αρχείο απλώς δεν άνοιγε; Δεν είστε μόνοι. Σε πολλά πραγματικά έργα ένα κατεστραμμένο αρχείο Word μπορεί να σταματήσει μια ροή εργασίας, ειδικά όταν το έγγραφο περιέχει κρίσιμες συμβάσεις ή αναφορές. Τα καλά νέα είναι ότι το Aspose.Words σας παρέχει έναν απλό τρόπο να **ανοίξετε κατεστραμμένο docx** και να εκτελέσετε μια διαδικασία **load word document recovery** — όλα από την Python.

Σε αυτό το tutorial θα περάσουμε βήμα-βήμα όλα όσα χρειάζεται να γνωρίζετε: εγκατάσταση της βιβλιοθήκης, ρύθμιση της σωστής λειτουργίας ανάκτησης, φόρτωση του κατεστραμμένου αρχείου και, τέλος, επαλήθευση ότι το έγγραφο είναι ξανά χρησιμοποιήσιμο. Χωρίς ασαφείς αναφορές, μόνο ένα πλήρες, εκτελέσιμο παράδειγμα που μπορείτε να αντιγράψετε‑επικολλήσετε στο δικό σας έργο.

## Τι Θα Χρειαστείτε

- Python 3.8 ή νεότερο (ο κώδικας χρησιμοποιεί type hints, αλλά είναι προαιρετικά)
- Μία ενεργή συνδρομή Aspose.Words for Python ή κλειδί δοκιμής δωρεάν
- Η διαδρομή προς το κατεστραμμένο `.docx` που θέλετε να διορθώσετε
- Βασική κατανόηση των εισαγωγών Python και του χειρισμού εξαιρέσεων (αν έχετε γράψει ποτέ ένα `try/except`, είστε εντάξει)

Αυτό είναι όλο—χωρίς επιπλέον πακέτα, χωρίς χειρισμό εγγενών DLL. Το Aspose.Words διαχειρίζεται το βάρος εσωτερικά.

## Βήμα 1: Εγκατάσταση Aspose.Words for Python

Πρώτα απ' όλα, χρειάζεστε το πακέτο Aspose.Words. Ο πιο απλός τρόπος είναι μέσω `pip`:

```bash
pip install aspose-words
```

**Συμβουλή:** Αν εργάζεστε σε εικονικό περιβάλλον (συνιστάται έντονα), ενεργοποιήστε το πριν εκτελέσετε την εντολή. Αυτό διατηρεί τις εξαρτήσεις σας τακτοποιημένες και αποτρέπει συγκρούσεις εκδόσεων με άλλα έργα.

## Βήμα 2: Ρύθμιση LoadOptions για Ανάκτηση

Τώρα που η βιβλιοθήκη είναι διαθέσιμη, μπορούμε να ρυθμίσουμε τις επιλογές ανάκτησης. Η κλάση `LoadOptions` σας επιτρέπει να πείτε στο Aspose.Words πώς να συμπεριφέρεται όταν συναντά μια κατεστραμμένη δομή. Η πιο συνηθισμένη επιλογή είναι `RecoveryMode.RECOVER`, η οποία προσπαθεί να διασώσει όσο το δυνατόν περισσότερο περιεχόμενο.

```python
# Step 2: Import required classes and set up recovery
from aspose.words import Document, LoadOptions, RecoveryMode

# Create a LoadOptions instance
load_options = LoadOptions()
# Choose the recovery mode – RECOVER tries to fix the file
load_options.recovery_mode = RecoveryMode.RECOVER  # Options: RECOVER, THROW, IGNORE
```

**Γιατί είναι σημαντικό:**  
- **RECOVER** – Προσπαθεί να ξαναχτίσει το έγγραφο, παραλείποντας τα μη αναγνώσιμα μέρη.  
- **THROW** – Εγείρει εξαίρεση στην πρώτη ένδειξη προβλήματος (χρήσιμο για εντοπισμό σφαλμάτων).  
- **IGNORE** – Παραλείπει σιωπηλά τα κατεστραμμένα τμήματα, κάτι που μπορεί να αφήσει ένα ελλιπές αρχείο.

Για τις περισσότερες παραγωγικές περιπτώσεις, το `RECOVER` προσφέρει την καλύτερη ισορροπία μεταξύ διατήρησης δεδομένων και σταθερότητας.

## Βήμα 3: Φόρτωση του Κατεστραμμένου Εγγράφου

Με τη λειτουργία ανάκτησης ρυθμισμένη, η φόρτωση του κατεστραμμένου αρχείου γίνεται παιχνιδάκι. Δώστε τη διαδρομή προς το κατεστραμμένο `.docx` και το `LoadOptions` που μόλις διαμορφώσατε.

```python
# Step 3: Load the (potentially corrupted) DOCX
corrupted_path = r"C:\path\to\your\corrupted.docx"

try:
    doc = Document(corrupted_path, load_options)
    print("✅ Document loaded successfully – recovery mode applied.")
except Exception as e:
    print(f"❌ Failed to load document: {e}")
```

Αν το αρχείο είναι πραγματικά μη αναγνώσιμο, το Aspose.Words θα προσπαθήσει ακόμη να ανασυνθέσει τα τμήματα που μπορεί. Το μπλοκ `try/except` εξασφαλίζει ότι θα λάβετε ένα σαφές μήνυμα αντί για ένα ακατανόητο stack trace.

## Βήμα 4: Επαλήθευση και Αποθήκευση του Ανακτημένου Αρχείου

Μετά τη φόρτωση, θα θέλετε να βεβαιωθείτε ότι το έγγραφο φαίνεται σωστό. Ένας γρήγορος τρόπος είναι να το αποθηκεύσετε σε νέα τοποθεσία και να το ανοίξετε στο Microsoft Word (ή οποιονδήποτε συμβατό προβολέα). Μπορείτε επίσης να ελέγξετε προγραμματιστικά τους αριθμούς κόμβων, τις παραγράφους ή τις εικόνες.

```python
# Step 4: Save the recovered document for verification
recovered_path = r"C:\path\to\your\recovered.docx"

# Save in the same format (DOCX) – you could also choose PDF, HTML, etc.
doc.save(recovered_path)

print(f"💾 Recovered file saved to: {recovered_path}")
```

**Αναμενόμενο αποτέλεσμα:**  
- Το νέο `recovered.docx` ανοίγει χωρίς την προειδοποίηση “το αρχείο είναι κατεστραμμένο”.  
- Τα περισσότερα από το αρχικό κείμενο, η μορφοποίηση και οι εικόνες διατηρούνται.  
- Οποιαδήποτε τμήματα που ήταν ακατόρθωτα παραλείπονται — τίποτα δεν καταρρέει την εφαρμογή σας.

## Προαιρετικό: Προγραμματισμένοι Έλεγχοι (Ασφαλές Άνοιγμα Κατεστραμμένου DOCX)

Αν χρειάζεται να αυτοματοποιήσετε τον έλεγχο ποιότητας — π.χ., σε μια αλυσίδα επεξεργασίας παρτίδας — μπορείτε να ερωτήσετε τη δομή του εγγράφου μετά τη φόρτωση:

```python
# Example: Count paragraphs to ensure content was recovered
paragraph_count = doc.get_child_nodes(aspose.words.NodeType.PARAGRAPH, True).count
print(f"Document contains {paragraph_count} paragraphs after recovery.")
```

Αυτό το απόσπασμα σας βοηθά να αποφασίσετε αν το ανακτημένο αρχείο πληροί ένα ελάχιστο όριο περιεχομένου πριν το παραδώσετε σε επόμενα συστήματα.

## Συνηθισμένα Πίπτα και Πώς να τα Αποφύγετε

| Πίπτ | Γιατί Συμβαίνει | Διόρθωση |
|---------|----------------|-----|
| **Χρήση λανθασμένου `RecoveryMode`** | `THROW` διακόπτει στην πρώτη σφάλμα, αφήνοντάς σας χωρίς αρχείο. | Μείνετε στο `RECOVER` εκτός αν κάνετε εντοπισμό σφαλμάτων. |
| **Σκληρή κωδικοποίηση διαδρομών σε διαφορετικά OS** | Τα Windows χρησιμοποιούν backslashes· τα Linux/macOS χρησιμοποιούν forward slashes. | Χρησιμοποιήστε `os.path.join` ή raw strings (`r"..."`) για φορητότητα. |
| **Παράλειψη κλεισίματος του εγγράφου** | Μεγάλα αρχεία μπορούν να κρατούν ανοιχτά file handles. | Χρησιμοποιήστε έναν διαχειριστή περιβάλλοντος `with` (`with Document(...) as doc:`) στις νεότερες εκδόσεις του Aspose. |
| **Υπόθεση ότι οι εικόνες πάντα επιβιώνουν** | Ορισμένα ενσωματωμένα αντικείμενα μπορεί να είναι κατεστραμμένα πέρα από την επισκευή. | Μετά την ανάκτηση, σαρώστε `doc.get_child_nodes(NodeType.SHAPE, True)` για να καταγράψετε τα ελλείποντα στοιχεία. |
