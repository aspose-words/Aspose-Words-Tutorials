# Πώς να καλέσετε το LLM σε C# – Πλήρης Οδηγός με Τοπικό Μοντέλο

Έχετε αναρωτηθεί ποτέ **πώς να καλέσετε το LLM** από μια εφαρμογή .NET χωρίς να στέλνετε δεδομένα στο σύννεφο; Δεν είστε μόνοι. Πολλοί προγραμματιστές θέλουν να διατηρούν τα prompts και τα έγγραφά τους on‑premises, ειδικά όταν χειρίζονται ευαίσθητο κείμενο. Σε αυτό το tutorial θα περάσουμε από ένα πραγματικό σενάριο: χρήση ενός αυτο‑φιλοξενημένου LLM endpoint για την επανεγγραφή ενός εγγράφου Word, την αφαίρεση όλου του περιεχομένου, την επεξεργασία του αρχείου, και τελικά **πώς να αποθηκεύσετε docx** ξανά στο δίσκο.

Θα καλύψουμε επίσης **χρήση τοπικού LLM**, θα σας δείξουμε τον ακριβή κώδικα για **αφαίρεση όλου του περιεχομένου** από ένα Aspose.Words `Document`, και θα εξηγήσουμε τις λεπτομέρειες της προγραμματιστικής επεξεργασίας αρχείων Word. Στο τέλος θα έχετε μια λύση copy‑and‑paste που λειτουργεί με Aspose.Words 7+ και οποιοδήποτε τοπικό μοντέλο συμβατό με OpenAI.

## Προαπαιτούμενα – Τι Χρειάζεστε Πριν Ξεκινήσετε

- **.NET 6+** (ή .NET Framework 4.7.2 αν προτιμάτε κλασικό)
- **Aspose.Words for .NET** πακέτο NuGet (`Aspose.Words` και `Aspose.Words.AI`)
- Ένα **τοπικό LLM** που εκθέτει ένα OpenAI‑compatible `/v1` endpoint (π.χ., ένας διακομιστής GPT‑Neo στο `http://localhost:8000/v1`)
- Ένα δείγμα `input.docx` τοποθετημένο σε φάκελο που ελέγχετε
- Visual Studio, Rider, ή οποιονδήποτε επεξεργαστή προτιμάτε – θα χρησιμοποιήσω VS Code στις στιγμιότυπες

> **Pro tip:** Αν δεν έχετε ακόμη τοπικό μοντέλο, δείτε την δωρεάν εικόνα Docker για GPT‑Neo 2.7B – ξεκινά σε λιγότερο από ένα λεπτό και τηρεί το ίδιο API contract που χρησιμοποιούμε εδώ.

## Βήμα 1 – Διαμόρφωση του Τοπικού LLM Endpoint (Πώς να Καλέσετε το LLM)

Το πρώτο πράγμα που πρέπει να κάνετε όταν θέλετε να **πώς να καλέσετε το llm** από C# είναι να δημιουργήσετε ένα αντικείμενο client που δείχνει στην αυτο‑φιλοξενημένη υπηρεσία σας. Το Aspose.Words.AI περιλαμβάνει έναν βοηθό `LocalLargeLanguageModel` που αφαιρεί τις κλήσεις HTTP.

```csharp
using Aspose.Words;
using Aspose.Words.AI;

// Configure the self‑hosted LLM endpoint
var llm = new LocalLargeLanguageModel
{
    Endpoint = "http://localhost:8000/v1",   // your local server
    ModelName = "my-gpt-neo"                // name as registered in the server
};
```

> **Γιατί είναι σημαντικό:** Διαμορφώνοντας το endpoint μόνοι σας διατηρείτε πλήρη έλεγχο πάνω στα payload των αιτήσεων, την αυθεντικοποίηση και την καθυστέρηση. Είναι ο πυρήνας του **πώς να καλέσετε το llm** χωρίς να βασίζεστε σε εξωτερικές υπηρεσίες.

## Βήμα 2 – Φόρτωση του Πηγαίου Εγγράφου Word (Πώς να Επεξεργαστείτε το Word)

Στη συνέχεια, φορτώνουμε το αρχικό `.docx` σε ένα Aspose `Document`. Αυτό είναι το κλασικό βήμα “πώς να επεξεργαστείτε το word”: μόλις το αρχείο είναι στη μνήμη, μπορείτε να το ερωτήσετε, να το τροποποιήσετε ή να αντικαταστήσετε πλήρως το περιεχόμενό του.

```csharp
// Load the source document from disk
Document document = new Document("YOUR_DIRECTORY/input.docx");
```

Αν το αρχείο δεν υπάρχει, θα λάβετε ένα `FileNotFoundException`, οπότε βεβαιωθείτε ότι η διαδρομή είναι σωστή. Μπορείτε επίσης να φορτώσετε από ένα `Stream` αν διαχειρίζεστε ανεβάσματα.

## Βήμα 3 – Δημιουργία Αναθεωρημένου Κειμένου Χρησιμοποιώντας το Τοπικό LLM (Πώς να Καλέσετε το LLM)

Τώρα έρχεται η μαγεία: ζητάμε από το LLM να επανεγγράψει ολόκληρο το κείμενο σε επίσημο τόνο. Το prompt δημιουργείται συνενώνοντας μια σύντομη οδηγία με το ακατέργαστο κείμενο που εξάγεται μέσω `document.GetText()`.

```csharp
// Ask the model to rewrite the whole document in a formal tone
string prompt = "Rewrite the following in formal tone:\n" + document.GetText();

string revisedText = llm.GenerateText(prompt);
```

> **Edge case:** Αν το πηγαίο έγγραφο είναι τεράστιο (πάνω από 10 k tokens) μπορεί να υπερβείτε το όριο περιεχομένου του μοντέλου. Σε αυτή την περίπτωση χωρίστε το κείμενο σε παραγράφους και καλέστε `GenerateText` για κάθε τμήμα.

## Βήμα 4 – Αφαίρεση Όλου του Υπάρχοντος Περιεχομένου (Αφαίρεση Όλου του Περιεχομένου)

Πριν εισάγουμε το νέο κείμενο, πρέπει να καθαρίσουμε το έγγραφο. Το Aspose παρέχει τη μέθοδο `RemoveAllChildren()` που διαγράφει ενότητες, παραγράφους, πίνακες—όλα. Αυτή είναι η κανονική μέθοδος για **αφαίρεση όλου του περιεχομένου** από ένα αρχείο Word.

```csharp
// Clear the document completely
document.RemoveAllChildren();
```

> **Τι γίνεται αν θέλετε μόνο να διαγράψετε το σώμα αλλά να κρατήσετε τις κεφαλίδες;** Χρησιμοποιήστε `document.Sections.Clear()` και στη συνέχεια ξαναχτίστε τις ενότητες που χρειάζεστε.

## Βήμα 5 – Εισαγωγή του Αναθεωρημένου Κειμένου (Πώς να Επεξεργαστείτε το Word)

Με ένα καθαρό φύλλο, μπορούμε να γράψουμε ξανά το κείμενο που δημιούργησε το LLM. Η `DocumentBuilder` είναι η φιλική διεπαφή που σας επιτρέπει να προσθέτετε παραγράφους, πίνακες, εικόνες κ.λπ. Εδώ απλώς γράφουμε ολόκληρη τη συμβολοσειρά ως μία ενιαία παράγραφο.

```csharp
// Re‑populate the document with the revised text
DocumentBuilder builder = new DocumentBuilder(document);
builder.Writeln(revisedText);
```

Αν χρειάζεστε πιο πλούσια μορφοποίηση (έντονα, κεφαλίδες) μπορείτε να αναλύσετε την έξοδο του LLM για δείκτες markdown και να εφαρμόσετε τις ρυθμίσεις `builder.Font` ανάλογα.

## Βήμα 6 – Αποθήκευση του Ενημερωμένου Εγγράφου (Πώς να Αποθηκεύσετε Docx)

Τέλος, αποθηκεύουμε τις αλλαγές σε ένα νέο αρχείο. Αυτό δείχνει **πώς να αποθηκεύσετε docx** μετά από προγραμματιστικές επεξεργασίες.

```csharp
// Save the edited document
document.Save("YOUR_DIRECTORY/output.docx");
```

Η μέθοδος `Save` ανιχνεύει αυτόματα τη μορφή από την επέκταση του αρχείου, οπότε μπορείτε επίσης να εξάγετε σε PDF, HTML ή ODT με μια αλλαγή μιας γραμμής.

### Αναμενόμενο Αποτέλεσμα

Όταν ανοίξετε το `output.docx` θα πρέπει να δείτε όλο το αρχικό περιεχόμενο επανεγγραμμένο σε μια γυαλισμένη, επίσημη μορφή. Δεν υπάρχουν υπόλοιπα πίνακες, κεφαλίδες ή υποσέλιδα από το πηγαίο έγγραφο—μόνο το νέο κείμενο που ζήτησατε από το LLM.

## Συχνές Ερωτήσεις & Αντιμετώπιση Προβλημάτων

### 1. “Τι γίνεται αν το LLM μου επιστρέψει σφάλμα;”

Η μέθοδος `GenerateText` ρίχνει ένα `HttpRequestException` για απαντήσεις που δεν είναι 2xx. Τυλίξτε την κλήση σε `try/catch` και ελέγξτε το `ex.Message`. Συχνά το πρόβλημα είναι η έλλειψη κεφαλίδας API key ή η υπέρβαση του ορίου tokens του μοντέλου.

```csharp
try
{
    string revisedText = llm.GenerateText(prompt);
}
catch (HttpRequestException ex)
{
    Console.WriteLine($"LLM call failed: {ex.Message}");
    // fallback logic, e.g., return the original text
}
```

### 2. “Μπορώ να επεξεργαστώ συγκεκριμένα μέρη του εγγράφου αντί να διαγράψω τα πάντα;”

Απολύτως. Χρησιμοποιήστε `document.GetChildNodes(NodeType.Paragraph, true)` για να απαριθμήσετε τις παραγράφους, μετά αντικαταστήστε την ιδιότητα `Paragraph.Text` μόνο όπου χρειάζεστε αλλαγές. Αυτή η προσέγγιση σας επιτρέπει να **πώς να επεξεργαστείτε το word** σε λεπτομερή επίπεδο ενώ διατηρείτε τα στυλ.

### 3. “Υπάρχει τρόπος να διατηρήσω την αρχική μορφοποίηση;”

Αν θέλετε να διατηρήσετε τα στυλ, σκεφτείτε να επιστρέψετε την έξοδο του LLM ως απλό κείμενο και στη συνέχεια να εφαρμόσετε `builder.Font.StyleIdentifier` σε κάθε παράγραφο βάσει του προτύπου σας. Εναλλακτικά, χρησιμοποιήστε `DocumentBuilder.InsertHtml()` αν το LLM μπορεί να παράγει HTML.

### 4. “Πώς να διαχειριστώ μεγάλα έγγραφα;”

Διαιρέστε το έγγραφο σε ενότητες (`document.Sections`) και επεξεργαστείτε κάθε μία ξεχωριστά. Αυτό όχι μόνο αποφεύγει τα όρια tokens αλλά επίσης μειώνει την πίεση μνήμης.

## Συμβουλές Απόδοσης

- **Επαναχρησιμοποίηση του αντικειμένου `LocalLargeLanguageModel`** σε πολλαπλές κλήσεις· ο υποκείμενος `HttpClient` θα διατηρεί τη σύνδεση ζωντανή.
- **Κρυφή μνήμη (cache) του αναθεωρημένου κειμένου** αν αναμένετε να εκτελείτε το ίδιο prompt επανειλημμένα—οι κλήσεις LLM μπορεί να είναι δαπανηρές ακόμη και σε τοπικό υλικό.
- **Παράλληλη επεξεργασία** των ενοτήτων με `Parallel.ForEach` όταν έχετε CPU πολλαπλών πυρήνων και έναν thread‑safe πελάτη LLM.

Τώρα που γνωρίζετε **πώς να καλέσετε το llm**, **χρήση τοπικού llm**, **αφαίρεση όλου του περιεχομένου**, **πώς να επεξεργαστείτε το word**, και **πώς να αποθηκεύσετε docx**, ίσως θέλετε να εξερευνήσετε:

- **Επεξεργασία σε παρτίδες**: Επανάληψη πάνω σε φάκελο `.docx` αρχείων και εφαρμογή της ίδιας λογικής επανεγγραφής.

- **Ενσωμάτωση με ASP.NET Core**: Εκθέστε ένα HTTP endpoint που δέχεται ανέβασμα αρχείου, εκτελεί το LLM και επιστρέφει το επεξεργασμένο έγγραφο.
- **Προηγμένη μορφοποίηση**: Αναλύστε markdown από το LLM και αντιστοιχίστε το σε στυλ Word χρησιμοποιώντας `DocumentBuilder`.
