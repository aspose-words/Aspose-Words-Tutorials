# Ενεργοποίηση Προειδοποιήσεων Γραμματοσειρών στο Aspose.Words – Πλήρης Οδηγός C#

Έχετε αναρωτηθεί ποτέ γιατί ένα έγγραφο Word φαίνεται ξαφνικά διαφορετικό μετά τη φόρτωσή του προγραμματιστικά; **Enable Font Warnings** και θα γνωρίζετε αμέσως πότε το Aspose.Words αντικαθιστά μια ελλιπή γραμματοσειρά με εναλλακτική. Σε αυτό το σεμινάριο θα περάσουμε από ένα πρακτικό παράδειγμα που όχι μόνο εντοπίζει αυτές τις αντικαταστάσεις αλλά και εξηγεί *γιατί* συμβαίνουν.

Θα καλύψουμε όλα όσα χρειάζεστε για να ξεκινήσετε: το απαιτούμενο πακέτο NuGet, τη συγκεκριμένη διαμόρφωση `LoadOptions` και μια καθαρή έξοδο στην κονσόλα που σας λέει ποιες γραμματοσειρές αντικαταστάθηκαν. Στο τέλος θα έχετε ένα σταθερό, επαναχρησιμοποιήσιμο πρότυπο για **C# document processing** που λειτουργεί με οποιαδήποτε έκδοση του Aspose.Words.

## Τι Θα Μάθετε

- Πώς να δημιουργήσετε μια παρουσία `LoadOptions` που παρακολουθεί τις αλλαγές γραμματοσειρών.  
- Ο σκοπός του γεγονότος `SubstitutionWarning` και πώς να το συνδέσετε.  
- Ένα πλήρες, εκτελέσιμο δείγμα κώδικα που εκτυπώνει σαφείς προειδοποιήσεις στην κονσόλα.  
- Συμβουλές για τη διαχείριση ειδικών περιπτώσεων, όπως έγγραφα που περιέχουν μόνο τυπικές γραμματοσειρές.  

Δεν απαιτείται προηγούμενη εμπειρία με το Aspose.Words—απλώς μια βασική εξοικείωση με το C# και το .NET.

## Βήμα 1: Ρύθμιση LoadOptions και Ενεργοποίηση Προειδοποιήσεων Γραμματοσειρών

Το πρώτο πράγμα που χρειάζεστε είναι ένα αντικείμενο `LoadOptions`. Αυτό το δοχείο λέει στο Aspose.Words πώς να αντιμετωπίσει το αρχείο που πρόκειται να φορτώσετε. Αναθέτοντας μια νέα παρουσία `FontSettings` ανοίγετε την πόρτα σε γεγονότα σχετιζόμενα με τις γραμματοσειρές.

```csharp
using Aspose.Words;
using Aspose.Words.Fonts;

// Step 1: Prepare load options and enable font substitution warnings
LoadOptions loadOptions = new LoadOptions
{
    // Attach a new FontSettings object – this is where warnings are emitted.
    FontSettings = new FontSettings()
};
```

**Γιατί είναι σημαντικό:**  
Αν παραλείψετε την ανάθεση του `FontSettings`, το Aspose.Words θα συνεχίσει να αντικαθιστά τις ελλιπείς γραμματοσειρές, αλλά δεν θα λάβετε καμία ειδοποίηση. Ο μηχανισμός προειδοποίησης βρίσκεται μέσα στο `FontSettings`, οπότε η αρχικοποίησή του είναι *καίρια* για τον στόχο μας.

> **Συμβουλή:** Μπορείτε επίσης να κατευθύνετε το `FontSettings` σε έναν προσαρμοσμένο φάκελο γραμματοσειρών χρησιμοποιώντας το `SetFontsFolder`. Αυτό μειώνει τον αριθμό των προειδοποιήσεων που θα δείτε, επειδή το Aspose.Words μπορεί πραγματικά να βρει τις ελλιπείς γραμματοσειρές.

## Βήμα 2: Εγγραφή στο γεγονός SubstitutionWarning (αντικατάσταση γραμματοσειράς)

Τώρα που υπάρχει το αντικείμενο `FontSettings`, συνδέουμε το `SubstitutionWarning` event του. Αυτό το γεγονός ενεργοποιείται **κάθε φορά** που το Aspose.Words αντικαθιστά μια ζητούμενη γραμματοσειρά με κάτι άλλο.

```csharp
// Step 2: Subscribe to the SubstitutionWarning event to be notified when a font is replaced
loadOptions.FontSettings.SubstitutionWarning += (sender, e) =>
{
    // e.FontName – the name that the document asked for
    // e.SubstitutedFontName – the font that Aspose.Words actually used
    Console.WriteLine($"[Warning] Font \"{e.FontName}\" was substituted with \"{e.SubstitutedFontName}\".");
};
```

**Γιατί είναι σημαντικό:**  
Χωρίς αυτόν τον ακροατή δεν θα έχετε καμία ορατότητα στη διαδικασία αντικατάστασης. Η γραμμή στην κονσόλα σας παρέχει ένα γρήγορο ίχνος ελέγχου, το οποίο είναι ιδιαίτερα χρήσιμο κατά τις αυτοματοποιημένες κατασκευές ή όταν δημιουργείτε PDF για βιομηχανίες με αυστηρές απαιτήσεις συμμόρφωσης.

> **Κοινή ερώτηση:** *Τι γίνεται αν θέλω να καταστέλλω τις προειδοποιήσεις;*  
> Μπορείτε απλώς να αποσυνδέσετε τον χειριστή ή να ορίσετε `FontSettings.SubstitutionWarning += null;`. Ωστόσο, η διατήρηση των προειδοποιήσεων είναι συνήθως η πιο ασφαλής επιλογή, επειδή οι σιωπηλές αντικαταστάσεις μπορούν να προκαλέσουν προβλήματα διάταξης.

## Βήμα 3: Φόρτωση του Εγγράφου σας με τις Διαμορφωμένες Επιλογές (C# document processing)

Με το σύστημα προειδοποιήσεων έτοιμο, η φόρτωση του εγγράφου είναι απλή. Περνάτε την παρουσία `LoadOptions` στον κατασκευαστή `Document`, και το Aspose.Words θα κάνει τα υπόλοιπα.

```csharp
// Step 3: Load the document using the configured options
string filePath = @"C:\Docs\DocumentWithMissingFont.docx";

Document doc = new Document(filePath, loadOptions);

// Optional: Save to PDF to see the visual impact of the substitution
doc.Save(@"C:\Docs\Output.pdf");
```

**Γιατί είναι σημαντικό:**  
Το αντικείμενο `LoadOptions` είναι η γέφυρα μεταξύ του ακατέργαστου αρχείου και της υποδομής προειδοποιήσεων. Αν το παραλείψετε, το έγγραφο φορτώνεται σιωπηλά, και οποιεσδήποτε ελλιπείς γραμματοσειρές αντικαθίστανται χωρίς ίχνος.

> **Ειδική περίπτωση:** Κάποια έγγραφα ενσωματώνουν τα ακριβή αρχεία γραμματοσειρών που χρειάζονται. Σε αυτήν την περίπτωση δεν θα εμφανιστεί καμία προειδοποίηση επειδή το Aspose.Words βρίσκει την ενσωματωμένη γραμματοσειρά. Ο παραπάνω κώδικας λειτουργεί ακόμη· θα δείτε απλώς μια κενή έξοδο στην κονσόλα.

## Βήμα 4: Επαλήθευση της Εξόδου και Συνηθισμένα Παράπλονα

Εκτελέστε το πρόγραμμα από τη γραμμή εντολών ή τον αποσφαλματωτή του IDE σας. Αν το πηγαίο έγγραφο περιέχει μια γραμματοσειρά που δεν είναι εγκατεστημένη στο μηχάνημα (ή δεν είναι διαθέσιμη στον προσαρμοσμένο φάκελο γραμματοσειρών), θα δείτε γραμμές όπως:

```
[Warning] Font "Comic Sans MS" was substituted with "Arial".
[Warning] Font "MyCustomFont" was substituted with "Times New Roman".
```

Αν δεν εμφανιστεί τίποτα, είτε:

1. Βρέθηκαν όλες οι γραμματοσειρές, **ή**  
2. Ο χειριστής `SubstitutionWarning` δεν συνδέθηκε σωστά (επαληθεύστε ξανά το Βήμα 2).

### Γιατί Συμβαίνουν οι Αντικαταστάσεις Γραμματοσειρών;

- **Missing system font:** Το λειτουργικό σύστημα δεν διαθέτει τη ζητούμενη γραμματοσειρά.  
- **Unsupported font format:** Το Aspose.Words μπορεί να διαβάσει TrueType και OpenType, αλλά όχι κάθε ιδιόκτητη μορφή.  
- **License restrictions:** Ορισμένες εμπορικές γραμματοσειρές εμποδίζουν την ενσωμάτωση, αναγκάζοντας σε εναλλακτική.

Η κατανόηση του *γιατί* σας βοηθά να αποφασίσετε αν θα συμπεριλάβετε τις ελλιπείς γραμματοσειρές στην εφαρμογή σας ή θα προσαρμόσετε το στυλ του εγγράφου.

## Bonus: Έλεγχος της Εναλλακτικής Γραμματοσειράς

Αν θέλετε κάθε ελλιπής γραμματοσειρά να υποκαθίσταται από μια συγκεκριμένη οικογένεια (π.χ., “Calibri”), μπορείτε να ορίσετε έναν καθολικό κανόνα αντικατάστασης:

```csharp
loadOptions.FontSettings.SubstitutionSettings.FontSubstitutes.AddSubstitutes(
    "AnyMissingFont", // wildcard – applies to any font not found
    new[] { "Calibri" } // the preferred substitute
);
```

Τώρα η κονσόλα θα εξακολουθεί να σας προειδοποιεί, αλλά το οπτικό αποτέλεσμα θα είναι συνεπές για όλες τις ελλιπείς γραμματοσειρές.

## Ανακεφαλαίωση

- **Enable Font Warnings** δημιουργώντας ένα `LoadOptions` με μια νέα `FontSettings`.  
- Συνδέστε το γεγονός `SubstitutionWarning` για να λαμβάνετε ειδοποιήσεις σε πραγματικό χρόνο κάθε φορά που αντικαθίσταται μια γραμματοσειρά.  
- Φορτώστε το έγγραφό σας χρησιμοποιώντας τις διαμορφωμένες επιλογές και, προαιρετικά, αποθηκεύστε σε PDF για να δείτε το οπτικό αποτέλεσμα.  
- Διαγνώστε γιατί συνέβη η αντικατάσταση και, εάν χρειάζεται, επιβάλετε μια συγκεκριμένη εναλλακτική γραμματοσειρά.

Μόλις προσθέσατε ένα δίχτυ ασφαλείας στη ροή εργασίας του **Aspose.Words** που αποτρέπει τις σιωπηλές αλλαγές διάταξης. Στη συνέχεια, μπορείτε να εξερευνήσετε τις **ρυθμίσεις γραμματοσειρών** όπως το `DefaultFontName` ή να εμβαθύνετε στις επιλογές **απόδοσης εγγράφου** για να βελτιώσετε την έξοδο PDF.

### Τι να Δοκιμάσετε Στη Σύντομη Μελλοντική Περίοδο;

- **Εξερευνήστε άλλες δυνατότητες του FontSettings**: `SetFontsFolder`, `LoadFontSources` και `DefaultFontName`.  
- **Συνδυάστε τις προειδοποιήσεις με πλαίσια καταγραφής** (Serilog, NLog) για διαγνωστικά επιπέδου παραγωγής.  
- **Πειραματιστείτε με διαφορετικές μορφές εγγράφων** (`.doc`, `.rtf`, `.html`) για να δείτε πώς η κάθε μία διαχειρίζεται τις ελλιπείς γραμματοσειρές.
