# Αλλαγή των στηλοθετών περιεχομένων σε έγγραφο του Word

## Εισαγωγή

Αναρωτηθήκατε ποτέ πώς να αναβαθμίσετε τον Πίνακα Περιεχομένων (TOC) στα έγγραφά σας στο Word; Ίσως θέλετε τα στηλοθέτες να ευθυγραμμίζονται τέλεια για επαγγελματική πινελιά. Βρίσκεστε στο σωστό μέρος! Σήμερα, θα εμβαθύνουμε στο πώς μπορείτε να αλλάξετε τα στηλοθέτες του TOC χρησιμοποιώντας το Aspose.Words για .NET. Μείνετε μαζί μας και σας υπόσχομαι ότι θα φύγετε με όλη την τεχνογνωσία για να κάνετε τον Πίνακα Περιεχομένων σας να φαίνεται κομψός και κομψός.

## Προαπαιτούμενα

Πριν ξεκινήσουμε, ας βεβαιωθούμε ότι έχετε όλα όσα χρειάζεστε:

1. Aspose.Words για .NET: Μπορείτε [κατεβάστε το εδώ](https://releases.aspose.com/words/net/).
2. Περιβάλλον Ανάπτυξης: Visual Studio ή οποιοδήποτε IDE συμβατό με C#.
3. Ένα έγγραφο του Word: Συγκεκριμένα, ένα που περιέχει έναν πίνακα περιεχομένων.

Τα κατάλαβες όλα αυτά; Τέλεια! Ας ξεκινήσουμε.

## Εισαγωγή χώρων ονομάτων

Πρώτα απ 'όλα, θα χρειαστεί να εισαγάγετε τους απαραίτητους χώρους ονομάτων. Αυτό είναι σαν να συσκευάζετε τα εργαλεία σας πριν ξεκινήσετε ένα έργο.

```csharp
using Aspose.Words;
using Aspose.Words.Tables;
```

Ας αναλύσουμε αυτήν τη διαδικασία σε απλά και εύπεπτα βήματα. Θα εξετάσουμε τη φόρτωση του εγγράφου, την τροποποίηση των στηλοθετών στον πίνακα περιεχομένων και την αποθήκευση του ενημερωμένου εγγράφου.

## Βήμα 1: Φόρτωση του εγγράφου

Γιατί; Πρέπει να έχουμε πρόσβαση στο έγγραφο του Word που περιέχει τον Πίνακα Περιεχομένων που θέλουμε να τροποποιήσουμε.

Πώς; Ακολουθεί ένα απλό απόσπασμα κώδικα για να ξεκινήσετε:

```csharp
// Διαδρομή προς τον κατάλογο εγγράφων σας
string dataDir = "YOUR DOCUMENTS DIRECTORY";

// Φόρτωση του εγγράφου που περιέχει τον πίνακα περιεχομένων
Document doc = new Document(dataDir + "Table of contents.docx");
```

Φανταστείτε ότι το έγγραφό σας είναι σαν ένα κέικ και πρόκειται να προσθέσουμε λίγο γλάσο. Το πρώτο βήμα είναι να βγάλετε το κέικ από το κουτί.

## Βήμα 2: Προσδιορίστε τις παραγράφους του Πίνακα Περιεχομένων

Γιατί; Πρέπει να προσδιορίσουμε με ακρίβεια τις παραγράφους που αποτελούν τον Πίνακα Περιεχομένων. 

Πώς; Περιηγηθείτε στις παραγράφους και ελέγξτε το στυλ τους:

```csharp
foreach(Paragraph para in doc.GetChildNodes(NodeType.Paragraph, true))
{
    if (para.ParagraphFormat.Style.StyleIdentifier >= StyleIdentifier.Toc1 &&
        para.ParagraphFormat.Style.StyleIdentifier <= StyleIdentifier.Toc9)
    {
        // Βρέθηκε παράγραφος Περιεχομένου
    }
}
```

Σκεφτείτε το σαν να σαρώνετε ένα πλήθος για να βρείτε τους φίλους σας. Εδώ, αναζητούμε παραγράφους με στυλ καταχωρίσεις στον Πίνακα Περιεχομένων.

## Βήμα 3: Τροποποίηση των στηλοθετών

Γιατί; Εδώ συμβαίνει η μαγεία. Η αλλαγή των στηλοθετών δίνει στο TOC σας μια πιο καθαρή εμφάνιση.

Πώς; Αφαιρέστε το υπάρχον στηλοθέτη και προσθέστε ένα νέο σε μια τροποποιημένη θέση:

```csharp
foreach(Paragraph para in doc.GetChildNodes(NodeType.Paragraph, true))
{
    if (para.ParagraphFormat.Style.StyleIdentifier >= StyleIdentifier.Toc1 &&
        para.ParagraphFormat.Style.StyleIdentifier <= StyleIdentifier.Toc9)
    {
        TabStop tab = para.ParagraphFormat.TabStops[0];
        para.ParagraphFormat.TabStops.RemoveByPosition(tab.Position);
        para.ParagraphFormat.TabStops.Add(tab.Position - 50, tab.Alignment, tab.Leader);
    }
}
```

Είναι σαν να προσαρμόζεις τα έπιπλα στο σαλόνι σου μέχρι να σου φανούν άψογα. Προσπαθούμε να βελτιώσουμε τις ρυθμίσεις για να πετύχουμε την τελειότητα.

## Βήμα 4: Αποθήκευση του τροποποιημένου εγγράφου

Γιατί; Για να διασφαλίσετε ότι όλη η σκληρή δουλειά σας αποθηκεύεται και μπορεί να προβληθεί ή να κοινοποιηθεί.

Πώς; Αποθηκεύστε το έγγραφο με νέο όνομα για να διατηρήσετε το πρωτότυπο άθικτο:

```csharp
// Αποθήκευση του τροποποιημένου εγγράφου
doc.Save(dataDir + "WorkingWithTableOfContent.ChangeTocTabStops.docx");
```

Και ιδού! Ο πίνακας περιεχομένων σας έχει πλέον τους στηλοθέτες ακριβώς εκεί που τους θέλετε.
