## Εισαγωγή πακέτων

Πριν γράψουμε οποιονδήποτε κώδικα, πρέπει να εισαγάγουμε τους απαραίτητους χώρους ονομάτων για την εργασία με το Aspose.Words για .NET. Προσθέστε τις ακόλουθες εισαγωγές στην αρχή του αρχείου κώδικά σας:

```csharp
using Aspose.Words;
using Aspose.Words.Tables;
```

Αυτοί οι χώροι ονομάτων σας δίνουν πρόσβαση σε κλάσεις όπως `Document`, `Table`, `Cell`και άλλα που θα χρησιμοποιήσουμε σε αυτό το σεμινάριο.

## Βήμα 1: Φόρτωση του εγγράφου

Το πρώτο πράγμα που πρέπει να κάνουμε είναι να φορτώσουμε το έγγραφο του Word που περιέχει τον πίνακα. Για αυτό, θα χρησιμοποιήσουμε το `Document` κλάση από το Aspose.Words. Αυτή η κλάση σάς επιτρέπει να ανοίγετε και να χειρίζεστε αρχεία Word μέσω προγραμματισμού.

```csharp
// Η διαδρομή προς τον κατάλογο εγγράφων.
string dataDir = "YOUR DOCUMENTS DIRECTORY";

Document doc = new Document(dataDir + "Table spanning two pages.docx");
```

Σε αυτό το απόσπασμα κώδικα, καθορίζουμε την τοποθεσία του εγγράφου μας. Αντικατάσταση `"YOUR DOCUMENTS DIRECTORY"` με τον πραγματικό κατάλογο όπου είναι αποθηκευμένο το έγγραφό σας.

## Βήμα 2: Πρόσβαση στον πίνακα

Μόλις φορτωθεί το έγγραφο, το επόμενο βήμα είναι να αποκτήσουμε πρόσβαση στον πίνακα που θέλουμε να διατηρήσουμε μαζί. Σε αυτό το παράδειγμα, υποθέτουμε ότι ο πίνακας είναι ο πρώτος πίνακας στο έγγραφο.

```csharp
Table table = (Table) doc.GetChild(NodeType.Table, 0, true);
```

Αυτή η γραμμή κώδικα βρίσκει τον πρώτο πίνακα στο έγγραφο. `GetChild` Η μέθοδος ανακτά έναν συγκεκριμένο τύπο κόμβου, ο οποίος σε αυτήν την περίπτωση είναι `NodeType.Table`Το `0` υποδεικνύει ότι θέλουμε τον πρώτο πίνακα, και το `true` Η flag διασφαλίζει ότι αναζητούμε αναδρομικά σε όλους τους θυγατρικούς κόμβους.

## Βήμα 3: Επανάληψη μέσω κελιών πίνακα

Τώρα, πρέπει να κάνουμε επανάληψη σε κάθε κελί του πίνακα. Δεδομένου ότι ένας πίνακας περιέχει πολλές γραμμές και κάθε γραμμή περιέχει πολλά κελιά, θα επαναλάβουμε την επεξεργασία κάθε κελιού και θα διασφαλίσουμε ότι δεν θα διακόπτεται σε σελίδες.

```csharp
foreach (Cell cell in table.GetChildNodes(NodeType.Cell, true))
{
    cell.EnsureMinimum();
```

Εδώ, `GetChildNodes` ανακτά όλα τα κελιά στον πίνακα και κάνουμε επανάληψη σε καθένα από αυτά. Το `EnsureMinimum()` Η μέθοδος διασφαλίζει ότι κάθε κελί περιέχει τουλάχιστον μία παράγραφο, καθώς ένα κενό κελί θα μπορούσε να προκαλέσει προβλήματα αργότερα.

## Βήμα 4: Ορισμός ιδιότητας KeepWithNext

Για να αποτρέψουμε την κατάτμηση του πίνακα σε διάφορες σελίδες, πρέπει να ορίσουμε το `KeepWithNext` ιδιότητα για κάθε παράγραφο μέσα στον πίνακα. Αυτή η ιδιότητα διασφαλίζει ότι η παράγραφος παραμένει με την επόμενη παράγραφο, αποτρέποντας αποτελεσματικά τις αλλαγές σελίδας μεταξύ τους.

```csharp
    foreach (Paragraph para in cell.Paragraphs)
        if (!(cell.ParentRow.IsLastRow && para.IsEndOfCell))
            para.ParagraphFormat.KeepWithNext = true;
```

Αυτός ο βρόχος ελέγχει κάθε παράγραφο μέσα σε κάθε κελί. Η συνθήκη διασφαλίζει ότι δεν εφαρμόζουμε το `KeepWithNext` ιδιότητα στην τελευταία παράγραφο της τελευταίας γραμμής. Διαφορετικά, η ιδιότητα δεν θα έχει καμία επίδραση, καθώς δεν υπάρχει επόμενη παράγραφος.

## Βήμα 5: Αποθήκευση του εγγράφου

Τέλος, μετά την εφαρμογή του `KeepWithNext` ιδιότητα, πρέπει να αποθηκεύσουμε το τροποποιημένο έγγραφο.

```csharp
doc.Save(dataDir + "WorkingWithTables.KeepTableTogether.docx");
```

Αυτή η γραμμή αποθηκεύει το ενημερωμένο έγγραφο με νέο όνομα, διατηρώντας το αρχικό αρχείο. Τώρα μπορείτε να ανοίξετε το αρχείο που προκύπτει και να δείτε ότι ο πίνακας δεν είναι πλέον χωρισμένος σε δύο σελίδες!
