# Προτιμώμενες ρυθμίσεις πλάτους

## Εισαγωγή

Οι πίνακες αποτελούν έναν ισχυρό τρόπο οργάνωσης και παρουσίασης πληροφοριών στα έγγραφά σας στο Word. Όταν εργάζεστε με πίνακες στο Aspose.Words για .NET, έχετε αρκετές επιλογές για να ορίσετε το πλάτος των κελιών του πίνακα, ώστε να διασφαλίσετε ότι ταιριάζουν τέλεια στη διάταξη του εγγράφου σας. Αυτός ο οδηγός θα σας καθοδηγήσει στη διαδικασία δημιουργίας πινάκων με τις προτιμώμενες ρυθμίσεις πλάτους χρησιμοποιώντας το Aspose.Words για .NET, εστιάζοντας στις απόλυτες, σχετικές και αυτόματες επιλογές αλλαγής μεγέθους. 

## Προαπαιτούμενα

Πριν ξεκινήσετε το σεμινάριο, βεβαιωθείτε ότι έχετε τα εξής:

1. Aspose.Words για .NET: Βεβαιωθείτε ότι έχετε εγκαταστήσει το Aspose.Words για .NET στο περιβάλλον ανάπτυξής σας. Μπορείτε να το κατεβάσετε. [εδώ](https://releases.aspose.com/words/net/).

2. Περιβάλλον ανάπτυξης .NET: Να έχετε ρυθμίσει ένα περιβάλλον ανάπτυξης .NET, όπως το Visual Studio.

3. Βασικές γνώσεις C#: Η εξοικείωση με τον προγραμματισμό C# θα σας βοηθήσει να κατανοήσετε καλύτερα τα αποσπάσματα κώδικα και τα παραδείγματα.

4. Τεκμηρίωση Aspose.Words: Ανατρέξτε στο [Τεκμηρίωση Aspose.Words](https://reference.aspose.com/words/net/) για λεπτομερείς πληροφορίες API και περαιτέρω ανάγνωση.

## Εισαγωγή χώρων ονομάτων

Πριν ξεκινήσετε τον προγραμματισμό, πρέπει να εισαγάγετε τους απαραίτητους χώρους ονομάτων στο έργο σας σε C#:

```csharp
using Aspose.Words;
using Aspose.Words.Tables;
```

Αυτοί οι χώροι ονομάτων παρέχουν πρόσβαση στις βασικές λειτουργίες του Aspose.Words και του αντικειμένου Table, επιτρέποντάς σας να χειρίζεστε πίνακες εγγράφων.

Ας αναλύσουμε τη διαδικασία δημιουργίας ενός πίνακα με διαφορετικές προτιμώμενες ρυθμίσεις πλάτους σε σαφή και διαχειρίσιμα βήματα.

## Βήμα 1: Αρχικοποίηση του Εγγράφου και του DocumentBuilder

Επικεφαλίδα: Δημιουργία νέου εγγράφου και DocumentBuilder

Εξήγηση: Ξεκινήστε δημιουργώντας ένα νέο έγγραφο του Word και ένα `DocumentBuilder` παράδειγμα. Το `DocumentBuilder` Η κλάση παρέχει έναν απλό τρόπο για να προσθέσετε περιεχόμενο στο έγγραφό σας.

```csharp
// Ορίστε τη διαδρομή για την αποθήκευση του εγγράφου.
string dataDir = "YOUR DOCUMENT DIRECTORY";

// Δημιουργήστε ένα νέο έγγραφο.
Document doc = new Document();

// Δημιουργήστε ένα DocumentBuilder για αυτό το έγγραφο.
DocumentBuilder builder = new DocumentBuilder(doc);
```

Εδώ, καθορίζετε τον κατάλογο όπου θα αποθηκευτεί το έγγραφο και αρχικοποιείτε το `Document` και `DocumentBuilder` αντικείμενα.

## Βήμα 2: Εισαγωγή του πρώτου κελιού πίνακα με απόλυτο πλάτος

Εισαγάγετε το πρώτο κελί στον πίνακα με σταθερό πλάτος 40 σημείων. Αυτό θα διασφαλίσει ότι αυτό το κελί διατηρεί πάντα πλάτος 40 σημείων ανεξάρτητα από το μέγεθος του πίνακα.

```csharp
// Εισαγάγετε ένα κελί απόλυτου μεγέθους.
builder.InsertCell();
builder.CellFormat.PreferredWidth = PreferredWidth.FromPoints(40);
builder.CellFormat.Shading.BackgroundPatternColor = Color.LightYellow;
builder.Writeln("Cell at 40 points width");
```

Σε αυτό το βήμα, ξεκινάτε τη δημιουργία του πίνακα και εισάγετε ένα κελί με απόλυτο πλάτος. `PreferredWidth.FromPoints(40)` Η μέθοδος ορίζει το πλάτος του κελιού σε 40 σημεία, και `Shading.BackgroundPatternColor` εφαρμόζει ένα ανοιχτό κίτρινο χρώμα φόντου.

## Βήμα 3: Εισαγωγή κελιού σχετικού μεγέθους

Εισαγάγετε ένα άλλο κελί με πλάτος που είναι 20% του συνολικού πλάτους του πίνακα. Αυτό το σχετικό μέγεθος διασφαλίζει ότι το κελί προσαρμόζεται αναλογικά με το πλάτος του πίνακα.

```csharp
// Εισαγάγετε ένα κελί σχετικού (ποσοστού) μεγέθους.
builder.InsertCell();
builder.CellFormat.PreferredWidth = PreferredWidth.FromPercent(20);
builder.CellFormat.Shading.BackgroundPatternColor = Color.LightBlue;
builder.Writeln("Cell at 20% width");
```

Το πλάτος αυτού του κελιού θα είναι 20% του συνολικού πλάτους του πίνακα, καθιστώντας το προσαρμόσιμο σε διαφορετικά μεγέθη οθόνης ή διατάξεις εγγράφων.

### Βήμα 4: Εισαγωγή κελιού με αυτόματη προσαρμογή μεγέθους

Τέλος, εισαγάγετε ένα κελί που αλλάζει αυτόματα το μέγεθός του με βάση τον υπόλοιπο διαθέσιμο χώρο στον πίνακα.

```csharp
// Εισαγάγετε ένα κελί με αυτόματο μέγεθος.
builder.InsertCell();
builder.CellFormat.PreferredWidth = PreferredWidth.Auto;
builder.CellFormat.Shading.BackgroundPatternColor = Color.LightGreen;
builder.Writeln("Cell automatically sized. Ο size of this cell is calculated from the table preferred width.");
builder.Writeln("In this case the cell will fill up the rest of the available space.");
```

The `PreferredWidth.Auto` Η ρύθμιση επιτρέπει σε αυτό το κελί να επεκτείνεται ή να συστέλλεται με βάση τον χώρο που απομένει αφού ληφθούν υπόψη τα άλλα κελιά. Αυτό διασφαλίζει ότι η διάταξη του πίνακα φαίνεται ισορροπημένη και επαγγελματική.

## Βήμα 5: Οριστικοποίηση και αποθήκευση του εγγράφου

Μόλις εισαγάγετε όλα τα κελιά σας, συμπληρώστε τον πίνακα και αποθηκεύστε το έγγραφο στην καθορισμένη διαδρομή σας.

```csharp
// Αποθηκεύστε το έγγραφο.
doc.Save(dataDir + "WorkingWithTables.PreferredWidthSettings.docx");
```

Αυτό το βήμα οριστικοποιεί τον πίνακα και αποθηκεύει το έγγραφο με το όνομα αρχείου "WorkingWithTables.PreferredWidthSettings.docx" στον καθορισμένο κατάλογο.

## Σύναψη

Η δημιουργία πινάκων με προτιμώμενες ρυθμίσεις πλάτους στο Aspose.Words για .NET είναι απλή υπόθεση, αρκεί να κατανοήσετε τις διαφορετικές διαθέσιμες επιλογές μεγέθους. Είτε χρειάζεστε σταθερό, σχετικό είτε αυτόματο πλάτος κελιών, το Aspose.Words παρέχει την ευελιξία να χειρίζεστε αποτελεσματικά διάφορα σενάρια διάταξης πίνακα. Ακολουθώντας τα βήματα που περιγράφονται σε αυτόν τον οδηγό, μπορείτε να διασφαλίσετε ότι οι πίνακές σας είναι καλά δομημένοι και οπτικά ελκυστικοί στα έγγραφά σας στο Word.

## Συχνές ερωτήσεις

### Ποια είναι η διαφορά μεταξύ απόλυτου και σχετικού πλάτους κελιών;
Τα απόλυτα πλάτη των κελιών είναι σταθερά και δεν αλλάζουν, ενώ τα σχετικά πλάτη προσαρμόζονται με βάση το συνολικό πλάτος του πίνακα.

### Μπορώ να χρησιμοποιήσω αρνητικά ποσοστά για σχετικά πλάτη;
Όχι, τα αρνητικά ποσοστά δεν είναι έγκυρα για πλάτη κελιών. Επιτρέπονται μόνο θετικά ποσοστά.

### Πώς λειτουργεί η λειτουργία αυτόματης προσαρμογής μεγέθους;
Η αυτόματη αλλαγή μεγέθους προσαρμόζει το πλάτος του κελιού για να γεμίσει οποιοδήποτε κενό χώρο στον πίνακα μετά την αλλαγή μεγέθους άλλων κελιών.

### Μπορώ να εφαρμόσω διαφορετικά στυλ σε κελιά με διαφορετικές ρυθμίσεις πλάτους;
Ναι, μπορείτε να εφαρμόσετε διάφορα στυλ και μορφοποίηση σε κελιά ανεξάρτητα από τις ρυθμίσεις πλάτους τους.

### Τι συμβαίνει εάν το συνολικό πλάτος του πίνακα είναι μικρότερο από το άθροισμα όλων των πλάτους των κελιών;
Ο πίνακας θα προσαρμόσει αυτόματα το πλάτος των κελιών ώστε να χωράει στον διαθέσιμο χώρο, κάτι που μπορεί να προκαλέσει συρρίκνωση ορισμένων κελιών.
