# Μετατροπή DOCX σε PDF με Java – Πλήρης Οδηγός

Έχετε ποτέ χρειαστεί να **μετατρέψετε DOCX σε PDF** μέσα από μια εφαρμογή Java και αναρωτηθήκατε γιατί τα παραδείγματα πάντα παραλείπουν το δύσκολο κομμάτι των πλωτών σχημάτων; Δεν είστε μόνοι. Σε πολλά πραγματικά έργα, η απλή κλήση του `doc.save("out.pdf")` αφαιρεί εικόνες, πλαίσια κειμένου ή διαγράμματα από τη ροή, κάνοντας το PDF να φαίνεται χαλασμένο.  

Σε αυτόν τον οδηγό θα περάσουμε βήμα‑βήμα από μια **πλήρη, εκτελέσιμη λύση** που όχι μόνο **αποθηκεύει το Word ως PDF** αλλά επίσης διατηρεί τα πλωτά σχήματα ενσωματωμένα ώστε η διάταξη να παραμένει πιστή. Στο τέλος θα έχετε ένα αυτόνομο απόσπασμα κώδικα, θα καταλάβετε *γιατί* κάθε ρύθμιση είναι σημαντική και θα ξέρετε πώς να το προσαρμόσετε για ειδικές περιπτώσεις.

> **Τι θα χρειαστείτε**  
> • Java 17 (ή οποιοδήποτε πρόσφατο JDK)  
> • Βιβλιοθήκη Aspose.Words for Java (η δωρεάν δοκιμή λειτουργεί)  
> • Ένα αρχείο DOCX με τουλάχιστον ένα πλωτό σχήμα (π.χ., ένα πλαίσιο κειμένου)  

Αν τα έχετε, ας ξεκινήσουμε.

## Πώς να Μετατρέψετε DOCX σε PDF με Java (Κύρια Λέξη‑Κλειδί σε Δράση)

Η βασική ιδέα είναι απλή: φορτώστε το πηγαίο έγγραφο, πείτε στον δημιουργό PDF πώς να αντιμετωπίσει τα πλωτά σχήματα, και στη συνέχεια αποθηκεύστε. Οι επόμενες ενότητες αναλύουν κάθε βήμα, εξηγούν τη λογική και δείχνουν τον ακριβή κώδικα που μπορείτε να αντιγράψετε‑και‑επικολλήσετε.

![Screenshot of a Java IDE showing convert docx to pdf code](/images/convert-docx-to-pdf.png "convert docx to pdf example")

## Βήμα 1 – Ρυθμίστε το Έργο σας για Προγραμματισμένη Δημιουργία PDF

Πριν γράψετε οποιονδήποτε κώδικα, βεβαιωθείτε ότι το JAR του Aspose.Words βρίσκεται στο classpath σας. Αν χρησιμοποιείτε Maven, προσθέστε:

```xml
<dependency>
    <groupId>com.aspose</groupId>
    <artifactId>aspose-words</artifactId>
    <version>24.5</version> <!-- Check for the latest version -->
</dependency>
```

> **Συμβουλή:** Η βιβλιοθήκη είναι βαριά (~30 MB). Αν χρειάζεστε μόνο τη μετατροπή, σκεφτείτε το ελαφρύ SDK `aspose-words-cloud`, αλλά το JAR on‑premise σας δίνει πλήρη έλεγχο πάνω στις επιλογές αποθήκευσης.

## Βήμα 2 – Φορτώστε το Πηγαίο Έγγραφο

Χρειάζεστε ένα αντικείμενο `Document` που αντιπροσωπεύει το DOCX που θέλετε να μετατρέψετε. Ο κατασκευαστής δέχεται διαδρομή αρχείου, `InputStream`, ή ακόμη και πίνακα byte. Η χρήση διαδρομής κρατά το παράδειγμα σύντομο:

```java
import com.aspose.words.Document;
import com.aspose.words.PdfSaveOptions;

public class DocxToPdfConverter {

    public static void main(String[] args) throws Exception {
        // 👉 Load the source DOCX file
        Document doc = new Document("YOUR_DIRECTORY/input.docx");
```

**Γιατί είναι σημαντικό:** Η φόρτωση του αρχείου δημιουργεί μια αναπαράσταση στη μνήμη όλων των αντικειμένων του Word—παραγράφων, πινάκων, και των ενοχλητικών πλωτών σχημάτων. Αν το αρχείο δεν βρεθεί, το Aspose ρίχνει ένα σαφές `FileNotFoundException`, το οποίο μπορείτε να πιάσετε αργότερα αν χρειάζεστε ευγενική διαχείριση σφαλμάτων.

## Βήμα 3 – Διαμορφώστε τις Επιλογές Αποθήκευσης PDF για Ενσωματωμένα Σχήματα

Η προεπιλεγμένη μετατροπή θα *ισοπεδώσει* τα πλωτά σχήματα, συχνά τα μετακινώντας στην πάνω‑αριστερή γωνία της σελίδας. Για να διατηρήσετε τη οπτική ροή, ενεργοποιούμε τη σημαία `ExportFloatingShapesAsInlineTag`:

```java
        // 👉 Configure PDF options to keep floating shapes inline
        PdfSaveOptions pdfSaveOptions = new PdfSaveOptions();
        pdfSaveOptions.setExportFloatingShapesAsInlineTag(true);
        // Optional: set compliance level, image quality, etc.
        // pdfSaveOptions.setCompliance(PdfCompliance.PDF_A_1B);
```

**Εξήγηση:**  
- `setExportFloatingShapesAsInlineTag(true)` λέει στον δημιουργό PDF να τυλίξει κάθε πλωτό σχήμα σε μια αόρατη ετικέτα inline. Όταν το PDF αποδοθεί, το σχήμα συμπεριφέρεται σαν κανονικό κείμενο—διατηρώντας τη θέση του σε σχέση με τις γύρω παραγράφους.  
- Μπορείτε επίσης να ρυθμίσετε DPI, να ενσωματώσετε γραμματοσειρές, ή να επιβάλετε συμμόρφωση PDF/A· αυτά είναι εκτός του πλαισίου αυτού του οδηγού αλλά αξίζει να τα εξερευνήσετε για παραγωγικά PDFs.

## Βήμα 4 – Αποθηκεύστε το Έγγραφο ως PDF

Τώρα γράφουμε πραγματικά το αρχείο PDF. Η μέθοδος `save` δέχεται τη διαδρομή προορισμού και τις επιλογές που μόλις δημιουργήσαμε:

```java
        // 👉 Save the document as a PDF using the configured options
        doc.save("YOUR_DIRECTORY/output.pdf", pdfSaveOptions);
        System.out.println("Conversion complete! Check output.pdf");
    }
}
```

**Τι θα δείτε:** Το παραγόμενο `output.pdf` θα μοιάζει σχεδόν ακριβώς με το αρχικό αρχείο Word, με πλαίσια κειμένου, διαγράμματα και εικόνες να παραμένουν εκεί που τα τοποθετήσατε. Αν ανοίξετε το PDF στο Adobe Reader, θα παρατηρήσετε ότι κανένα στοιχείο δεν έχει χαθεί ή μετακινηθεί.
