# Μετατροπή DOCX σε PDF – Οδηγός Εξαγωγής Ενσωματωμένων Σχημάτων

Κάποτε χρειάστηκε να **μετατρέψετε DOCX σε PDF** αλλά ανησυχείτε ότι οι πλωτές εικόνες ή τα πλαίσια κειμένου θα εξαφανιστούν ή θα μετακινηθούν; Δεν είστε μόνοι. Σε πολλά έργα—σκεφτείτε αυτόματους δημιουργούς αναφορών ή δίκτυα επεξεργασίας παρτίδας—η διατήρηση της ακριβούς διάταξης ενός εγγράφου Word είναι αδιαπραγμάτευτη.  

Τα καλά νέα; Με λίγες γραμμές κώδικα μπορείτε να **αποθηκεύσετε το Word ως PDF** και να ελέγξετε αν αυτά τα πλωτά σχήματα θα μετατραπούν σε ετικέτες ενσωματωμένες ή θα παραμείνουν ως στοιχεία επιπέδου μπλοκ. Παρακάτω θα δείτε ακριβώς **πώς να εξάγετε σχήματα** όπως θέλετε, συν ένα σύνολο συμβουλών που σας σώζουν από κοινά προβλήματα.

## Τι Θα Μάθετε

* Φορτώστε ένα αρχείο `.docx` από το δίσκο.  
* Διαμορφώστε το `PdfSaveOptions` ώστε τα πλωτά σχήματα να εξάγονται ως ετικέτες ενσωματωμένες.  
* Γράψτε το παραγόμενο PDF σε φάκελο της επιλογής σας.  
* Κατανοήστε γιατί η σημαία `setExportFloatingShapesAsInlineTag` είναι σημαντική και πότε μπορεί να την αλλάξετε.  

Χωρίς εξωτερικές υπηρεσίες, χωρίς μαγικό UI «κλικ‑για‑λήψη»—απλώς καθαρός κώδικας Java που μπορείτε να ενσωματώσετε σε οποιοδήποτε έργο Maven ή Gradle.

## Βήμα 1 – Φόρτωση του Πηγαίου Εγγράφου  

Πρώτα δημιουργούμε μια παρουσία `Document` που δείχνει στο `.docx` που θέλετε να μετατρέψετε. Ο κατασκευαστής διαβάζει το αρχείο στη μνήμη, αναλύει το πακέτο OpenXML και προετοιμάζει το εσωτερικό μοντέλο αντικειμένων.

```java
import com.aspose.words.Document;
import com.aspose.words.SaveFormat;

// Adjust the path to your environment
String inputPath = "YOUR_DIRECTORY/input.docx";

Document doc = new Document(inputPath);
```

> **Συμβουλή:** Αν επεξεργάζεστε πολλά αρχεία σε βρόχο, επαναχρησιμοποιήστε ένα μόνο αντικείμενο `Document` μόνο αφού έχετε καλέσει `doc.close()` (ή αφήστε τον garbage collector να το διαχειριστεί). Αυτό αποτρέπει διαρροές χειριστών αρχείων στα Windows.

---

## Βήμα 2 – Διαμόρφωση των Επιλογών Αποθήκευσης PDF για Εξαγωγή Σχημάτων  

Η καρδιά του οδηγού βρίσκεται εδώ. Το `PdfSaveOptions` σας επιτρέπει να καθορίσετε πώς συμπεριφέρεται η μετατροπή. Η ρύθμιση `setExportFloatingShapesAsInlineTag(true)` αναγκάζει κάθε πλωτό σχήμα να αντιμετωπίζεται ως στοιχείο *ενσωματωμένο* στη δομή ετικετών του PDF. Αυτό σημαίνει ότι οι αναγνώστες οθόνης θα διαβάζουν το σχήμα στην ίδια σειρά με το περιβάλλον κείμενο, κάτι που συχνά απαιτείται για τη συμμόρφωση προσβασιμότητας.

```java
import com.aspose.words.PdfSaveOptions;

PdfSaveOptions pdfOptions = new PdfSaveOptions();
// true → inline tagging (shape behaves like a character)
// false → block‑level tagging (shape sits in its own block)
pdfOptions.setExportFloatingShapesAsInlineTag(true);
```

**Πότε θα το θέσετε σε `false`;**  
Αν το PDF προορίζεται μόνο για εκτύπωση και θέλετε τα σχήματα να διατηρήσουν την αρχική τους θέση χωρίς να επηρεάζουν τη λογική σειρά ανάγνωσης, ίσως προτιμήσετε ετικετοποίηση επιπέδου μπλοκ. Η προεπιλογή είναι `false`, οπότε ενεργοποιούμε ρητά τη συμπεριφορά ενσωμάτωσης για αυτόν τον οδηγό.

## Βήμα 3 – Αποθήκευση του Εγγράφου ως PDF  

Τώρα που οι επιλογές είναι έτοιμες, καλέστε `save` με το όνομα αρχείου προορισμού και το αντικείμενο επιλογών. Η βιβλιοθήκη αναλαμβάνει το βαρέως εργασίας: μηχανή διάταξης, ενσωμάτωση γραμματοσειρών και δημιουργία ετικετών.

```java
String outputPath = "YOUR_DIRECTORY/shapes.pdf";
doc.save(outputPath, pdfOptions);
```

Μετά το τέλος της κλήσης, θα βρείτε το `shapes.pdf` στον καθορισμένο φάκελο. Ανοίξτε το στο Adobe Acrobat ή σε οποιονδήποτε προβολέα PDF που εμφανίζει ετικέτες (συνήθως κάτω από **File → Properties → Tags**) και θα δείτε ότι το πλωτό σχήμα εμφανίζεται ως ετικέτα ενσωματωμένη.

## Πλήρες, Εκτελέσιμο Παράδειγμα  

Συνδυάζοντας όλα μαζί, εδώ είναι μια αυτόνομη κλάση Java που μπορείτε να μεταγλωττίσετε και να εκτελέσετε. Βεβαιωθείτε ότι το JAR του Aspose.Words βρίσκεται στο classpath σας.

```java
import com.aspose.words.*;

public class DocxToPdfWithShapes {
    public static void main(String[] args) {
        try {
            // 1️⃣ Load the source DOCX
            String inputPath = "YOUR_DIRECTORY/input.docx";
            Document doc = new Document(inputPath);

            // 2️⃣ Configure PDF options – export floating shapes as inline tags
            PdfSaveOptions pdfOptions = new PdfSaveOptions();
            pdfOptions.setExportFloatingShapesAsInlineTag(true); // true → inline tagging

            // 3️⃣ Save as PDF
            String outputPath = "YOUR_DIRECTORY/shapes.pdf";
            doc.save(outputPath, pdfOptions);

            System.out.println("✅ Conversion complete! PDF saved to: " + outputPath);
        } catch (Exception e) {
            System.err.println("❌ Something went wrong: " + e.getMessage());
            e.printStackTrace();
        }
    }
}
```

**Αναμενόμενο αποτέλεσμα:**  
- Το αρχείο PDF περιέχει το ίδιο κειμενικό περιεχόμενο με το αρχικό DOCX.  
- Όλες οι πλωτές εικόνες ή τα πλαίσια κειμένου είναι τώρα ετικετοποιημένα *ενσωματωμένα*, πράγμα που σημαίνει ότι εμφανίζονται στη σειρά ανάγνωσης αντί για ξεχωριστά μπλοκ.  
- Αν ανοίξετε το πάνελ **Tags** του PDF, θα δείτε ένα στοιχείο `<Figure>` ενσωματωμένο μέσα σε ένα `<Paragraph>`—ακριβώς αυτό που εγγυάται η `setExportFloatingShapesAsInlineTag(true)`.

## Συχνές Ερωτήσεις & Ακραίες Περιπτώσεις  

### 1️⃣ Λειτουργεί αυτό με αρχεία DOCX προστατευμένα με κωδικό;  
Ναι—απλώς παρέχετε τον κωδικό πριν τη φόρτωση:

```java
LoadOptions loadOptions = new LoadOptions();
loadOptions.setPassword("mySecret");
Document doc = new Document(inputPath, loadOptions);
```

### 2️⃣ Τι γίνεται με εικόνες SVG ή EMF μέσα στο αρχείο Word;  
Το Aspose.Words αυτόματα rasterizes τα διανυσματικά γραφικά κατά την αποθήκευση σε PDF. Αν χρειάζεστε να παραμείνουν διανυσματικά, ορίστε:

```java
pdfOptions.setRasterizeTransformedElements(false);
```

### 3️⃣ Πώς διατηρώ τους υπερσυνδέσμους κατά τη μετατροπή;  
Οι σύνδεσμοι διατηρούνται από προεπιλογή. Ωστόσο, αν απενεργοποιήσετε τις ετικέτες (`pdfOptions.setSaveFormat(SaveFormat.PDF)` χωρίς επιλογές), μπορεί να χάσετε τη λογική δομή. Διατηρήστε το αντικείμενο `PdfSaveOptions` για να διατηρήσετε και τις ετικέτες και τους συνδέσμους.

### 4️⃣ Μπορώ να επεξεργαστώ παρτίδα φακέλου με αρχεία DOCX;  
Απολύτως. Τυλίξτε τη λογική `DocxToPdfWithShapes` σε έναν βρόχο που επαναλαμβάνει πάνω από `Files.list(Paths.get("YOUR_DIRECTORY"))`. Θυμηθείτε να διαχειρίζεστε τις εξαιρέσεις ανά αρχείο ώστε ένα κακό έγγραφο να μην σταματήσει όλη τη διαδικασία.

## Συμβουλές από την Πρακτική  

* **Προσοχή στα ελλιπή γραμματοσειρά.** Αν το πηγαίο DOCX χρησιμοποιεί προσαρμοσμένη γραμματοσειρά που δεν είναι εγκατεστημένη στον διακομιστή, το PDF θα αντικαταστήσει με εναλλακτική, ενδεχομένως διασπώντας τη διάταξη. Χρησιμοποιήστε `pdfOptions.setFontEmbeddingMode(FontEmbeddingMode.EMBED_ALL)` για να εξαναγκάσετε την ενσωμάτωση.  
* **Δοκιμή προσβασιμότητας.** Μετά τη μετατροπή, τρέξτε το **Accessibility Checker** του Acrobat. Η ενσωματωμένη ετικετοποίηση συνήθως βελτιώνει το σκορ, αλλά ίσως χρειαστεί να προσθέσετε εναλλακτικό κείμενο στις εικόνες χειροκίνητα.  
* **Συμβουλή απόδοσης:** Για μεγάλα έγγραφα (100+ σελίδες), ενεργοποιήστε `pdfOptions.setMemoryOptimization(true)` για να μειώσετε τη χρήση μνήμης heap.
