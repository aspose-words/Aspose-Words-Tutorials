# Δημιουργία Επιλογών Φόρτωσης σε Java – Εντοπισμός Ελλειπόντων Γραμματοσειρών & Πώς να Φορτώσετε DOCX

Έχετε αναρωτηθεί ποτέ πώς να **δημιουργήσετε επιλογές φόρτωσης** που όχι μόνο διαβάζουν ένα DOCX αλλά και σας ενημερώνουν όταν λείπει μια γραμματοσειρά; Δεν είστε οι μόνοι. Οι ελλείπουσες γραμματοσειρές μπορούν να μετατρέψουν ένα τέλεια μορφοποιημένο έγγραφο σε ακατάληπτο χάος, και η έγκαιρη ανίχνευσή τους εξοικονομεί ώρες εντοπισμού σφαλμάτων. Σε αυτό το tutorial θα περάσουμε βήμα‑βήμα τις ακριβείς ενέργειες για **εντοπισμό ελλειπόντων γραμματοσειρών** ενώ θα σας δείξουμε **πώς να φορτώσετε αρχεία DOCX** με μια προσαρμοσμένη κλήση προειδοποίησης.

## Τι Θα Μάθετε

- Πώς να δημιουργήσετε ένα αντικείμενο `LoadOptions` και να διαμορφώσετε έναν διαχειριστή προειδοποιήσεων.  
- Γιατί η κλήση προειδοποίησης είναι απαραίτητη για την ανίχνευση προβλημάτων αντικατάστασης γραμματοσειρών.  
- Ο ακριβής κώδικας που απαιτείται για **ασφαλή φόρτωση ενός DOCX** αρχείου, μαζί με μερικές πρακτικές συμβουλές για πραγματικά έργα.  
- Διαχείριση ειδικών περιπτώσεων, όπως η αντιμετώπιση άλλων τύπων προειδοποιήσεων ή η φόρτωση PDF με την ίδια προσέγγιση.

Δεν απαιτείται εξωτερική τεκμηρίωση — όλα όσα χρειάζεστε είναι εδώ.

## Προαπαιτούμενα

- Java 17 ή νεότερη (το API λειτουργεί και σε παλαιότερες εκδόσεις, αλλά η 17 είναι η ιδανική).  
- Βιβλιοθήκη Aspose.Words for Java προστιθέμενη στο έργο σας (`aspose-words-x.x.jar`).  
- Βασική κατανόηση του χειρισμού εξαιρέσεων σε Java.  

Αν τα έχετε, ας ξεκινήσουμε.

## Βήμα 1: Δημιουργία Επιλογών Φόρτωσης (Πώς να Φορτώσετε DOCX)

Το πρώτο που πρέπει να κάνετε είναι **να δημιουργήσετε επιλογές φόρτωσης**. Αυτό το αντικείμενο λέει στην Aspose.Words πώς να συμπεριφέρεται όταν ανοίγει ένα αρχείο. Σκεφτείτε το ως ένα σύνολο οδηγιών που παραδίδετε στη βιβλιοθήκη πριν ακόμη δει το DOCX.

```java
// Step 1: Instantiate LoadOptions
LoadOptions loadOptions = new LoadOptions();
```

Γιατί να μην καλέσετε απλώς `new Document("file.docx")`; Διότι χωρίς `LoadOptions` χάνετε τη δυνατότητα αντίδρασης σε προειδοποιήσεις — όπως ελλείπουσες γραμματοσειρές — μέχρι το έγγραφο να έχει ήδη φορτωθεί, κάτι που μπορεί να είναι πολύ αργά για ορισμένες ροές εργασίας.

## Βήμα 2: Ρύθμιση Κλήσης Προειδοποίησης για Εντοπισμό Ελλειπόντων Γραμματοσειρών

Τώρα προσθέτουμε μια κλήση που θα εκτελείται όποτε η Aspose.Words αντιμετωπίζει μια κατάσταση που θέλει να σας προειδοποιήσει. Στην περίπτωσή μας, μας ενδιαφέρει το `WarningType.FONT_SUBSTITUTION`.

```java
// Step 2: Register a warning callback
loadOptions.setWarningCallback(new IWarningCallback() {
    @Override
    public void warning(WarningInfo info) {
        // React only to font substitution warnings
        if (info.getWarningType() == WarningType.FONT_SUBSTITUTION) {
            System.out.println("Missing font detected: " + info.getDescription());
        }
    }
});
```

Μερικά σημεία προς σημείωση:

- **Γιατί μια κλήση;** Εκτελείται *κατά τη διάρκεια* της διαδικασίας φόρτωσης, δίνοντάς σας την ευκαιρία να καταγράψετε ή ακόμη και να ακυρώσετε τη λειτουργία πριν το έγγραφο υλοποιηθεί πλήρως.  
- **Γιατί να ελέγξετε το `WarningType.FONT_SUBSTITUTION`;** Αυτή είναι η ακριβής τιμή enum που χρησιμοποιεί η Aspose.Words για σενάρια ελλείπουσας γραμματοσειράς. Άλλοι τύποι προειδοποιήσεων (π.χ., `TABLE_STRUCTURE`) μπορούν να φιλτραριστούν παρόμοια αν τα χρειάζεστε.  
- **Συμβουλή απόδοσης:** Η κλήση είναι ελαφριά· αποφύγετε βαριά I/O μέσα σε αυτήν. Αν χρειάζεται να γράψετε σε αρχείο, τοποθετήστε τα μηνύματα σε ουρά και εκκαθαρίστε τα μετά τη φόρτωση.

## Βήμα 3: Φόρτωση του Αρχείου DOCX με τις Διαμορφωμένες Επιλογές

Με τις επιλογές και την κλήση έτοιμες, μπορείτε τελικά να φορτώσετε το DOCX. Αυτό είναι το τμήμα που απαντά στο **πώς να φορτώσετε docx** ενώ σέβεστε τις προειδοποιήσεις που έχετε ορίσει.

```java
// Step 3: Load the document using the configured LoadOptions
try {
    Document document = new Document("YOUR_DIRECTORY/input.docx", loadOptions);
    System.out.println("Document loaded successfully.");
} catch (Exception e) {
    System.err.println("Failed to load document: " + e.getMessage());
}
```

**Τι συμβαίνει στο παρασκήνιο;** Καθώς το αρχείο ρέει, η Aspose.Words ελέγχει κάθε αναφορά γραμματοσειράς. Αν μια αναφερόμενη γραμματοσειρά δεν είναι εγκατεστημένη, ενεργοποιεί την κλήση προειδοποίησης που ορίσαμε νωρίτερα. Θα δείτε έξοδο όπως:

```
Missing font detected: Font 'Calibri' is not installed. Substituted with 'Arial'.
Document loaded successfully.
```

Αυτή η άμεση ανάδραση είναι ανεκτίμητη όταν επεξεργάζεστε δέσμες αρχείων σε έναν διακομιστή.

## Πλήρες Παράδειγμα Λειτουργίας

Συνδυάζοντας όλα, εδώ είναι ένα αυτόνομο πρόγραμμα που μπορείτε να αντιγράψετε‑επικολλήσετε στο IDE σας.

```java
import com.aspose.words.*;

public class DetectMissingFonts {
    public static void main(String[] args) {
        // 1️⃣ Create LoadOptions
        LoadOptions loadOptions = new LoadOptions();

        // 2️⃣ Register warning callback to detect missing fonts
        loadOptions.setWarningCallback(new IWarningCallback() {
            @Override
            public void warning(WarningInfo info) {
                if (info.getWarningType() == WarningType.FONT_SUBSTITUTION) {
                    System.out.println("Missing font: " + info.getDescription());
                }
            }
        });

        // 3️⃣ Load the DOCX using the configured options
        try {
            Document doc = new Document("YOUR_DIRECTORY/input.docx", loadOptions);
            System.out.println("DOCX loaded – you can now work with it.");
        } catch (Exception ex) {
            System.err.println("Error loading DOCX: " + ex.getMessage());
        }
    }
}
```

**Αναμενόμενη έξοδος**

```
Missing font: Font 'Times New Roman' is not installed. Substituted with 'Arial'.
DOCX loaded – you can now work with it.
```

Αν το αρχείο δεν περιέχει ελλείπουσες γραμματοσειρές, η κλήση παραμένει σιωπηλή και εμφανίζεται η γραμμή “DOCX loaded”.

## Pro Συμβουλές & Ειδικές Περιπτώσεις

| Κατάσταση | Τι να Κάνετε |
|-----------|--------------|
| **Πολλαπλές ελλείπουσες γραμματοσειρές** | Η κλήση ενεργοποιείται για κάθε μία, έτσι θα λάβετε μια γραμμή ανά γραμματοσειρά. Συγκεντρώστε τις σε μια `List<String>` αν χρειάζεστε σύνοψη αργότερα. |
| **Θέλετε επίσης να εντοπίσετε άλλες προειδοποιήσεις** | Προσθέστε κλάδους `else if` για `WarningType.TABLE_STRUCTURE`, `WarningType.UNKNOWN_FILE_FORMAT`, κ.λπ. |
| **Φόρτωση μεγάλων αρχείων DOCX** | Χρησιμοποιήστε `LoadOptions.setLoadFormat(LoadFormat.DOCX)` για να υποδείξετε τη μορφή και να επιταχύνετε τον εντοπισμό. |
| **Εκτέλεση σε web service** | Αποφύγετε το `System.out.println`; αντί αυτού, ενσωματώστε έναν logger (`SLF4J`, `Log4j`) μέσα στην κλήση. |
| **Οι γραμματοσειρές εγκαθίστανται κατά την εκτέλεση** | Αφού εντοπίσετε μια ελλείπουσα γραμματοσειρά, μπορείτε προγραμματιστικά να τη φορτώσετε μέσω `GraphicsEnvironment.registerFont(...)` και να ξαναφορτώσετε το έγγραφο. |

## Γιατί Αυτή η Προσέγγιση Υπερτερεί της Μεθόδου «Μόνο Try‑Catch»

Πολλοί προγραμματιστές απλώς τυλίγουν το `new Document(...)` σε ένα μπλοκ try‑catch, ελπίζοντας ότι μια εξαίρεση θα τους ενημερώσει για ελλείπουσες γραμματοσειρές. Δυστυχώς, η Aspose.Words αντιμετωπίζει την αντικατάσταση γραμματοσειρών ως *προειδοποίηση*, όχι σφάλμα, οπότε δεν ρίχνεται εξαίρεση. Με το **δημιουργία επιλογών φόρτωσης** και την προσθήκη κλήσης προειδοποίησης, αποκτάτε καθοριστική εικόνα για προβλήματα γραμματοσειρών χωρίς να θυσιάζετε την απόδοση.

## Επόμενα Βήματα

- **Εντοπισμός ελλειπόντων γραμματοσειρών σε PDF** – το ίδιο μοτίβο `LoadOptions` λειτουργεί για PDF, απλώς αλλάξτε τη διαδρομή αρχείου και τη μορφή φόρτωσης.  
- **Αυτοματοποίηση εγκατάστασης γραμματοσειρών** – συνδυάστε την κλήση με ένα script που αντλεί τις ελλείπουσες γραμματοσειρές από ένα κοινόχρηστο αποθετήριο.  
- **Εξερευνήστε άλλους τύπους προειδοποιήσεων** – η Aspose.Words μπορεί να σας ειδοποιήσει για παρωχημένες ετικέτες, πολύπλοκους πίνακες και άλλα.  

Νιώστε ελεύθεροι να πειραματιστείτε: αντικαταστήστε τον κατασκευαστή `Document` με ένα stream (`new Document(InputStream, loadOptions)`) αν εργάζεστε με δεδομένα στη μνήμη, ή συνδέστε πολλαπλές κλήσεις χρησιμοποιώντας ένα σύνθετο μοτίβο για μεγάλες γραμμές επεξεργασίας.
