# Πώς να Ανακτήσετε Αρχεία DOCX με το Aspose.Words – Ενεργοποίηση Λειτουργίας Ανάκτησης

Έχετε αναρωτηθεί ποτέ **πώς να ανακτήσετε docx** όταν το αρχείο αρνείται να ανοίξει; Ίσως λάβατε μια αναφορά που δημιούργησε ο πελάτης και προκαλεί κατάρρευση του προβολέα σας, ή ίσως ένα σφάλμα δικτύου άφησε ένα έγγραφο Word μισο‑γραμμένο. Σε αυτές τις στιγμές, το τελευταίο που θέλετε είναι να αρχίσετε να ξαναχτίζετε τις σελίδες χειροκίνητα — υπάρχει ένας καλύτερος τρόπος.

Τα καλά νέα είναι ότι το Aspose.Words for Java περιλαμβάνει ενσωματωμένη **λειτουργία ανάκτησης** που μπορεί να εντοπίσει κατεστραμμένα τμήματα και να δημιουργήσει ένα χρησιμοποιήσιμο έγγραφο. Σε αυτό το tutorial θα δούμε **πώς να ενεργοποιήσετε τη λειτουργία ανάκτησης**, να φορτώσουμε ένα πιθανώς κατεστραμμένο DOCX, **να ελέγξουμε αν το έγγραφο ανακτήθηκε**, και τέλος να αποθηκεύσουμε ένα καθαρό αντίγραφο. Στο τέλος θα έχετε ένα έτοιμο πρόγραμμα Java που μετατρέπει ένα σπασμένο .docx σε ένα φρέσκο .docx — χωρίς χειροκίνητη αντιγραφή‑επικόλληση.

> **Τι θα πάρετε:** ένα πλήρες, εκτελέσιμο παράδειγμα, εξηγήσεις για το γιατί κάθε γραμμή είναι σημαντική, συμβουλές για ακραίες περιπτώσεις, και έναν γρήγορο τρόπο να επαληθεύσετε ότι το αρχείο ανακτήθηκε πραγματικά.

## Προαπαιτούμενα

Πριν προχωρήσουμε, βεβαιωθείτε ότι έχετε:

- **Java Development Kit (JDK) 8+** – ο κώδικας χρησιμοποιεί τυπικά Java APIs.
- **Aspose.Words for Java** JAR (τελευταία έκδοση έως Μάρτιο 2026). Μπορείτε να το κατεβάσετε από το αποθετήριο Maven Central:

```xml
<dependency>
    <groupId>com.aspose</groupId>
    <artifactId>aspose-words</artifactId>
    <version>24.9</version>
</dependency>
```

- Ένα **input DOCX** που υποπτεύεστε ότι είναι κατεστραμμένο (για την επίδειξη θα το ονομάσουμε `input-corrupt.docx`).
- Έναν φάκελο στον οποίο έχετε δικαίωμα εγγραφής για το ανακτημένο αποτέλεσμα.

Αν χρησιμοποιείτε εργαλείο κατασκευής όπως Maven ή Gradle, απλώς προσθέστε την εξάρτηση και είστε έτοιμοι.

## Πώς να Ανακτήσετε DOCX – Ενεργοποίηση Λειτουργίας Ανάκτησης

Το πρώτο που πρέπει να κάνετε είναι να ενημερώσετε το Aspose.Words ότι αναμένετε προβλήματα. Αυτό γίνεται διαμορφώνοντας ένα αντικείμενο `LoadOptions` και ενεργοποιώντας τη **λειτουργία ανάκτησης**.

```java
// Step 1: Create LoadOptions and enable recovery mode
LoadOptions loadOptions = new LoadOptions();
loadOptions.setRecoveryMode(LoadOptions.RecoveryModeEnum.RECOVER);
```

> **Γιατί είναι σημαντικό:** Από προεπιλογή το Aspose.Words θα ρίξει εξαίρεση αν συναντήσει ένα κατεστραμμένο τμήμα. Ορίζοντας `RecoveryModeEnum.RECOVER` λέτε στη βιβλιοθήκη να συνεχίσει, προσπαθώντας να διασώσει όσο το δυνατόν περισσότερο. Σκεφτείτε το ως ένα δίχτυ ασφαλείας που πιάει τα σπασμένα κομμάτια αντί να αφήσει ολόκληρη τη διαδικασία φόρτωσης να καταρρεύσει.

### Συμβουλή Pro
Αν θέλετε μόνο να *καταγράψετε* προβλήματα χωρίς να τα διορθώσετε, χρησιμοποιήστε `RECOVER_WITH_WARNINGS`. Η επιλογή `RECOVER`, όμως, είναι αυτή που χρειάζεστε όταν θέλετε πραγματικά ένα χρησιμοποιήσιμο έγγραφο πίσω.

## Βήμα 2: Φόρτωση του Πιθανώς Κατεστραμμένου DOCX

Τώρα που η λειτουργία ανάκτησης είναι ενεργοποιημένη, φορτώστε το αρχείο. Ο κατασκευαστής δέχεται τη διαδρομή του αρχείου και το `LoadOptions` που μόλις προετοιμάσαμε.

```java
// Step 2: Load the DOCX using the recovery options
String inputPath = "YOUR_DIRECTORY/input-corrupt.docx";
Document document = new Document(inputPath, loadOptions);
```

> **Τι συμβαίνει στο παρασκήνιο;** Το Aspose αναλύει τη δομή OPC (Open Packaging Conventions), διορθώνει τις ελλιπείς σχέσεις και επαναδημιουργεί τυχόν σπασμένα τμήματα XML. Αν το αρχείο είναι μόνο ελαφρώς κατεστραμμένο, θα λάβετε ένα πλήρως λειτουργικό αντικείμενο `Document`.

### Ακραία περίπτωση
Αν το αρχείο είναι *σοβαρά* κατεστραμμένο (π.χ. λείπει το τμήμα `[Content_Types].xml`), το Aspose μπορεί ακόμη να επιστρέψει ένα έγγραφο, αλλά πολλά στοιχεία μπορεί να λείπουν. Σε τέτοιες περιπτώσεις ίσως θελήσετε να εξετάσετε το `OriginalFileInfo` για περισσότερες λεπτομέρειες.

## Βήμα 3: Επαλήθευση Εάν το Έγγραφο Ανακτήθηκε

Μετά τη φόρτωση, μπορείτε να ρωτήσετε τη βιβλιοθήκη αν πιστεύει ότι πραγματοποίησε κάποια εργασία ανάκτησης. Εδώ έρχεται σε χρήση η λέξη‑κλειδί **check document recovered**.

```java
// Step 3: Check if recovery actually occurred
boolean recovered = document.getOriginalFileInfo().isRecovered();
System.out.println("Recovered? " + recovered);
```

Τυπική έξοδος κονσόλας:

```
Recovered? true
```

Αν η έξοδος είναι `false`, το αρχείο ήταν είτε ήδη υγιές είτε η βιβλιοθήκη δεν μπόρεσε να το ανακτήσει. Μπορείτε επίσης να ερωτήσετε `getOriginalFileInfo().getRecoveryWarnings()` για μια λίστα προειδοποιήσεων που εξηγούν τι διορθώθηκε.

### Γιατί πρέπει να ελέγξετε
Ακόμη και όταν το έγγραφο φορτώνεται, μπορεί να συμβεί ήπια απώλεια δεδομένων (π.χ. λείπουν εικόνες). Ελέγχοντας τη σημαία ανάκτησης και τις προειδοποιήσεις, αποφασίζετε αν θα αποδεχτείτε το αποτέλεσμα ή θα ζητήσετε από τον χρήστη διαφορετική πηγή.

## Βήμα 4: Αποθήκευση του Ανακτημένου Εγγράφου

Υποθέτοντας ότι η ανάκτηση πέτυχε — ή ότι δε σας πειράζουν οι προειδοποιήσεις — γράψτε το καθαρό έγγραφο έξω. Αυτό δημιουργεί ένα ολοκαίνουργιο DOCX που μπορεί να ανοιχθεί στο Microsoft Word, Google Docs ή οποιονδήποτε άλλο προβολέα.

```java
// Step 4: Persist the repaired document
String outputPath = "YOUR_DIRECTORY/recovered.docx";
document.save(outputPath);
System.out.println("Recovered document saved to: " + outputPath);
```

Τώρα έχετε το `recovered.docx` δίπλα στο αρχικό σπασμένο αρχείο. Ανοίξτε το στο Word· θα πρέπει να δείτε όλο το αρχικό κείμενο, τους πίνακες και τις περισσότερες εικόνες ανέπαφες.

## Πλήρες Παράδειγμα Εργασίας

Παρακάτω βρίσκεται η πλήρης κλάση Java που ενώνει όλα τα παραπάνω. Αντιγράψτε‑επικολλήστε την στο IDE σας, προσαρμόστε τις διαδρομές και τρέξτε.

```java
import com.aspose.words.*;

public class RecoveryModeDemo {
    public static void main(String[] args) throws Exception {
        // ----------------------------------------------------
        // 1️⃣ Prepare LoadOptions to enable recovery mode
        // ----------------------------------------------------
        LoadOptions loadOptions = new LoadOptions();
        loadOptions.setRecoveryMode(LoadOptions.RecoveryModeEnum.RECOVER);

        // ----------------------------------------------------
        // 2️⃣ Load the potentially corrupted DOCX using the options
        // ----------------------------------------------------
        String inputPath = "YOUR_DIRECTORY/input-corrupt.docx";
        Document document = new Document(inputPath, loadOptions);

        // ----------------------------------------------------
        // 3️⃣ Verify whether the document was recovered
        // ----------------------------------------------------
        boolean recovered = document.getOriginalFileInfo().isRecovered();
        System.out.println("Recovered? " + recovered);

        // Optional: print any warnings (helps with debugging)
        for (String warning : document.getOriginalFileInfo().getRecoveryWarnings()) {
            System.out.println("Warning: " + warning);
        }

        // ----------------------------------------------------
        // 4️⃣ Save the recovered document
        // ----------------------------------------------------
        String outputPath = "YOUR_DIRECTORY/recovered.docx";
        document.save(outputPath);
        System.out.println("Recovered document saved to: " + outputPath);
    }
}
```

**Αναμενόμενο αποτέλεσμα:** Όταν εκτελέσετε το πρόγραμμα, η κονσόλα θα εκτυπώσει `Recovered? true` (ή `false` αν δεν χρειαζόταν ανάκτηση) ακολουθούμενο από επιβεβαίωση ότι το αρχείο αποθηκεύτηκε. Το άνοιγμα του `recovered.docx` θα πρέπει να δείχνει ένα τέλεια αναγνώσιμο έγγραφο.

| **Τι γίνεται αν το αρχείο είναι .doc (δυαδικό) αντί για .docx;** | Η λειτουργία ανάκτησης λειτουργεί και με τις δύο μορφές. Απλώς αλλάξτε την επέκταση του αρχείου· το Aspose θα ανιχνεύσει αυτόματα τη μορφή. |

| **Είναι η λειτουργία ανάκτησης ασφαλής για νήματα (thread‑safe);** | Ναι, κάθε αντικείμενο `Document` είναι ανεξάρτητο. Απλώς αποφύγετε το κοινόχρηστο `LoadOptions` μεταξύ νημάτων χωρίς κατάλληλο συγχρονισμό. |
| **Πώς να διαχειριστώ μεγάλα αρχεία (>100 MB);** | Σκεφτείτε να χρησιμοποιήσετε `LoadOptions.setLoadFormat(LoadFormat.DOCX)` για να εξαναγκάσετε τον αναλυτή και να αυξήσετε τη μνήμη heap της JVM (`-Xmx2g`). Η λειτουργία ανάκτησης προσθέτει μικρό κόστος, αλλά παραμένει γραμμική ως προς το μέγεθος του αρχείου. |

## Συμβουλές Pro για Πραγματικές Καταστάσεις

- **Επεξεργασία παρτίδας:** Τυλίξτε τον κώδικα επίδειξης σε βρόχο που σαρώει έναν φάκελο για αρχεία `*.docx`. Καταγράψτε την κατάσταση `isRecovered` κάθε αρχείου σε CSV για σκοπούς ελέγχου.
- **Καταγραφή προειδοποιήσεων:** Η λίστα `getRecoveryWarnings()` μπορεί να γραφτεί σε αρχείο καταγραφής. Αυτό σας βοηθά να εντοπίσετε μοτίβα — ίσως κάποιο πρόσθετο τρίτου μέρους να καταστρέφει έγγραφα.
- **Επαλήθευση μετά την ανάκτηση:** Μετά την αποθήκευση, ίσως θέλετε να φορτώσετε ξανά το νέο αρχείο και να εκτελέσετε έναν γρήγορο έλεγχο λογικής (π.χ. βεβαιωθείτε ότι ο αριθμός σελίδων ταιριάζει με τις προσδοκίες). Αυτός ο διπλός έλεγχος εντοπίζει σπάνιες ακραίες περιπτώσεις όπου η πρώτη φόρτωση πέτυχε αλλά το αποθηκευμένο αρχείο εξακολουθεί να έχει κρυφά προβλήματα.
- **Συνδυασμός με OCR:** Αν το κατεστραμμένο DOCX περιέχει σαρωμένες εικόνες, μπορείτε να περάσετε το ανακτημένο έγγραφο σε βιβλιοθήκη OCR (π.χ. Tesseract) για εξαγωγή αναζητήσιμου κειμένου.
