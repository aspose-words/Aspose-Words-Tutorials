# Πώς να Ανακτήσετε Αρχεία DOCX – Πλήρης Οδηγός Java

Έχετε αναρωτηθεί ποτέ **πώς να ανακτήσετε docx** αρχεία που αρνούνται να ανοίξουν; Ίσως ο συνεργάτης σας έστειλε ένα έγγραφο Word που φαίνεται εντάξει στον εξερευνητή αρχείων αλλά καταρρέει το Word αμέσως. Είναι ένα απογοητευτικό σενάριο, ειδικά όταν το περιεχόμενο είναι κρίσιμο χρονικά. Τα καλά νέα; Με το Aspose.Words for Java μπορείτε να **ορίσετε λειτουργία ανάκτησης**, **επισκευάσετε ένα κατεστραμμένο αρχείο Word**, και **αποθηκεύσετε το ανακτημένο έγγραφο** χωρίς κανένα κόπο.

Σε αυτό το tutorial θα περάσουμε από ένα πραγματικό παράδειγμα που καλύπτει τα πάντα, από τη φόρτωση ενός κατεστραμμένου `.docx` μέχρι την αποθήκευση ενός καθαρού αντιγράφου. Στο τέλος θα γνωρίζετε ακριβώς πώς να ανακτήσετε αρχεία docx, γιατί κάθε βήμα είναι σημαντικό, και ποιες παγίδες να αποφύγετε. Δεν χρειάζεται εξωτερική τεκμηρίωση—απλώς κώδικας έτοιμος για αντιγραφή‑επικόλληση και σαφείς εξηγήσεις.

## Τι Θα Χρειαστείτε

- **Aspose.Words for Java** (τελευταία έκδοση, 23.x τη στιγμή της συγγραφής).  
- Ένα IDE συμβατό με Java (IntelliJ IDEA, Eclipse ή VS Code).  
- Ένα κατεστραμμένο αρχείο `corrupted.docx` που θέλετε να διορθώσετε.  
- Βασική εξοικείωση με τη διαχείριση εξαιρέσεων Java (τίποτα εξωπραγματικό).

> **Συμβουλή:** Αν δεν έχετε ακόμη άδεια, η δωρεάν λειτουργία αξιολόγησης λειτουργεί τέλεια για εργασίες ανάκτησης· απλώς θυμηθείτε ότι προσθέτει υδατογράφημα στα αποθηκευμένα αρχεία.

## Βήμα 1 – Επιλέξτε τη Σωστή Λειτουργία Ανάκτησης (Primary Keyword: how to recover docx)

Πριν ακόμη αγγίξουμε το αρχείο, πρέπει να πούμε στο Aspose.Words **πώς να ανακτήσει docx** όταν αντιμετωπίζει κατεστραμμένα δεδομένα. Η βιβλιοθήκη προσφέρει δύο στρατηγικές μέσω του `RecoveryMode`:

| Λειτουργία | Συμπεριφορά |
|------------|--------------|
| `RECOVERY_MODE_PROMOTE_TO_OLE` | Προσπαθεί να διασώσει όσο το δυνατόν περισσότερο περιεχόμενο, προωθώντας τα μη αναγνώσιμα τμήματα σε αντικείμενα OLE. |
| `RECOVERY_MODE_IGNORE` | Παραλείπει σιωπηλά τα κατεστραμμένα τμήματα, κάτι που μπορεί να οδηγήσει σε απώλεια περιεχομένου αλλά παράγει ένα καθαρό αρχείο. |

Για τις περισσότερες περιπτώσεις, το `RECOVERY_MODE_PROMOTE_TO_OLE` προσφέρει την καλύτερη ισορροπία μεταξύ διατήρησης δεδομένων και ακεραιότητας αρχείου.

```java
// Step 1: Create LoadOptions and set the desired recovery mode
LoadOptions loadOptions = new LoadOptions();
loadOptions.setRecoveryMode(RecoveryMode.RECOVERY_MODE_PROMOTE_TO_OLE);
// Alternative: loadOptions.setRecoveryMode(RecoveryMode.RECOVERY_MODE_IGNORE);
```

*Γιατί είναι σημαντικό:* Αν παραλείψετε αυτή τη ρύθμιση, το Aspose.Words θα διακόψει τη φόρτωση του εγγράφου εντελώς, αφήνοντάς σας με μια γενική εξαίρεση “το αρχείο είναι κατεστραμμένο”. Ορίζοντας τη λειτουργία **ρητά** λέτε στη μηχανή να προσπαθήσει μια επιχείρηση διάσωσης.

## Βήμα 2 – Φορτώστε το Κατεστραμμένο Έγγραφο με τις Επιλογές Σας

Τώρα που ορίσαμε τη στρατηγική ανάκτησης, μπορούμε πραγματικά να φορτώσουμε το προβληματικό αρχείο. Ο κατασκευαστής `Document` δέχεται μια διαδρομή και τα `LoadOptions` που μόλις διαμορφώσαμε.

```java
// Step 2: Load the corrupted DOCX using the configured LoadOptions
String corruptedPath = "YOUR_DIRECTORY/corrupted.docx";
Document document = new Document(corruptedPath, loadOptions);
```

Αν το αρχείο είναι σοβαρά κατεστραμμένο, θα λάβετε ακόμη ένα αντικείμενο `Document`—απλώς δεν μπορεί να είναι κάθε στοιχείο άθικτο. Η βιβλιοθήκη καταγράφει προειδοποιήσεις εσωτερικά, τις οποίες μπορείτε να συλλέξετε μέσω του `Document.getWarnings()` εάν χρειάζεστε λεπτομερή αναφορά.

## Βήμα 3 – Επαληθεύστε Ποια Λειτουργία Ανάκτησης Εφαρμόστηκε (Προαιρετικό αλλά Χρήσιμο)

Μερικές φορές μπορεί να κάνετε αποσφαλμάτωση ή να εκτελείτε τον κώδικα σε μεγαλύτερο pipeline. Η γνώση της ακριβούς λειτουργίας που εφαρμόστηκε μπορεί να εξοικονομήσει ώρες ενασχόλησης.

```java
// Step 3: Output the active recovery mode (useful for debugging)
System.out.println("Loaded with recovery mode: " + loadOptions.getRecoveryMode());
```

Η κονσόλα θα εκτυπώσει κάτι όπως:

```
Loaded with recovery mode: RECOVERY_MODE_PROMOTE_TO_OLE
```

Αν δείτε `RECOVERY_MODE_IGNORE`, γνωρίζετε ότι η μηχανή επέλεξε να απορρίψει τα μη αναγνώσιμα τμήματα—ίσως χρειάζεται να μεταβείτε στη λειτουργία προώθησης για περισσότερα δεδομένα.

## Βήμα 4 – Αποθηκεύστε το Ανακτημένο Έγγραφο (Primary Keyword: how to recover docx)

Το τελευταίο κομμάτι του παζλ είναι η αποθήκευση του καθαρισμένου αρχείου. Μπορείτε να αποθηκεύσετε σε οποιαδήποτε μορφή υποστηρίζει το Aspose.Words (`.docx`, `.pdf`, `.html`, …). Εδώ θα το κρατήσουμε απλό και θα **αποθηκεύσουμε το ανακτημένο έγγραφο** σε ένα νέο `.docx`.

```java
// Step 4: Save the recovered document to a new file
String recoveredPath = "YOUR_DIRECTORY/recovered.docx";
document.save(recoveredPath);
System.out.println("Recovered file saved to: " + recoveredPath);
```

Όταν ανοίξετε το `recovered.docx` στο Microsoft Word, θα πρέπει να δείτε το αρχικό περιεχόμενο με μόνο μικρές ατέλειες διάταξης—χωρίς περισσότερα παράθυρα κατάρρευσης.

> **Αναμενόμενο αποτέλεσμα:** Η κονσόλα εκτυπώνει τη λειτουργία ανάκτησης και τη διαδρομή του αποθηκευμένου αρχείου. Το άνοιγμα του νέου αρχείου στο Word πρέπει να εμφανίζει το έγγραφο χωρίς σφάλματα.

## Πλήρες Παράδειγμα Εργασίας

Παρακάτω βρίσκεται η πλήρης, έτοιμη προς εκτέλεση κλάση Java που ενώνει τα τέσσερα βήματα. Αντικαταστήστε το `YOUR_DIRECTORY` με το πραγματικό φάκελο στο μηχάνημά σας.

```java
import com.aspose.words.*;

public class RecoveryDemo {
    public static void main(String[] args) throws Exception {

        // Step 1: Create LoadOptions and choose a recovery mode for damaged files
        LoadOptions loadOptions = new LoadOptions();
        loadOptions.setRecoveryMode(RecoveryMode.RECOVERY_MODE_PROMOTE_TO_OLE); // or RECOVERY_MODE_IGNORE

        // Step 2: Load the corrupted document using the configured options
        Document document = new Document("YOUR_DIRECTORY/corrupted.docx", loadOptions);

        // Step 3: (Optional) Verify which recovery mode was applied
        System.out.println("Loaded with recovery mode: " + loadOptions.getRecoveryMode());

        // Step 4: Save the recovered document to a new file
        document.save("YOUR_DIRECTORY/recovered.docx");
        System.out.println("Recovered file saved successfully.");
    }
}
```

Εκτελέστε αυτήν την κλάση από το IDE σας ή μέσω `java RecoveryDemo`. Αν όλα είναι ρυθμισμένα σωστά, η κονσόλα θα επιβεβαιώσει τη λειτουργία και τη θέση του νέου αρχείου.

## Ακραίες Περιπτώσεις & Συνηθισμένες Παγίδες

| Κατάσταση | Τι να κάνετε |
|-----------|--------------|
| **Το αρχείο είναι κρυπτογραφημένο** | Το Aspose.Words δεν μπορεί να ανακτήσει κρυπτογραφημένα έγγραφα χωρίς τον κωδικό. Αποκρυπτογραφήστε πρώτα, μετά εφαρμόστε τη λειτουργία ανάκτησης. |
| **Μόνο οι εικόνες παραμένουν** | Όταν η κατεστραμμένη κατάσταση είναι σοβαρή, μπορεί να καταλήξετε με ένα έγγραφο που περιέχει μόνο αντικείμενα OLE. Σκεφτείτε να εξάγετε τις εικόνες χειροκίνητα μέσω του `Document.getPageInfo()` και να ξαναχτίσετε το αρχείο. |
| **Μεγάλα αρχεία (>100 MB)** | Η φόρτωση μπορεί να καταναλώσει σημαντική μνήμη. Αυξήστε τη μνήμη heap της JVM (`-Xmx2g`) ή επεξεργαστείτε το αρχείο σε κομμάτια χρησιμοποιώντας το `DocumentBuilder`. |
| **Απροσδόκητες προειδοποιήσεις** | Καλέστε το `document.getWarnings()` μετά τη φόρτωση για να εξετάσετε τα αντικείμενα `WarningInfo`. Συχνά υποδεικνύουν ελλιπή τμήματα ή μη υποστηριζόμενες λειτουργίες. |
| **Αποθήκευση σε φάκελο μόνο για ανάγνωση** | Βεβαιωθείτε ότι ο φάκελος προορισμού έχει δικαίωμα εγγραφής· διαφορετικά το `document.save()` πετάει `IOException`. |

Η κατανόηση αυτών των λεπτομερειών κάνει τη διαδικασία **επισκευής κατεστραμμένου αρχείου Word** πιο ομαλή και αποτρέπει σιωπηλή απώλεια δεδομένων.

## Πότε να Χρησιμοποιήσετε `RECOVERY_MODE_IGNORE` vs. `RECOVERY_MODE_PROMOTE_TO_OLE`

- **`PROMOTE_TO_OLE`** – Η καλύτερη επιλογή όταν χρειάζεστε *μέγιστη διατήρηση δεδομένων*. Διατηρεί τα άγνωστα τμήματα ως ενσωματωμένα αντικείμενα, τα οποία το Word μπορεί ακόμη να εμφανίσει (αν και ως εικονίδια).  
- **`IGNORE`** – Πιο γρήγορη και παράγει πιο καθαρό αποτέλεσμα εάν μπορείτε να ανεχθείτε ελλιπή τμήματα. Χρήσιμη για επεξεργασία σε παρτίδες όπου η ταχύτητα υπερισχύει της πληρότητας.

Δοκιμάστε και τις δύο σε ένα αντίγραφο του κατεστραμμένου αρχείου σας για να δείτε ποιο προσφέρει το πιο χρήσιμο αποτέλεσμα.

## Bonus: Αυτοματοποίηση Ανάκτησης για Πολλαπλά Αρχεία

Αν έχετε έναν φάκελο γεμάτο σπασμένα έγγραφα, τυλίξτε τη λογική σε βρόχο:

```java
File folder = new File("YOUR_DIRECTORY");
for (File file : folder.listFiles((dir, name) -> name.endsWith(".docx"))) {
    try {
        Document doc = new Document(file.getAbsolutePath(), loadOptions);
        String outPath = file.getParent() + "/recovered_" + file.getName();
        doc.save(outPath);
        System.out.println("Recovered: " + outPath);
    } catch (Exception e) {
        System.err.println("Failed to recover " + file.getName() + ": " + e.getMessage());
    }
}
```

Αυτό το απόσπασμα **ορίζει τη λειτουργία ανάκτησης** μία φορά και την επαναχρησιμοποιεί, μειώνοντας δραστικά την χειροκίνητη προσπάθεια όταν χρειάζεται να **ανακτήσετε κατεστραμμένα docx** αρχεία μαζικά.
