# Ανάκτηση Κατεστραμμένου Εγγράφου Word – Πλήρης Java Tutorial

Έχετε ποτέ χρειαστεί να **ανακτήσετε ένα κατεστραμμένο έγγραφο Word** και αναρωτηθήκατε αν υπάρχει αξιόπιστος τρόπος να ανοίξετε ένα κατεστραμμένο .docx χωρίς να χάσετε τα πάντα; Δεν είστε μόνοι. Σε πολλά πραγματικά έργα, ένας χρήστης μπορεί να ανεβάσει ένα αρχείο που κατεστράφη κατά τη μεταφορά, ή μια αυτοματοποιημένη διαδικασία μπορεί να δημιουργήσει ένα μερικώς γραμμένο έγγραφο. Τα καλά νέα; Η Aspose.Words σας παρέχει μια ενσωματωμένη λειτουργία ανάκτησης που μπορεί να **ανοίξει κατεστραμμένο αρχείο docx** και να διατηρήσει όσο το δυνατόν περισσότερο περιεχόμενο.

Σε αυτόν τον οδηγό θα περάσουμε βήμα‑βήμα τις ακριβείς ενέργειες για να **φορτώσετε ένα έγγραφο Word με ασφάλεια** χρησιμοποιώντας τις δυνατότητες ανάκτησης της Aspose.Words. Στο τέλος θα έχετε ένα έτοιμο προς εκτέλεση πρόγραμμα Java που εκτυπώνει τον αριθμό σελίδων του ανακτηθέντος εγγράφου, μαζί με συμβουλές για τη διαχείριση ειδικών περιπτώσεων, καταγραφής και κοινών παγίδων.

## Τι Θα Χρειαστεί

- **Java 17** (ή οποιοδήποτε πρόσφατο JDK) – ο κώδικας μεταγλωττίζεται και με παλαιότερες εκδόσεις, αλλά το 17 είναι το ιδανικό σημείο για σύγχρονα εργαλεία.  
- **Aspose.Words for Java** βιβλιοθήκη – έκδοση 23.9 ή νεότερη (κατεβάστε από τον επίσημο ιστότοπο της Aspose ή λάβετε από το Maven Central).  
- Ένα **κατεστραμμένο .docx** αρχείο που θέλετε να δοκιμάσετε (ονομάστε το `input-corrupt.docx` και τοποθετήστε το σε φάκελο που μπορείτε να αναφέρετε).  
- Ένα IDE ή απλή ρύθμιση κατασκευής μέσω γραμμής εντολών (Maven/Gradle λειτουργούν καλά).  

Αυτό είναι όλο. Χωρίς επιπλέον εξαρτήσεις, χωρίς ασαφείς αρχεία ρυθμίσεων.

## Βήμα 1: Ρύθμιση LoadOptions με RecoveryMode

### Γιατί είναι σημαντικό

`LoadOptions` λέει στην Aspose.Words πώς να αντιμετωπίσει το εισερχόμενο αρχείο. Από προεπιλογή, η βιβλιοθήκη πετάει εξαίρεση τη στιγμή που εντοπίζει κατεστραμμένο αρχείο. Η αλλαγή του `RecoveryMode` σε `RECOVER` αλλάζει αυτή τη συμπεριφορά: ο parser προσπαθεί να διασώσει ό,τι μπορεί, παραλείποντας μη αναγνώσιμα τμήματα και γεμίζοντας τα κενά με εναλλακτικά στοιχεία. Σκεφτείτε το ως λειτουργία «best‑effort».

### Κώδικας

```java
import com.aspose.words.*;

public class RecoveryModeDemo {
    public static void main(String[] args) throws Exception {
        // 1️⃣ Create LoadOptions and enable recovery
        LoadOptions loadOptions = new LoadOptions();
        loadOptions.setRecoveryMode(RecoveryMode.RECOVER); // alternatives: SKIP, ALLOW_CORRUPTION
```

> **Συμβουλή:** Αν σας ενδιαφέρει μόνο η παράλειψη των κατεστραμμένων τμημάτων και δεν χρειάζεται να διατηρήσετε τη μορφοποίηση, το `RecoveryMode.SKIP` μπορεί να είναι λίγο πιο γρήγορο. Για πλήρη ανάκτηση, παραμείνετε στο `RECOVER`.

## Βήμα 2: Φόρτωση του Πιθανώς Κατεστραμμένου Εγγράφου

### Γιατί είναι σημαντικό

Ο κατασκευαστής `Document` δέχεται τη διαδρομή του αρχείου σας **και** τις `LoadOptions` που μόλις διαμορφώσαμε. Αυτό είναι το σημείο όπου η Aspose.Words προσπαθεί πραγματικά να διαβάσει το αρχείο. Αν το έγγραφο είναι σοβαρά κατεστραμμένο, θα λάβετε ακόμη ένα αντικείμενο `Document`—απλώς με λιγότερα στοιχεία.

### Κώδικας (συνέχεια)

```java
        // 2️⃣ Load the file using the recovery options
        Document document = new Document("YOUR_DIRECTORY/input-corrupt.docx", loadOptions);
```

Αντικαταστήστε το `YOUR_DIRECTORY` με την απόλυτη ή σχετική διαδρομή όπου αποθηκεύσατε το `input-corrupt.docx`. Η κλήση δεν θα πετάξει εξαίρεση για τις περισσότερες περιπτώσεις κατεστραμμένων αρχείων, που είναι ακριβώς αυτό που θέλουμε όταν **ανοίγουμε κατεστραμμένο αρχείο docx**.

## Βήμα 3: Επαλήθευση Φόρτωσης – Εκτύπωση Αριθμού Σελίδων

### Γιατί είναι σημαντικό

Μια γρήγορη έλεγχος λογικής σας βοηθά να επιβεβαιώσετε ότι το έγγραφο φορτώθηκε πράγματι. Ο αριθμός σελίδων είναι αξιόπιστος δείκτης επειδή η Aspose.Words τον υπολογίζει βάσει της αναλυθείσας διάταξης. Αν δείτε μη μηδενικό αριθμό, η ανάκτηση πέτυχε τουλάχιστον εν μέρει.

### Κώδικας (τελικό μέρος)

```java
        // 3️⃣ Verify loading succeeded by printing the page count
        System.out.println("Document loaded with " + document.getPageCount() + " pages.");
    }
}
```

Όταν εκτελέσετε το πρόγραμμα, θα πρέπει να δείτε κάτι όπως:

```
Document loaded with 12 pages.
```

Ακόμη και αν το αρχικό αρχείο είχε 15 σελίδες, μια ανακτημένη έκδοση με 12 σελίδες σας παρέχει ακόμα πολύτιμο περιεχόμενο για εργασία.

## Βήμα 4: Προαιρετικό – Αποθήκευση του Ανακτημένου Εγγράφου

Μερικές φορές θέλετε να διατηρήσετε την επισκευασμένη έκδοση για επεξεργασία αργότερα. Η Aspose.Words σας επιτρέπει να την αποθηκεύσετε σε οποιαδήποτε υποστηριζόμενη μορφή.

```java
        // Optional: Save the recovered file as a new, clean .docx
        document.save("YOUR_DIRECTORY/recovered-output.docx");
```

Τώρα έχετε ένα αποτέλεσμα **φορτώστε έγγραφο Word με ασφάλεια** που μπορείτε να δώσετε σε επόμενες υπηρεσίες (π.χ., μετατροπή σε PDF, εξαγωγή κειμένου ή OCR).

| **Μερικό κείμενο εμφανίζεται ως ακαταλαβίστικο** | Χρησιμοποιήστε `RecoveryMode.ALLOW_CORRUPTION` αν χρειάζεστε τα ακατέργαστα bytes, αλλά περιμένετε κακοσχηματισμένο markup. | Αυτή η λειτουργία είναι πιο επιεικής αλλά μπορεί να παράγει παράξενους χαρακτήρες. |

## Συχνές Ερωτήσεις

**Ε: Λειτουργεί αυτό με παλαιότερα αρχεία .doc;**  
Α: Απόλυτα. Η ίδια κλάση `LoadOptions` εφαρμόζεται σε όλες τις μορφές Word. Απλώς δείξτε τη διαδρομή σε ένα `.doc` και η Aspose.Words θα διαχειριστεί τη μετατροπή εσωτερικά.

**Ε: Μπορώ να ανακτήσω εικόνες ενσωματωμένες σε κατεστραμμένο αρχείο;**  
Α: Στις περισσότερες περιπτώσεις, ναι. Οι εικόνες που επιβιώνουν τη διαδικασία ανάλυσης θα διατηρηθούν. Αν η ροή εικόνας είναι κατεστραμμένη, η Aspose.Words θα την παραλείψει και θα δείτε ένα εναλλακτικό στοιχείο.

**Ε: Τι γίνεται αν χρειαστεί να ανοίξω το αρχείο σε υπηρεσία web χωρίς να το γράψω στο δίσκο;**  
Α: Περάστε ένα `InputStream` στον κατασκευαστή `Document` μαζί με τις `LoadOptions`. Η λογική ανάκτησης λειτουργεί με τον ίδιο τρόπο.

```java
try (InputStream is = new FileInputStream("input-corrupt.docx")) {
    Document doc = new Document(is, loadOptions);
    // continue as before
}
```

## Πλήρες Παράδειγμα Λειτουργίας

Παρακάτω είναι το πλήρες, αυτόνομο πρόγραμμα Java που μπορείτε να αντιγράψετε‑επικολλήσετε στο IDE σας. Περιλαμβάνει όλες τις εισαγωγές, τη διαμόρφωση ανάκτησης και την προαιρετική λογική αποθήκευσης.

```java
import com.aspose.words.*;

public class RecoveryModeDemo {
    public static void main(String[] args) throws Exception {
        // Step 1: Create and configure LoadOptions for recovery
        LoadOptions loadOptions = new LoadOptions();
        loadOptions.setRecoveryMode(RecoveryMode.RECOVER); // alternatives: SKIP, ALLOW_CORRUPTION

        // Step 2: Load the potentially corrupted document
        String inputPath = "YOUR_DIRECTORY/input-corrupt.docx";
        Document document = new Document(inputPath, loadOptions);

        // Step 3: Verify loading succeeded
        System.out.println("Document loaded with " + document.getPageCount() + " pages.");

        // Optional Step 4: Save the repaired document for future use
        String outputPath = "YOUR_DIRECTORY/recovered-output.docx";
        document.save(outputPath);
        System.out.println("Recovered document saved to " + outputPath);
    }
}
```

**Αναμενόμενη έξοδος** (υπόθεση ότι το αρχείο είχε ανακτήσιμο περιεχόμενο):

```
Document loaded with 12 pages.
Recovered document saved to YOUR_DIRECTORY/recovered-output.docx
```

Αν το αρχείο είναι πέρα από την επισκευή, θα δείτε `Document loaded with 0 pages.` και το αποθηκευμένο αρχείο θα είναι ουσιαστικά κενό.
