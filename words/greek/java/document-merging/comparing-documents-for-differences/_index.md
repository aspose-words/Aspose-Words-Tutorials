# Πώς να Συγκρίνετε Αρχεία DOCX – Σύγκριση Εγγράφων για Διαφορές

## Πώς να Συγκρίνετε Αρχεία DOCX – Εισαγωγή

Έχετε αναρωτηθεί **πώς να συγκρίνετε docx** αρχεία και να εντοπίσετε κάθε αλλαγή μεταξύ δύο εγγράφων Word; Ίσως επεξεργάζεστε ένα συμβόλαιο, ελέγχετε μια συνεργατική αναφορά ή χρειάζεστε έλεγχο νομικών εγγράφων. Οι χειροκίνητες συγκρίσεις είναι κουραστικές και επιρρεπείς σε λάθη, αλλά με το Aspose.Words for Java η αυτοματοποίηση της διαδικασίας γίνεται παιχνιδάκι. Αυτή η βιβλιοθήκη σας επιτρέπει να συγκρίνετε έγγραφα, να επισημαίνετε τις αλλαγές και να συγχωνεύετε τις τροποποιήσεις με λίγες μόνο γραμμές κώδικα.

## Εισαγωγή Πακέτων

Για να χρησιμοποιήσετε το Aspose.Words, πρέπει να εισάγετε τις απαραίτητες κλάσεις. Παρακάτω φαίνονται οι απαιτούμενες εισαγωγές:

```java
import com.aspose.words.*;
import java.util.Date;
```

Βεβαιωθείτε ότι αυτά τα πακέτα έχουν προστεθεί σωστά στις εξαρτήσεις του έργου σας.

Σε αυτήν την ενότητα, θα διασπάσουμε τη διαδικασία σε απλά βήματα.

## Βήμα 1: Ρύθμιση των Εγγράφων σας

Για να ξεκινήσετε, χρειάζεστε δύο έγγραφα: ένα που αντιπροσωπεύει το αρχικό και το άλλο την επεξεργασμένη έκδοση. Δείτε πώς τα δημιουργείτε:

```java
Document doc1 = new Document();
DocumentBuilder builder = new DocumentBuilder(doc1);
builder.writeln("This is the original document.");

Document doc2 = new Document();
builder = new DocumentBuilder(doc2);
builder.writeln("This is the edited document.");
```

Αυτό δημιουργεί δύο έγγραφα στη μνήμη με βασικό περιεχόμενο. Μπορείτε επίσης να φορτώσετε υπάρχοντα αρχεία Word χρησιμοποιώντας `new Document("path/to/document.docx")`.

## Βήμα 2: Έλεγχος Υπάρχουσας Αναθεώρησης

Οι αναθεωρήσεις σε έγγραφα Word αντιπροσωπεύουν παρακολουθούμενες αλλαγές. Πριν τη σύγκριση, βεβαιωθείτε ότι κανένα από τα έγγραφα δεν περιέχει προ‑υπάρχουσες αναθεωρήσεις:

```java
if (doc1.getRevisions().getCount() == 0 && doc2.getRevisions().getCount() == 0) {
    System.out.println("No revisions found. Proceeding with comparison...");
}
```

Αν υπάρχουν αναθεωρήσεις, ίσως θελήσετε να τις αποδεχτείτε ή να τις απορρίψετε πριν προχωρήσετε.

## Βήμα 3: Σύγκριση των Εγγράφων

Χρησιμοποιήστε τη μέθοδο `compare` για να βρείτε τις διαφορές. Αυτή η μέθοδος συγκρίνει το στόχο (`doc2`) με το πηγή (`doc1`):

```java
doc1.compare(doc2, "AuthorName", new Date());
```

Εδώ:
- **AuthorName** είναι το όνομα του ατόμου που κάνει τις αλλαγές.  
- **Date** είναι η χρονική σήμανση της σύγκρισης.

## Βήμα 4: Επεξεργασία Αναθεωρήσεων

Μετά τη σύγκριση, το Aspose.Words δημιουργεί αναθεωρήσεις στο έγγραφο πηγής (`doc1`). Ας αναλύσουμε αυτές τις αναθεωρήσεις:

```java
for (Revision r : doc1.getRevisions()) {
    System.out.println("Revision type: " + r.getRevisionType());
    System.out.println("Node type: " + r.getParentNode().getNodeType());
    System.out.println("Changed text: " + r.getParentNode().getText());
}
```

Αυτός ο βρόχος παρέχει λεπτομερείς πληροφορίες για κάθε αναθεώρηση, όπως ο τύπος της αλλαγής και το κείμενο που επηρεάζεται.

## Βήμα 5: Αποδοχή Όλων των Αναθεωρήσεων

Αν θέλετε το έγγραφο πηγής (`doc1`) να ταιριάζει με το έγγραφο στόχο (`doc2`), αποδεχτείτε όλες τις αναθεωρήσεις:

```java
doc1.getRevisions().acceptAll();
```

Αυτό ενημερώνει το `doc1` ώστε να αντικατοπτρίζει όλες τις αλλαγές που έγιναν στο `doc2`.

## Βήμα 6: Αποθήκευση του Ενημερωμένου Εγγράφου

Τέλος, αποθηκεύστε το ενημερωμένο έγγραφο στο δίσκο:

```java
doc1.save("Document.Compare.docx");
```

Για να επιβεβαιώσετε τις αλλαγές, φορτώστε ξανά το έγγραφο και ελέγξτε ότι δεν υπάρχουν υπόλοιπες αναθεωρήσεις:

```java
doc1 = new Document("Document.Compare.docx");
if (doc1.getRevisions().getCount() == 0) {
    System.out.println("Documents are now identical.");
}
```

## Βήμα 7: Επαλήθευση Ισότητας Εγγράφων

Για να βεβαιωθείτε ότι τα έγγραφα είναι πραγματικά ταυτόσημα, συγκρίνετε το απλό κείμενό τους:

```java
if (doc1.getText().trim().equals(doc2.getText().trim())) {
    System.out.println("Documents are equal.");
}
```

Αν τα κείμενα ταιριάζήγησητοποιήσετε τη διαδικασία, να δημιουργήσετε αρχεία ελέγχου και να ενσωματώσετε τη λογική σύγκρισης σε μεγαλύτερα‑υπάρχουσες αναθεωρήσεις:** Πάντα καθαρίζετε ή αποδέχεστε τις υπάρχουσες αναθεωρήσεις πριν καλέσετε `compare`, διαφορετικά το API μπορεί να τις θεωρήσει νέες αλλαγές.  
- **Μεγάλα έγγραφα:** Για πολύ μεγάλα αρχεία, σκεφτείτε να αυξήσετε το μέγεθος της μνήμης heap της JVM για να αποφύγετε `OutOfMemoryError`.  
- **Προσαρμοσμένο στυλ αναθεώρησης:**Options` για να αλλάξετε τον τρόπο εμφάνισης εισαγωγών/διαγραφών (π.χ., χρώμα επισήμανσης).

## Συχνές Ερωτήσεις (FAQ)

**Ε: Πώς συγκρίνω δύο υπάρχοντα αρχεία .docx στο δίσκο;**  
Α: Φορτώστε τα με `new Document("path/to/file.docx")` και στη συνέχεια καλέστε `compare` στο έγγραφο πηγής.

**Ε: Μπορώ να αγνοήσω αλλαγές μορφοποίησης κατά τη σύγκριση;**  
Α: Χρησιμοποιήστε `ComparisonOptions` για να ορίσετε `IgnoreFormatting` σε `true` αν σας ενδιαφέρουν μόνο οι κειμενικές διαφορές.

**Ε: Είναι δυνατόν να εξάγω τη λίστα αναθεωρήσεων σε αρχείο CSV;**  
Α: Διασχίστε το `doc.getRevisions()` και γράψτε τις ιδιότητες κάθε `Revision` σε CSV χρησιμοποιώντας τις τυπικές κλάσεις I/O της Java.

**Ε: Ποια έκδοση του Aspose.Words απαιτείται;**  
Α: Η τελευταία σταθερή έκδοση (π.χ., 24.11) υποστηρίζει πλήρως το API `compare`; παλαιότερες εκδόσεις μπορεί να έχουν περιορισμένες δυνατότητες.

**Ε: Το API διαχειρίζεται έγγραφα με κωδικό πρόσβασης;**  
Α: Ναι—περάστε τον κωδικό στην κατασκευή `Document` όταν φορτώνετε ένα προστατευμένο αρχείο.
