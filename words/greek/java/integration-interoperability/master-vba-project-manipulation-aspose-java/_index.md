## Οδηγός Εφαρμογής

Θα εξερευνήσουμε βασικές λειτουργίες του Aspose.Words για Java, εστιάζοντας στον χειρισμό έργων VBA.

### Δημιουργία νέου έργου VBA

#### Επισκόπηση
Η δημιουργία ενός νέου έργου VBA σάς επιτρέπει να ενσωματώσετε προσαρμοσμένες μακροεντολές στα έγγραφα του Word σας μέσω προγραμματισμού.

#### Βήματα:
**Βήμα 1: Αρχικοποίηση και ρύθμιση του έργου VBA**
```java
Document doc = new Document();
VbaProject project = new VbaProject();
project.setName("Aspose.Project");
doc.setVbaProject(project);
```
*Εξήγηση:* Δημιουργούμε ένα νέο `Document` παράδειγμα, αρχικοποιήστε ένα `VbaProject`, ορίστε το όνομά του και αντιστοιχίστε το στο έγγραφο.

**Βήμα 2: Δημιουργία και διαμόρφωση μιας ενότητας**
```java
VbaModule module = new VbaModule();
module.setName("Aspose.Module");
module.setType(VbaModuleType.PROCEDURAL_MODULE);
module.setSourceCode("New source code");
```
*Εξήγηση:* ΕΝΑ `VbaModule` δημιουργείται με συγκεκριμένο όνομα, τύπο (διαδικαστικό) και αρχικό πηγαίο κώδικα.

**Βήμα 3: Προσθήκη της ενότητας στο έργο**
```java
doc.getVbaProject().getModules().add(module);
```
*Εξήγηση:* Η ενότητα προστίθεται στη συλλογή ενοτήτων του έργου.

**Αποθήκευση του εγγράφου**
```java
doc.save("YOUR_OUTPUT_DIRECTORY/CreateNewVbaProject.docm");
```

### Κλωνοποίηση έργου VBA

#### Επισκόπηση
Η κλωνοποίηση ενός έργου VBA σάς επιτρέπει να αντιγράψετε υπάρχουσες μακροεντολές και λειτουργικές μονάδες σε ένα άλλο έγγραφο.

#### Βήματα:
**Βήμα 1: Κλωνοποίηση σε βάθος του αρχικού έργου VBA**
```java
Document originalDoc = new Document("YOUR_DOCUMENT_DIRECTORY/VBA_project.docm");
Document destDoc = new Document();
VbaProject copyVbaProject = originalDoc.getVbaProject().deepClone();
destDoc.setVbaProject(copyVbaProject);
```
*Εξήγηση:* Κλωνοποιούμε σε βάθος το έργο VBA από ένα υπάρχον έγγραφο και το τοποθετούμε σε ένα νέο έγγραφο προορισμού.

**Βήμα 2: Τροποποίηση ενοτήτων σε κλωνοποιημένο έργο**
```java
VbaModule oldVbaModule = destDoc.getVbaProject().getModules().get("Module1");
VbaModule copyVbaModule = originalDoc.getVbaProject().getModules().get("Module1").deepClone();
destDoc.getVbaProject().getModules().remove(oldVbaModule);
destDoc.getVbaProject().getModules().add(copyVbaModule);
```
*Εξήγηση:* Μια υπάρχουσα ενότητα αφαιρείται και αντικαθίσταται με την αντίστοιχη που έχει κλωνοποιηθεί σε βάθος.

**Αποθήκευση του εγγράφου**
```java
destDoc.save("YOUR_OUTPUT_DIRECTORY/CloneVbaProject.docm");
```

### Κατάργηση αναφοράς VBA

#### Επισκόπηση
Η διαχείριση αναφορών βοηθά στη διατήρηση της καθαριότητας του έργου σας, αφαιρώντας αχρησιμοποίητες ή κατεστραμμένες βιβλιοθήκες.

#### Βήματα:
**Βήμα 1: Επανάληψη και κατάργηση συγκεκριμένων αναφορών**
```java
Document doc = new Document("YOUR_DOCUMENT_DIRECTORY/VBA_project.docm");
VbaReferenceCollection references = doc.getVbaProject().getReferences();
String BROKEN_PATH = "X:\\broken.dll";

for (int i = references.getCount() - 1; i >= 0; i--) {
    VbaReference reference = references.get(i);
    String path = getLibIdPath(reference);
    if (BROKEN_PATH.equals(path))
        references.removeAt(i);
}
```
*Εξήγηση:* Επαναλάβετε τις αναφορές και αφαιρέστε όσες ταιριάζουν με μια καθορισμένη διακεκομμένη διαδρομή.

**Βήμα 2: Κατάργηση πρόσθετης αναφοράς ανά ευρετήριο**
```java
references.remove(references.get(1));
```

**Αποθήκευση του εγγράφου**
```java
doc.save("YOUR_OUTPUT_DIRECTORY/RemoveVbaReference.docm");
```

### Ελέγξτε εάν το έργο VBA είναι προστατευμένο

#### Επισκόπηση
Προσδιορίστε εάν ένα έργο VBA προστατεύεται με κωδικό πρόσβασης, διασφαλίζοντας τον έλεγχο πρόσβασης.

#### Εκτέλεση:
```java
Document doc = new Document("YOUR_DOCUMENT_DIRECTORY/Vba_protected.docm");
boolean isProtected = doc.getVbaProject().isProtected();
System.out.println("Is VBA Project Protected? " + isProtected);
```

*Εξήγηση:* Αυτό το τμήμα κώδικα ελέγχει εάν το έργο διαθέτει προστασία με κωδικό πρόσβασης και εκτυπώνει το αποτέλεσμα.
