# Aspose.Words for Java में पृष्ठ संख्याएँ समायोजित करें और सामग्री तालिका उत्पन्न करें

इस ट्यूटोरियल में आप **पृष्ठ संख्याएँ समायोजित** करने और Aspose.Words for Java के साथ **सामग्री तालिका** (TOC) **डालने** का तरीका जानेंगे। एक अच्छी‑संरचित TOC लंबी दस्तावेज़ों को नेविगेट करना आसान बनाती है, और पृष्ठ‑संख्या संरेखण को बारीकी से समायोजित करने से आपके पाठकों को पेशेवर अनुभव मिलता है। हम एक दस्तावेज़ बनाना, TOC शैलियों को अनुकूलित करना, और टैब स्टॉप को इस तरह बदलना सीखेंगे कि पृष्ठ संख्याएँ ठीक वहीँ दिखें जहाँ आप चाहते हैं।

## त्वरित उत्तर
- **“पृष्ठ संख्याएँ समायोजित” का क्या अर्थ है?** TOC में पृष्ठ संख्याओं को संरेखित करने वाले टैब स्टॉप को बदलना।  
- **क्या मैं स्वचालित रूप से सामग्री तालिका डाल सकता हूँ?** हाँ – `FieldToc` क्लास का उपयोग करें।  
- **कोड चलाने के लिए लाइसेंस चाहिए?** विकास के लिए मुफ्त ट्रायल काम करता है; उत्पादन के लिए लाइसेंस आवश्यक है।  
- **कौन सा Aspose संस्करण समर्थित है?** उदाहरण नवीनतम Aspose.Words for Java रिलीज़ के साथ काम करते हैं।  
- **क्या TOC शैलियों को अनुकूलित किया जा सकता है?** बिल्कुल – आप फ़ॉन्ट, बोल्डनेस आदि बदल सकते हैं।

## Aspose.Words में सामग्री तालिका क्या है?
TOC एक फ़ील्ड है जो दस्तावेज़ में हेडिंग शैलियों (जैसे Heading 1, Heading 2) को स्कैन करता है और पृष्ठ संख्याओं के साथ प्रविष्टियों की सूची बनाता है। Aspose.Words आपको इस फ़ील्ड को प्रोग्रामेटिकली डालने और उसकी उपस्थिति पर पूर्ण नियंत्रण देता है।

## TOC में पृष्ठ संख्याएँ क्यों समायोजित करें?
टैब स्टॉप को समायोजित करने से आप पृष्ठ संख्याओं के स्थान पर सटीक नियंत्रण प्राप्त करते हैं, जो निम्नलिखित के लिए आवश्यक है:

- साफ़, कॉलम‑संरेखित लेआउट बनाए रखना।  
- कॉर्पोरेट शैली मार्गदर्शिकाओं से मेल खाना।  
- मुद्रित और डिजिटल दस्तावेज़ों में पठनीयता सुधारना।

## पूर्वापेक्षाएँ
- आपके प्रोजेक्ट में Aspose.Words for Java जोड़ा गया हो (Maven/Gradle)।  
- Java सिंटैक्स की बुनियादी समझ।

## चरण‑दर‑चरण मार्गदर्शिका

### चरण 1: नया दस्तावेज़ बनाएं
सबसे पहले, एक खाली `Document` ऑब्जेक्ट बनाएं जो आपकी सामग्री और TOC को रखेगा।

```java
Document doc = new Document();
```

### चरण 2: TOC शैलियों को अनुकूलित करें
आप प्रत्येक TOC स्तर की दिखावट बदल सकते हैं। इस उदाहरण में हम प्रथम‑स्तर की प्रविष्टियों को बोल्ड बनाते हैं, जो एक सामान्य फ़ॉर्मेटिंग अनुरोध है।

```java
doc.getStyles().getByStyleIdentifier(StyleIdentifier.TOC_1).getFont().setBold(true);
```

### चरण 3: अपने दस्तावेज़ में सामग्री जोड़ें
हेडिंग (जैसे `Heading1`, `Heading2`) और सामान्य पैराग्राफ डालें। TOC फ़ील्ड बाद में इन हेडिंग को स्वचालित रूप से पहचान लेगा। *(कोड संक्षिप्त किया गया है – मुख्य ध्यान TOC जनरेशन पर है।)*

### चरण 4: TOC फ़ील्ड डालें
TOC को उस स्थान पर रखें जहाँ आप चाहते हैं—आमतौर पर दस्तावेज़ की शुरुआत में।

```java
// Insert a TOC field at the desired location in your document.
FieldToc fieldToc = new FieldToc();
doc.getFirstSection().getBody().getFirstParagraph().appendChild(fieldToc);
```

### चरण 5: दस्तावेज़ सहेजें
दस्तावेज़ को डिस्क पर सहेजें। आप DOCX, PDF, या HTML जैसे किसी भी समर्थित फ़ॉर्मेट को चुन सकते हैं।

```java
doc.save("your_output_path_here");
```

## TOC में टैब स्टॉप को अनुकूलित करना (पृष्ठ संख्याएँ समायोजित करना)
यदि डिफ़ॉल्ट टैब स्टॉप पृष्ठ संख्याओं को इच्छित स्थान पर नहीं लाता, तो आप सभी TOC पैराग्राफ़ों पर इटररेट करके उनके टैब पोज़ीशन को बदल सकते हैं।

```java
Document doc = new Document("Table of contents.docx");

for (Paragraph para : (Iterable<Paragraph>) doc.getChildNodes(NodeType.PARAGRAPH, true))
{
    if (para.getParagraphFormat().getStyle().getStyleIdentifier() >= StyleIdentifier.TOC_1 &&
        para.getParagraphFormat().getStyle().getStyleIdentifier() <= StyleIdentifier.TOC_9)
    {
        // Get the first tab used in this paragraph, which aligns the page numbers.
        TabStop tab = para.getParagraphFormat().getTabStops().get(0);
        
        // Remove the old tab.
        para.getParagraphFormat().getTabStops().removeByPosition(tab.getPosition());
        
        // Insert a new tab at a modified position (e.g., 50 units to the left).
        para.getParagraphFormat().getTabStops().add(tab.getPosition() - 50.0, tab.getAlignment(), tab.getLeader());
    }
}

doc.save("output.docx");
```

अब TOC प्रविष्टियाँ पृष्ठ संख्याएँ ठीक वहीँ दिखाती हैं जहाँ आप चाहते हैं, जिससे आपके दस्तावेज़ को एक परिष्कृत रूप मिलता है।

## सामान्य समस्याएँ और सुझाव
- **TOC में हेडिंग नहीं दिख रही:** सुनिश्चित करें कि आपकी हेडिंग बिल्ट‑इन शैलियों (`Heading1`, `Heading2`, आदि) का उपयोग करती हैं या कस्टम शैलियों को TOC स्तरों से मैप करें।  
- **टैब स्टॉप लागू नहीं हुआ:** जाँचें कि पैराग्राफ वास्तव में TOC शैली (`TOC_1`‑`TOC_9`) से संबंधित है।  
- **बड़े दस्तावेज़ों में प्रदर्शन:** TOC डालने के बाद `doc.updateFields()` कॉल करें ताकि सभी प्रविष्टियों को एक बार में रिफ्रेश किया जा सके।

## अक्सर पूछे जाने वाले प्रश्न

**प्रश्न: TOC प्रविष्टियों का फ़ॉर्मेट कैसे बदलें?**  
उत्तर: `doc.getStyles().getByStyleIdentifier(StyleIdentifier.TOC_X)` का उपयोग करें जहाँ *X* स्तर (1‑9) है, और उसके फ़ॉन्ट, रंग या पैराग्राफ सेटिंग्स को संशोधित करें।

**प्रश्न: अपने TOC में अधिक स्तर कैसे जोड़ें?**  
उत्तर: `FieldToc` स्विच `\o "1-3"` (उदाहरण) को समायोजित करके अतिरिक्त हेडिंग स्तर शामिल करें, फिर संबंधित `TOC_X` शैलियों को अपडेट करें।

**प्रश्न: क्या विशिष्ट TOC प्रविष्टियों के लिए टैब स्टॉप पोज़ीशन बदल सकते हैं?**  
उत्तर: हाँ – “टैब स्टॉप को अनुकूलित करना” सेक्शन में दिखाए अनुसार पैराग्राफ़ों पर इटररेट करके प्रत्येक टैब स्टॉप को अलग‑अलग बदलें।

**प्रश्न: क्या PDF आउटपुट में TOC जनरेट किया जा सकता है?**  
उत्तर: बिल्कुल। TOC जनरेट होने के बाद दस्तावेज़ को PDF (`doc.save("output.pdf")`) के रूप में सहेजें; फ़ील्ड स्वचालित रूप से रेंडर हो जाएगा।

**प्रश्न: क्या `updateFields()` को मैन्युअली कॉल करना आवश्यक है?**  
उत्तर: जब आप `FieldToc` डालते हैं, तो Aspose.Words इसे सेव पर अपडेट करता है, लेकिन डिबगिंग के लिए तुरंत परिणाम पाने हेतु `doc.updateFields()` कॉल करना उपयोगी है।

## निष्कर्ष
आपने **पृष्ठ संख्याएँ समायोजित करना**, **सामग्री तालिका डालना**, और **TOC शैलियों को अनुकूलित करना** Aspose.Words for Java का उपयोग करके सीख लिया है। ये तकनीकें आपको साफ़, नेविगेबल और पेशेवर रूप से फ़ॉर्मेटेड दस्तावेज़ बनाने में मदद करती हैं, जो किसी भी प्रकाशन मानक को पूरा करती हैं।
