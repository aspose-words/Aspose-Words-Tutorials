## دليل التنفيذ

يغطي هذا القسم الوظائف الأساسية للتحكم في ترقيم الصفحات وتحليل التخطيط.

### التحكم في ترقيم الصفحات في الأقسام المتصلة (H2)

#### ملخص

قم بضبط كيفية إعادة تشغيل أرقام الصفحات في الأقسام المستمرة لتتوافق مع متطلبات التنسيق المحددة.

#### خطوات التنفيذ

**1. تهيئة المستند:**

قم بتحميل مستندك باستخدام Aspose.Words:

```python
doc = aw.Document('your-document.docx')
```

**2. ضبط خيارات ترقيم الصفحات:**

التحكم في سلوك إعادة تشغيل ترقيم الصفحات:

```python
# تعيين لإعادة تشغيل الترقيم فقط من الصفحات الجديدة
doc.layout_options.continuous_section_page_numbering_restart = aw.layout.ContinuousSectionRestart.FROM_NEW_PAGE_ONLY

# تحديث التخطيط لتطبيق التغييرات
doc.update_page_layout()
```

**3. حفظ التغييرات:**

تصدير المستند بالإعدادات المحدثة:

```python
doc.save('output.pdf')
```

#### خيارات تكوين المفاتيح

- `ContinuousSectionRestart`:اختر كيفية إعادة تشغيل ترقيم الصفحات.
  - **من الصفحة الجديدة فقط**:إعادة التشغيل على الصفحات الجديدة فقط.

### تحليل تخطيط المستند (H2)

#### ملخص

تعلم كيفية التنقل وتحليل كيانات التخطيط داخل مستندك.

#### خطوات التنفيذ

**1. تهيئة جامع التخطيط:**

إنشاء مجمع تخطيط للمستند:

```python
layout_collector = aw.layout.LayoutCollector(doc)
```

**2. تحديث تخطيط الصفحة:**

تأكد من أن مقاييس التخطيط محدثة:

```python
doc.update_page_layout()
```

**3. اجتياز الكيانات باستخدام مُعَدِّد التخطيط:**

استخدم `LayoutEnumerator` للتنقل عبر الكيانات:

```python
layout_enumerator = aw.layout.LayoutEnumerator(doc)

# نقل وطباعة تفاصيل كل كيان
while True:
    if not layout_enumerator.move_next():
        break
    print(f"Entity type: {layout_enumerator.type}, Page index: {layout_enumerator.page_index}")
```

#### خيارات تكوين المفاتيح

- **نوع كيان التخطيط:** فهم أنواع مختلفة مثل PAGE وROW وSPAN.
- **الترتيب البصري مقابل الترتيب المنطقي:** اختر ترتيب العبور بناءً على احتياجات التخطيط.

1. **وثائق متعددة الفصول:** تأكد من ترقيم الصفحات بشكل متسق عبر الفصول مع صفحات بداية متنوعة.
