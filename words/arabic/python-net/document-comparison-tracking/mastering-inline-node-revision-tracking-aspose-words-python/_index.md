## دليل التنفيذ
الآن، دعنا نستعرض الخطوات اللازمة لتنفيذ تتبع مراجعة العقدة المضمنة.
### الميزة: تتبع مراجعة العقدة المضمنة
تتيح لك هذه الميزة تحديد وإدارة أنواع مختلفة من المراجعات في مستند Word. لنشرحها خطوة بخطوة.
#### الخطوة 1: تحميل المستند الخاص بك
قم بتحميل مستندك باستخدام Aspose.Words:
```python
doc = aw.Document('YOUR_DOCUMENT_DIRECTORY/Revision_runs.docx')
```
هنا، `Document` هي الفئة المستخدمة لتمثيل مستندات Word ومعالجتها في Aspose.Words. تأكد من أن المسار يشير إلى مستند تم تتبع تغييراته.
#### الخطوة 2: التحقق من عدد المراجعات
قبل الخوض في المراجعات الفردية، دعونا نتحقق من عدد المراجعات الموجودة:
```python
assert len(doc.revisions) == 6  # قم بالتعديل وفقًا لعدد المراجعات الفعلي لديك
```
يتحقق هذا التأكيد من عدد المراجعات. إذا لم يتطابق مع العدد الفعلي لمستندك، فعدّله وفقًا لذلك.
#### الخطوة 3: تحديد أنواع المراجعة
تشمل أنواع المراجعات المختلفة الإدراجات، وتغييرات التنسيق، والنقل، والحذف. لنتعرف على هذه الأنواع:
```python
# احصل على عقدة الأصل للمراجعة الأولى ككائن تشغيل
run = doc.revisions[0].parent_node.as_run()
first_paragraph = run.parent_paragraph
runs = first_paragraph.runs

assert len(runs) == 6  # تأكد من وجود ستة أشواط في الفقرة
```
الآن، دعونا نحدد أنواعًا محددة من المراجعات:
- **إدراج المراجعة:**
```python
# تحقق مما إذا كانت الجولة الثالثة عبارة عن مراجعة إدراج
assert runs[2].is_insert_revision
```
- **مراجعة التنسيق:**
```python
# التحقق من تغييرات التنسيق ضمن نفس التشغيل
assert runs[2].is_format_revision
```
- **نقل المراجعات:**
  - من المراجعة:
```python
assert runs[4].is_move_from_revision  # الوضع الأصلي قبل التحرك
```
  - للمراجعة:
```python
assert runs[1].is_move_to_revision   # منصب جديد بعد الانتقال
```
- **حذف المراجعة:**
```python
# تأكيد مراجعة الحذف في التشغيل الأخير
assert runs[5].is_delete_revision
```
