# วิธีเรียกใช้ LLM ใน C# – คู่มือฉบับสมบูรณ์กับโมเดลโลคัล

เคยสงสัย **วิธีเรียกใช้ LLM** จากแอปพลิเคชัน .NET โดยไม่ต้องส่งข้อมูลไปยังคลาวด์หรือไม่? คุณไม่ได้เป็นคนเดียวที่คิดเช่นนั้น นักพัฒนาจำนวนมากต้องการเก็บพรอมต์และเอกสารไว้ในเครื่องโดยเฉพาะเมื่อจัดการกับข้อความที่เป็นความลับ ในบทแนะนำนี้เราจะเดินผ่านสถานการณ์จริง: ใช้ endpoint LLM ที่โฮสต์เองเพื่อเขียนใหม่เอกสาร Word, ลบเนื้อหาทั้งหมด, แก้ไขไฟล์, และสุดท้าย **วิธีบันทึก docx** กลับไปยังดิสก์  

เราจะครอบคลุม **การใช้ local LLM**, แสดงโค้ดที่ **ลบเนื้อหาทั้งหมด** จาก `Document` ของ Aspose.Words, และอธิบายรายละเอียดของการแก้ไขไฟล์ Word ด้วยโปรแกรมเมชัน เมื่อจบคุณจะได้โซลูชันคัดลอกและวางที่ทำงานกับ Aspose.Words 7+ และโมเดลโลคัลที่เข้ากันได้กับ OpenAI

## ข้อกำหนดเบื้องต้น – สิ่งที่คุณต้องมีก่อนเริ่ม

- **.NET 6+** (หรือ .NET Framework 4.7.2 หากคุณต้องการแบบคลาสสิก)
- **Aspose.Words for .NET** NuGet package (`Aspose.Words` และ `Aspose.Words.AI`)
- **local LLM** ที่เปิดเผย endpoint แบบ OpenAI‑compatible `/v1` (เช่น เซิร์ฟเวอร์ GPT‑Neo ที่ `http://localhost:8000/v1`)
- ตัวอย่างไฟล์ `input.docx` ที่วางไว้ในโฟลเดอร์ที่คุณควบคุม
- Visual Studio, Rider หรือเครื่องมือแก้ไขใด ๆ ที่คุณชอบ – ฉันจะใช้ VS Code ในภาพหน้าจอ

> **เคล็ดลับมืออาชีพ:** หากคุณยังไม่มีโมเดลโลคัล ลองดู Docker image ฟรีสำหรับ GPT‑Neo 2.7B – จะสตาร์ทในไม่กี่วินาทีและปฏิบัติตามสัญญา API เดียวกับที่เราใช้ที่นี่

## ขั้นตอนที่ 1 – ตั้งค่า Endpoint ของ Local LLM (วิธีเรียกใช้ LLM)

สิ่งแรกที่คุณต้องทำเมื่อ **วิธีเรียกใช้ llm** จาก C# คือสร้างอ็อบเจ็กต์ client ที่ชี้ไปยังบริการที่โฮสต์เอง Aspose.Words.AI มาพร้อมกับตัวช่วย `LocalLargeLanguageModel` ที่ทำให้การเรียก HTTP ง่ายขึ้น

```csharp
using Aspose.Words;
using Aspose.Words.AI;

// Configure the self‑hosted LLM endpoint
var llm = new LocalLargeLanguageModel
{
    Endpoint = "http://localhost:8000/v1",   // your local server
    ModelName = "my-gpt-neo"                // name as registered in the server
};
```

> **ทำไมเรื่องนี้ถึงสำคัญ:** การกำหนดค่า endpoint ด้วยตนเองทำให้คุณควบคุม payload ของคำขอ, การยืนยันตัวตน, และความหน่วงเวลาได้เต็มที่ นี่คือหัวใจของ **วิธีเรียกใช้ llm** โดยไม่ต้องพึ่งบริการภายนอก

## ขั้นตอนที่ 2 – โหลดเอกสาร Word ต้นฉบับ (วิธีแก้ไข Word)

ต่อไปเราจะดึงไฟล์ `.docx` ดั้งเดิมเข้าไปใน `Document` ของ Aspose นี่คือขั้นตอน “**วิธีแก้ไข word**” แบบคลาสสิก: เมื่อไฟล์อยู่ในหน่วยความจำแล้วคุณสามารถสอบถาม, แก้ไข, หรือแทนที่เนื้อหาได้ทั้งหมด

```csharp
// Load the source document from disk
Document document = new Document("YOUR_DIRECTORY/input.docx");
```

หากไฟล์ไม่พบคุณจะได้รับ `FileNotFoundException` ดังนั้นตรวจสอบให้แน่ใจว่าเส้นทางถูกต้อง คุณยังสามารถโหลดจาก `Stream` หากกำลังจัดการกับการอัปโหลดได้อีกด้วย

## ขั้นตอนที่ 3 – สร้างข้อความที่แก้ไขโดยใช้ Local LLM (วิธีเรียกใช้ LLM)

ตอนนี้มาถึงจุดมหัศจรรย์: เราขอให้ LLM เขียนใหม่ข้อความทั้งหมดในโทนทางการ คำสั่ง (prompt) ถูกสร้างโดยต่อข้อความสั้นกับข้อความดิบที่ดึงมาจาก `document.GetText()`

```csharp
// Ask the model to rewrite the whole document in a formal tone
string prompt = "Rewrite the following in formal tone:\n" + document.GetText();

string revisedText = llm.GenerateText(prompt);
```

> **กรณีขอบ:** หากเอกสารต้นฉบับมีขนาดใหญ่ (เกิน 10 k token) คุณอาจเจอขีดจำกัดของโมเดล ในกรณีนั้นให้แบ่งข้อความเป็นย่อหน้าและเรียก `GenerateText` สำหรับแต่ละส่วน

## ขั้นตอนที่ 4 – ลบเนื้อหาที่มีอยู่ทั้งหมด (Remove All Content)

ก่อนที่เราจะใส่ข้อความใหม่ เราต้องล้างเอกสารให้ว่าง Aspose มีเมธอด `RemoveAllChildren()` ที่ลบส่วน, ย่อหน้า, ตาราง—ทุกอย่าง นี่คือวิธีมาตรฐานในการ **ลบเนื้อหาทั้งหมด** จากไฟล์ Word

```csharp
// Clear the document completely
document.RemoveAllChildren();
```

> **ถ้าต้องการลบเฉพาะเนื้อหาในส่วน body แต่เก็บหัวข้อไว้?** ใช้ `document.Sections.Clear()` แล้วสร้างส่วนที่ต้องการใหม่

## ขั้นตอนที่ 5 – แทรกข้อความที่แก้ไข (วิธีแก้ไข Word)

เมื่อเริ่มต้นใหม่ เราสามารถเขียนข้อความที่ LLM สร้างกลับเข้าไปได้ `DocumentBuilder` เป็นตัวห่อที่เป็นมิตร ช่วยให้คุณเพิ่มย่อหน้า, ตาราง, รูปภาพ ฯลฯ ที่นี่เราจะเขียนสตริงทั้งหมดเป็นย่อหน้าเดียว

```csharp
// Re‑populate the document with the revised text
DocumentBuilder builder = new DocumentBuilder(document);
builder.Writeln(revisedText);
```

หากต้องการรูปแบบที่ซับซ้อนกว่า (ตัวหนา, หัวข้อ) คุณสามารถแยกผลลัพธ์ของ LLM ตามเครื่องหมาย markdown แล้วตั้งค่า `builder.Font` ตามที่ต้องการ

## ขั้นตอนที่ 6 – บันทึกเอกสารที่อัปเดต (วิธีบันทึก Docx)

สุดท้าย เราบันทึกการเปลี่ยนแปลงลงไฟล์ใหม่ นี่คือการสาธิต **วิธีบันทึก docx** หลังจากแก้ไขด้วยโปรแกรม

```csharp
// Save the edited document
document.Save("YOUR_DIRECTORY/output.docx");
```

เมธอด `Save` จะตรวจจับรูปแบบจากส่วนขยายไฟล์โดยอัตโนมัติ ดังนั้นคุณก็สามารถส่งออกเป็น PDF, HTML, หรือ ODT ได้ด้วยการเปลี่ยนบรรทัดเดียว

### ผลลัพธ์ที่คาดหวัง

เมื่อคุณเปิด `output.docx` คุณควรเห็นเนื้อหาต้นฉบับทั้งหมดถูกเขียนใหม่ในสไตล์ที่เป็นทางการ ไม่เหลือตาราง, หัวข้อ, หรือส่วนท้ายจากไฟล์ต้นฉบับ—มีเพียงข้อความใหม่ที่คุณสั่งให้ LLM สร้างเท่านั้น

## คำถามทั่วไป & การแก้ไขปัญหา

### 1. “ถ้า LLM ของฉันคืนค่าข้อผิดพลาดล่ะ?”

เมธอด `GenerateText` จะโยน `HttpRequestException` สำหรับการตอบกลับที่ไม่ใช่ 2xx ให้ห่อการเรียกใน `try/catch` แล้วตรวจสอบ `ex.Message` ปัญหาที่พบบ่อยคือหัวข้อ API key ที่หายไปหรือเกินขีดจำกัด token ของโมเดล

```csharp
try
{
    string revisedText = llm.GenerateText(prompt);
}
catch (HttpRequestException ex)
{
    Console.WriteLine($"LLM call failed: {ex.Message}");
    // fallback logic, e.g., return the original text
}
```

### 2. “ฉันสามารถแก้ไขส่วนเฉพาะของเอกสารแทนการลบทั้งหมดได้หรือไม่?”

ทำได้แน่นอน ใช้ `document.GetChildNodes(NodeType.Paragraph, true)` เพื่อวนลูปย่อหน้า แล้วแทนที่คุณสมบัติ `Paragraph.Text` เฉพาะที่ต้องการเปลี่ยน วิธีนี้ทำให้คุณ **วิธีแก้ไข word** อย่างละเอียดโดยยังคงสไตล์เดิมไว้

### 3. “มีวิธีใดบ้างที่จะรักษาการจัดรูปแบบเดิมไว้?”

หากต้องการรักษา style ให้พิจารณาแปลงผลลัพธ์ของ LLM เป็นข้อความธรรมดาแล้วใช้ `builder.Font.StyleIdentifier` กับแต่ละย่อหน้าตามเทมเพลต หรือใช้ `DocumentBuilder.InsertHtml()` หาก LLM สามารถส่งออกเป็น HTML

### 4. “ฉันจะจัดการกับเอกสารขนาดใหญ่อย่างไร?”

แบ่งเอกสารเป็นส่วน (`document.Sections`) แล้วประมวลผลแต่ละส่วนแยกกัน วิธีนี้ช่วยหลีกเลี่ยงขีดจำกัด token และลดความกดดันของหน่วยความจำ

## เคล็ดลับด้านประสิทธิภาพ

- **ใช้ซ้ำอินสแตนซ์ `LocalLargeLanguageModel`** ในหลายการเรียก; `HttpClient` พื้นฐานจะคงการเชื่อมต่อไว้
- **แคชข้อความที่แก้ไข** หากคุณคาดว่าจะรันพรอมต์เดียวกันหลายครั้ง—การเรียก LLM อาจมีค่าใช้จ่ายสูงแม้บนฮาร์ดแวร์โลคัล
- **ขนานประมวลผล** การประมวลผลส่วนด้วย `Parallel.ForEach` เมื่อคุณมี CPU หลายคอร์และ client LLM ที่ปลอดภัยต่อเธรด

## ขั้นตอนต่อไป – การขยาย Workflow

เมื่อคุณรู้ **วิธีเรียกใช้ llm**, **ใช้ local llm**, **ลบเนื้อหาทั้งหมด**, **วิธีแก้ไข word**, และ **วิธีบันทึก docx** แล้ว คุณอาจอยากสำรวจต่อ:

- **การประมวลผลเป็นชุด**: วนลูปโฟลเดอร์ที่มีไฟล์ `.docx` แล้วใช้ตรรกะการเขียนใหม่เดียวกัน
- **พรอมต์แบบกำหนดเอง**: ปรับคำสั่งเพื่อสร้างสรุป, รายการหัวข้อ, หรือการแปล
- **การผสานกับ ASP.NET Core**: เปิดเผย endpoint HTTP ที่รับไฟล์อัปโหลด, รัน LLM, แล้วคืนเอกสารที่แก้ไข
- **การจัดรูปแบบขั้นสูง**: แยก markdown จาก LLM แล้วแมปไปยังสไตล์ Word ด้วย `DocumentBuilder`

## สรุป

ในคู่มือนี้เราได้ครอบคลุม **วิธีเรียกใช้ llm** จาก C# ผ่าน endpoint ที่โฮสต์เอง, แสดง **การใช้ local llm**, แสดงวิธี **ลบเนื้อหาทั้งหมด** จากไฟล์ Word, อธิบาย **วิธีแก้ไข word** ด้วยโปรแกรม, และสรุปด้วยตัวอย่าง **วิธีบันทึก docx** ตัวอย่างโค้ดที่พร้อมรันสามารถนำไปวางในโปรเจกต์ .NET ใดก็ได้ พร้อมคำอธิบาย “ทำไม” ของแต่ละขั้นตอน เพื่อให้คุณสามารถปรับแต่ง, ขยาย, หรือดีบักได้อย่างมั่นใจ

ลองใช้, ทดลองกับพรอมต์ต่าง ๆ, และให้ local LLM ทำงานหนักให้กับ pipeline การอัตโนมัติเอกสารของคุณ หากเจออุปสรรคใด ๆ ส่วน “การแก้ไขปัญหา” จะช่วยชี้ทางให้คุณได้อย่างรวดเร็ว ขอให้สนุกกับการเขียนโค้ดและเพลิดเพลินกับพลังของ LLM บนเครื่องของคุณ!
