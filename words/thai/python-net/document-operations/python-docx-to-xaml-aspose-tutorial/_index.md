# แปลง DOCX เป็น XAML ในรูปแบบคงที่ใน Python โดยใช้ Aspose.Words: คู่มือฉบับสมบูรณ์

## การแนะนำ

ในภูมิทัศน์ดิจิทัลของปัจจุบัน การแปลงเอกสาร Word (DOCX) เป็นรูปแบบที่รองรับเว็บ เช่น XAML ถือเป็นสิ่งสำคัญสำหรับการเข้าถึงและการรักษาความเที่ยงตรงของการออกแบบบนแพลตฟอร์มต่างๆ คู่มือนี้เน้นที่การแปลงไฟล์ DOCX ให้เป็น XAML ที่มีรูปแบบคงที่โดยใช้การจัดการทรัพยากรโดยใช้ไลบรารี Aspose.Words อันทรงพลังสำหรับ Python การเชี่ยวชาญกระบวนการแปลงนี้จะช่วยให้คุณจัดการทรัพยากรที่เชื่อมโยงกัน เช่น รูปภาพและแบบอักษรได้อย่างมีประสิทธิภาพ

**สิ่งที่คุณจะได้เรียนรู้:**
- แปลงเอกสาร Word (DOCX) เป็นรูปแบบ XAML ที่มีรูปแบบคงที่
- จัดการทรัพยากรที่เชื่อมโยงด้วยโฟลเดอร์และนามแฝงที่ปรับแต่งได้
- ใช้การโทรกลับเพื่อประหยัดทรัพยากรเพื่อติดตาม URI ในระหว่างการแปลง

## คู่มือการใช้งาน

ในส่วนนี้ เราจะแนะนำคุณเกี่ยวกับการแปลงไฟล์ DOCX เป็น XAML ที่มีรูปแบบคงที่พร้อมการจัดการทรัพยากร เราจะอธิบายคุณลักษณะแต่ละอย่างทีละขั้นตอน

### การแปลงเอกสารเป็น XAML แบบฟอร์มคงที่

#### ภาพรวม
ส่วนนี้จะเน้นไปที่การใช้ Aspose.Words `save` วิธีการแปลงเอกสารของคุณเป็นรูปแบบ XAML ที่มีรูปแบบคงที่

#### ขั้นตอนที่ 1: โหลดเอกสารของคุณ
เริ่มต้นด้วยการโหลดไฟล์ DOCX ของคุณลงใน Aspose.Words `Document` วัตถุ:

```python
doc = aw.Document(MY_DIR + "Rendering.docx")
```

#### ขั้นตอนที่ 2: สร้างตัวเลือกการบันทึก
การเริ่มต้น `XamlFixedSaveOptions` เพื่อปรับแต่งกระบวนการบันทึก:

```python
options = aw.saving.XamlFixedSaveOptions()
```

#### ขั้นตอนที่ 3: กำหนดค่าการจัดการทรัพยากร
กำหนดวิธีการจัดการทรัพยากรที่เชื่อมโยงโดยการตั้งค่า `resources_folder`- `resources_folder_alias`และฟังก์ชั่นการโทรกลับ

```python
callback = ExXamlFixedSaveOptions.ResourceUriPrinter()
options.resource_saving_callback = callback
options.resources_folder = ARTIFACTS_DIR + "XamlFixedResourceFolder"
options.resources_folder_alias = ARTIFACTS_DIR + "XamlFixedFolderAlias"

# ตรวจสอบให้แน่ใจว่ามีโฟลเดอร์นามแฝงอยู่ก่อนที่จะบันทึกทรัพยากร
os.makedirs(options.resources_folder_alias)
```

#### ขั้นตอนที่ 4: บันทึกเอกสาร
สุดท้ายให้บันทึกเอกสารของคุณโดยใช้ตัวเลือกที่กำหนดค่าไว้:

```python
doc.save(ARTIFACTS_DIR + "XamlFixedSaveOptions.resource_folder.xaml", options)
```

### การติดตาม URI ของทรัพยากร
เพื่อตรวจสอบและพิมพ์ URI ทรัพยากรในระหว่างการแปลง ให้ใช้ `ResourceUriPrinter` คลาสที่นับและบันทึกแต่ละ URI

#### ภาพรวม
กลไกการโทรกลับจะช่วยติดตามทรัพยากรที่สร้างขึ้นในระหว่างการดำเนินการบันทึก

#### การใช้งานคลาส Callback
นี่คือวิธีกำหนดการเรียกกลับแบบกำหนดเองเพื่อจัดการการประหยัดทรัพยากร:

```python
class ResourceUriPrinter(aw.saving.IResourceSavingCallback):
    """Counts and prints URIs of resources created during conversion."""
    
    def __init__(self):
        self.resources = []  # ประเภท: รายการ[str]
    
    def resource_saving(self, args: aw.saving.ResourceSavingArgs):
        self.resources.append(f"Resource \"{args.resource_file_name}\"\n\t{args.resource_file_uri}")
        
        # เปลี่ยนเส้นทางสตรีมไปยังโฟลเดอร์นามแฝง
        args.resource_stream = open(args.resource_file_uri, 'wb')
        args.keep_resource_stream_open = False
```

### เคล็ดลับการแก้ไขปัญหา
- ตรวจสอบให้แน่ใจว่าไดเร็กทอรีทั้งหมดที่ระบุไว้ใน `resources_folder` และ `resources_folder_alias` มีอยู่ก่อนที่จะรันสคริปต์ของคุณ
- ตรวจสอบเส้นทางไฟล์อีกครั้งเพื่อดูว่ามีข้อผิดพลาดด้านการพิมพ์หรือไม่
