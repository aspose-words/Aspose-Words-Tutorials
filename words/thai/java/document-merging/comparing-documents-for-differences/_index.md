## วิธีเปรียบเทียบไฟล์ DOCX – บทนำ

เคยสงสัย **วิธีเปรียบเทียบ docx** ไฟล์และตรวจพบการเปลี่ยนแปลงทุกอย่างระหว่างเอกสาร Word สองไฟล์หรือไม่? บางทีคุณอาจกำลังแก้ไขสัญญา, ตรวจทานรายงานร่วมกัน, หรือจำเป็นต้องตรวจสอบเอกสารทางกฎหมาย การเปรียบเทียบด้วยตนเองนั้นน่าเบื่อและเสี่ยงต่อความผิดพลาด แต่ด้วย Aspose.Words for Java การทำงานอัตโนมัติจะง่ายดาย ไลบรารีนี้ช่วยให้คุณเปรียบเทียบเอกสาร, ไฮไลท์การแก้ไข, และรวมการเปลี่ยนแปลงด้วยเพียงไม่กี่บรรทัดของโค้ด

## นำเข้าแพ็กเกจ

เพื่อใช้ Aspose.Words, คุณต้องนำเข้าคลาสที่จำเป็น ด้านล่างเป็นรายการ import ที่ต้องใช้:

```java
import com.aspose.words.*;
import java.util.Date;
```

ตรวจสอบให้แน่ใจว่าแพ็กเกจเหล่านี้ถูกเพิ่มเข้าไปใน dependencies ของโปรเจกต์ของคุณแล้ว

ในส่วนนี้ เราจะอธิบายกระบวนการเป็นขั้นตอนง่าย ๆ

## ขั้นตอนที่ 1: ตั้งค่าเอกสารของคุณ

เริ่มต้นด้วยการมีเอกสารสองไฟล์: หนึ่งไฟล์เป็นต้นฉบับและอีกไฟล์เป็นเวอร์ชันที่แก้ไข นี่คือตัวอย่างการสร้างเอกสารในหน่วยความจำ:

```java
Document doc1 = new Document();
DocumentBuilder builder = new DocumentBuilder(doc1);
builder.writeln("This is the original document.");

Document doc2 = new Document();
builder = new DocumentBuilder(doc2);
builder.writeln("This is the edited document.");
```

โค้ดนี้สร้างเอกสารสองไฟล์ในหน่วยความจำพร้อมเนื้อหาเบื้องต้น คุณยังสามารถโหลดไฟล์ Word ที่มีอยู่โดยใช้ `new Document("path/to/document.docx")` ได้เช่นกัน

## ขั้นตอนที่ 2: ตรวจสอบการแก้ไขที่มีอยู่แล้ว

การแก้ไขในเอกสาร Word แสดงถึงการติดตามการเปลี่ยนแปลง ก่อนทำการเปรียบเทียบ, ตรวจให้แน่ใจว่าไม่มีการแก้ไขที่มีอยู่แล้วในเอกสารใดเอกสารหนึ่ง:

```java
if (doc1.getRevisions().getCount() == 0 && doc2.getRevisions().getCount() == 0) {
    System.out.println("No revisions found. Proceeding with comparison...");
}
```

หากพบการแก้ไข, คุณอาจต้องยอมรับหรือปฏิเสธก่อนดำเนินการต่อ

## ขั้นตอนที่ 3: เปรียบเทียบเอกสาร

ใช้เมธอด `compare` เพื่อค้นหาความแตกต่าง เมธอดนี้เปรียบเทียบเอกสารเป้าหมาย (`doc2`) กับเอกสารต้นฉบับ (`doc1`):

```java
doc1.compare(doc2, "AuthorName", new Date());
```

โดย:
- **AuthorName** คือชื่อของผู้ทำการเปลี่ยนแปลง  
- **Date** คือเวลาที่ทำการเปรียบเทียบ  

## ขั้นตอนที่ 4: ประมวลผลการแก้ไข

หลังจากการเปรียบเทียบ, Aspose.Words จะสร้างการแก้ไขในเอกสารต้นฉบับ (`doc1`) มาวิเคราะห์การแก้ไขเหล่านี้:

```java
for (Revision r : doc1.getRevisions()) {
    System.out.println("Revision type: " + r.getRevisionType());
    System.out.println("Node type: " + r.getParentNode().getNodeType());
    System.out.println("Changed text: " + r.getParentNode().getText());
}
```

ลูปนี้ให้ข้อมูลรายละเอียดของแต่ละการแก้ไข เช่น ประเภทของการเปลี่ยนแปลงและข้อความที่ได้รับผลกระทบ

## ขั้นตอนที่ 5: ยอมรับการแก้ไขทั้งหมด

หากต้องการให้เอกสารต้นฉบับ (`doc1`) ตรงกับเอกสารเป้าหมาย (`doc2`), ยอมรับการแก้ไขทั้งหมด:

```java
doc1.getRevisions().acceptAll();
```

โค้ดนี้จะอัปเดต `doc1` ให้สะท้อนการเปลี่ยนแปลงทั้งหมดจาก `doc2`

## ขั้นตอนที่ 6: บันทึกเอกสารที่อัปเดต

สุดท้าย, บันทึกเอกสารที่อัปเดตลงดิสก์:

```java
doc1.save("Document.Compare.docx");
```

เพื่อยืนยันการเปลี่ยนแปลง, โหลดเอกสารใหม่และตรวจสอบว่าไม่มีการแก้ไขค้างอยู่:

```java
doc1 = new Document("Document.Compare.docx");
if (doc1.getRevisions().getCount() == 0) {
    System.out.println("Documents are now identical.");
}
```

## ขั้นตอนที่ 7: ตรวจสอบความเท่าเทียมของเอกสาร

เพื่อให้แน่ใจว่าเอกสารตรงกันจริง ๆ, เปรียบเทียบข้อความธรรมดาของพวกมัน:

```java
if (doc1.getText().trim().equals(doc2.getText().trim())) {
    System.out.println("Documents are equal.");
}
```

หากข้อความตรงกัน, ยินดีด้วย – คุณได้เปรียบเทียบและซิงค์เอกสารสำเร็จแล้ว!

## ทำไมเรื่องนี้ถึงสำคัญ

การเข้าใจ **วิธีเปรียบเทียบ docx** ด้วยโปรแกรมช่วยประหยัดเวลาเป็นจำนวนมากในงานด้านกฎหมาย, การเผยแพร่, และสภาพแวดล้อมการทำงานร่วมกัน แทนการเลื่อนดูการแก้ไขด้วยตนเอง คุณสามารถทำอัตโนมัติ, สร้างบันทึกการตรวจสอบ, และรวมตรรกะการเปรียบเทียบเข้าไปในระบบจัดการเอกสารขนาดใหญ่ได้

## ข้อผิดพลาดทั่วไป & เคล็ดลับ

- **การแก้ไขที่มีอยู่แล้ว:** ควรล้างหรือยอมรับการแก้ไขที่มีอยู่ก่อนเรียก `compare` มิฉะนั้น API อาจถือว่าเป็นการเปลี่ยนแปลงใหม่  
- **เอกสารขนาดใหญ่:** สำหรับไฟล์ที่ใหญ่มาก, พิจารณาเพิ่มขนาด heap ของ JVM เพื่อหลีกเลี่ยง `OutOfMemoryError`  
- **การกำหนดสไตล์การแก้ไขแบบกำหนดเอง:** คุณสามารถปรับ `RevisionOptions` เพื่อเปลี่ยนวิธีการแสดงการแทรก/ลบ (เช่น สีไฮไลท์)
